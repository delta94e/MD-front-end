## §3. Error Handling: rejectReactions + .catch()

### 3.1 Bài Viết: The Reject Path

> Will: _"Suppose instead of getting back cute puppy, we got back some error — you're not allowed access to this adorable video! Our Promise style gives us a tool: rejectReactions — another hidden array. If we get an error, it automatically runs the REJECTED functions, not the fulfilled ones."_

```
PROMISE ERROR HANDLING:
═══════════════════════════════════════════════════════════════

  PROMISE OBJECT — FULL PICTURE:
  ┌──────────────────────────────────────────────────┐
  │ futureData (Promise):                            │
  │                                                  │
  │  [[result]]: undefined → data OR error!          │
  │                                                  │
  │  [[fulfillReactions]]:  [display]                │
  │   ↑ .then(display)                              │
  │   → Runs on SUCCESS! ✅                         │
  │                                                  │
  │  [[rejectReactions]]:   [handleError]            │
  │   ↑ .catch(handleError)                         │
  │   → Runs on ERROR! ❌                           │
  │                                                  │
  └──────────────────────────────────────────────────┘

  HOW IT WORKS:
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  fetch("tiktok.com/will")                        │
  │       │                                          │
  │  Response comes back:                            │
  │       │                                          │
  │  ┌────┴────┐                                     │
  │  │         │                                     │
  │  ▼         ▼                                     │
  │ SUCCESS   ERROR                                  │
  │  ✅        ❌                                     │
  │  │         │                                     │
  │  ▼         ▼                                     │
  │ result =  result =                               │
  │ "cute     "Access                                │
  │  puppy"    denied!"                              │
  │  │         │                                     │
  │  ▼         ▼                                     │
  │ RUN        RUN                                   │
  │ fulfill    reject                                │
  │ Reactions  Reactions                              │
  │ (.then)    (.catch)                              │
  │                                                  │
  │ display    handleError                           │
  │ ("cute     ("Access                              │
  │  puppy")    denied!")                             │
  │                                                  │
  └──────────────────────────────────────────────────┘

  CODE:
  ┌──────────────────────────────────────────────────┐
  │ const futureData = fetch("tiktok.com/will");     │
  │                                                  │
  │ futureData                                       │
  │   .then(display)           // ← fulfillReactions │
  │   .catch(handleError);     // ← rejectReactions  │
  │                                                  │
  │ → .then = push to fulfillReactions (hidden!)    │
  │ → .catch = push to rejectReactions (hidden!)    │
  │ → Error? → skip .then → run .catch!            │
  │ → Success? → run .then → skip .catch!          │
  └──────────────────────────────────────────────────┘
```

## §5. BONUS: AbortSignal — Cancel Background Work!

### 5.1 Bài Viết: "What If We Don't Want To Wait?"

> Will: _"What was added two years after fetch was the ability to ABORT a fetch background task — when we have a timeout, something that determines we don't want to keep it going. We don't want to wait for a response because the server may NEVER give us a response."_

```
AbortSignal — CANCEL FETCH!:
═══════════════════════════════════════════════════════════════

  PROBLEM:
  ┌──────────────────────────────────────────────────┐
  │ const data = fetch("tiktok.com/will");           │
  │                                                  │
  │ → What if TikTok server NEVER responds?         │
  │ → What if it takes 30 seconds?                  │
  │ → What if user navigates away?                  │
  │ → We're STUCK waiting forever!                  │
  │ → Need a way to CANCEL! ✋                      │
  └──────────────────────────────────────────────────┘

  SOLUTION — AbortSignal (2017 + 2022):
  ┌──────────────────────────────────────────────────┐
  │ // 2022: AbortSignal.timeout()                   │
  │ const signal = AbortSignal.timeout(200);         │
  │                                                  │
  │ fetch("tiktok.com/will", { signal })             │
  │   .then(display)                                 │
  │   .catch(handleError);                           │
  │                                                  │
  │ → If response takes > 200ms → ABORT!            │
  │ → Error passed to .catch → TimeoutError!        │

```

  const signal = AbortSignal.timeout(200);

  ┌──────────────────────────────────────────────────┐

  │   → Network request (talk to TikTok!)           │
  │   → Timer (200ms countdown!)                    │

  ┌──────────────────────────────────────────────────┐

  ┌──────────────────────────────────────────────────┐

```javascript
// CHỨNG MINH 2: Build AbortSignal from scratch!

class MyAbortSignal {
  constructor() {
    this.aborted = false;
    this.reason = undefined;
    this._listeners = [];
  }

  // Listen for abort:
  addEventListener(event, fn) {
    if (event === "abort") this._listeners.push(fn);
  }

  // Trigger abort:
  _abort(reason) {
    if (this.aborted) return; // already aborted!
    this.aborted = true;
    this.reason = reason;
    this._listeners.forEach((fn) => fn({ type: "abort", reason }));
  }

  // Static: create with timeout (2022 API!)
  static timeout(ms) {
    const signal = new MyAbortSignal();
    // TWO-PRONGED:
    // ① JS: return signal object immediately!
    // ② Browser: set up timer!
    setTimeout(() => {
      signal._abort(new Error(`TimeoutError: ${ms}ms exceeded`));
    }, ms);
    return signal;
  }
}

class MyPromise {
  constructor() {
    this._value = undefined;
    this._status = "pending";
    this._fulfillReactions = [];
    this._rejectReactions = [];
  }

  then(onFulfill) {
    if (this._status === "fulfilled") {
      onFulfill(this._value);
    } else {
      this._fulfillReactions.push(onFulfill);
    }
    return this;
  }

  // .catch → push to rejectReactions!
  catch(onReject) {
    if (this._status === "rejected") {
      onReject(this._value);
    } else {
      this._rejectReactions.push(onReject);
    }
    return this;
  }

  _resolve(value) {
    this._value = value;
    this._status = "fulfilled";
    this._fulfillReactions.forEach((fn) => fn(value));
  }

  _reject(error) {
    this._value = error;
    this._status = "rejected";
    this._rejectReactions.forEach((fn) => fn(error));
  }
}

// Two-pronged fetch with abort support:
function myFetch(url, options = {}) {
  const promise = new MyPromise();

  // Set up network request:
  const networkDelay = 270; // simulate TikTok response
  const networkTimer = setTimeout(() => {
    console.log(`[Network] Response from ${url}!`);
    promise._resolve("cute puppy 🐶");
  }, networkDelay);

  // If signal provided, set up abort bond:
  if (options.signal) {
    const signal = options.signal;

    // Already aborted?
    if (signal.aborted) {
      clearTimeout(networkTimer);
      promise._reject(signal.reason);
      return promise;
    }

    // Listen for future abort:
    signal.addEventListener("abort", (event) => {
      console.log(`[Abort] Signal triggered! Canceling network!`);
      clearTimeout(networkTimer); // CANCEL network request!
      promise._reject(event.reason);
    });
  }

  return promise;
}

// SCENARIO A: No abort (response < timeout):
console.log("═══ SCENARIO A: No abort ═══");
const signal1 = MyAbortSignal.timeout(500); // 500ms timeout
myFetch("tiktok.com/will", { signal: signal1 })
  .then((data) => console.log("Success:", data))
  .catch((err) => console.log("Error:", err.message));

// Response at 270ms < timeout 500ms → SUCCESS!
// Output: "Success: cute puppy 🐶"
```

## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Error Handling

```
5 WHYS: TẠI SAO CẦN rejectReactions?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao cần separate array cho errors?
  └→ Vì errors cần XỬ LÝ KHÁC data!
     Error → log, retry, show UI message!
     Data → display, process, store!

  WHY ②: Tại sao .catch() thay vì check result?
  └→ Vì result is HIDDEN!
     Can't check: if (promise.result instanceof Error)
     Need .catch() to access rejectReactions!

  WHY ③: Tại sao auto-skip fulfillReactions on error?
  └→ Vì SAFETY! Running display("error") = crash!
     Auto-skip = guaranteed correct path!

  WHY ④: Tại sao AbortSignal cần timer bond?
  └→ Vì network may NEVER respond!
     Timer = deadline! "If no response by 200ms → abort!"
     Without timer → infinite waiting!

  WHY ⑤: Tại sao abort cancels network (not just ignores)?
  └→ Vì RESOURCE MANAGEMENT!
     Cancel = free up browser network thread!
     Just ignoring = wastes bandwidth + memory!
```

### 9.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — PROMISE STATES (COMPLETE):
═══════════════════════════════════════════════════════════════

  ECMAScript 25.6 — Promise Objects:

  3 STATES:
  → pending: initial, not settled
  → fulfilled: operation succeeded
  → rejected: operation failed

  TRANSITIONS (irreversible!):
  ┌──────────────────────────────────────────────────┐
  │          ┌───→ fulfilled ──→ run fulfillReactions│
  │ pending ─┤                                       │
  │          └───→ rejected ───→ run rejectReactions │
  │                                                  │
  │ Once settled → NEVER changes again!             │
  │ Can't go: fulfilled → rejected!                 │
  │ Can't go: rejected → fulfilled!                 │
  │ Can't go: settled → pending!                    │
  └──────────────────────────────────────────────────┘

  AbortController API (DOM spec):

  → controller.abort(): trigger abort event
  → AbortSignal.timeout(ms): auto-abort after ms
  → signal.aborted: boolean (read-only)
  → signal.reason: Error or undefined
```

### 9.5 Pattern ⑤: Reverse Engineering

```javascript
// PROVE: .catch handles rejected promises

// Manual reject:
const p = new Promise((resolve, reject) => {
  reject(new Error("Server down!"));
});

p.then((data) => console.log("This will NOT run!")).catch((err) =>
  console.log("Caught:", err.message),
);
// → "Caught: Server down!"

// AbortSignal abort:
const controller = new AbortController();
setTimeout(() => controller.abort(), 50); // abort after 50ms

fetch("https://jsonplaceholder.typicode.com/todos/1", {
  signal: controller.signal,
})
  .then((r) => console.log("Success — might not reach here!"))
  .catch((err) => console.log("Aborted:", err.name));
// → "Aborted: AbortError"
```
