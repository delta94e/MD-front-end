## §6. Tại Sao Lại Bỏ Function Vào Function Khác Rồi Return Ra?

### 6.1 Bài Viết: Câu Hỏi Hợp Lý

Will thừa nhận: _"Why the heck would I put this in a different function first, only to return it out? We'll come to that."_

Trong ví dụ này — CHƯA CÓ LÝ DO! Code này tương đương:

```javascript
// CÁCH 1: Return từ function (rắc rối hơn):
function createFunction() {
  function multiplyBy2(num) {
    return num * 2;
  }
  return multiplyBy2;
}
const generatedFunc = createFunction();
generatedFunc(3); // 6

// CÁCH 2: Khai báo trực tiếp (đơn giản hơn):
function multiplyBy2(num) {
  return num * 2;
}
multiplyBy2(3); // 6

// → CÙNG KẾT QUẢ! Vậy tại sao dùng cách 1?
```

```
CÂU TRẢ LỜI: CLOSURE!
═══════════════════════════════════════════════════════════════

  Ví dụ hiện tại: CHƯA CÓ lý do rõ ràng.
  → Chỉ để HIỂU CƠ CHẾ return function.
  → Nền tảng cho bước tiếp theo!

  Bước tiếp (closure):
  ┌──────────────────────────────────────────────────────┐
  │ function createFunction() {                          │
  │   let counter = 0;           ← local data!          │
  │   function increment() {                             │
  │     counter++;               ← access local data!   │
  │     return counter;                                  │
  │   }                                                  │
  │   return increment;                                  │
  │ }                                                    │
  │                                                      │
  │ → increment "MANG THEO" counter khi return ra!       │
  │ → counter = PERSISTENT MEMORY!                       │
  │ → ĐÂY là lý do đặt function bên trong function!     │
  │ → Đó chính là CLOSURE!                               │
  └──────────────────────────────────────────────────────┘

  → Ví dụ hiện tại: multiplyBy2 KHÔNG dùng data
    từ createFunction → nên chưa thấy lý do.
  → Khi inner function dùng data từ outer → MAGIC!
```

## §7. Namespace Pollution & Modularization

Will đề cập lý do thứ hai — ít nổi bật hơn nhưng quan trọng:

> _"In part because we want to be able to label stuff just inside of that function's execution context and NOT have it POLLUTE the global namespace."_

```
NAMESPACE POLLUTION — VẤN ĐỀ:
═══════════════════════════════════════════════════════════════

  ❌ Mọi thứ ở global:
  ┌──────────────────────────────────────────────────────┐
  │ GLOBAL MEMORY:                                       │
  │   result = ...        │                              │
  │   data = ...          │ → 1000 biến!                 │
  │   temp = ...          │ → Trùng tên!                 │
  │   output = ...        │ → "Ai dùng result?"          │
  │   count = ...         │ → Override nhầm!             │
  │   result = ...        │ → 💥 Conflict!               │
  └──────────────────────────────────────────────────────┘

  ✅ Modularize bằng functions:
  ┌──────────────────────────────────────────────────────┐
  │ GLOBAL MEMORY:                                       │
  │   processData: ƒ      │ → Chỉ vài labels!           │
  │   renderUI: ƒ          │                              │
  │                                                      │
  │ Bên trong processData:                               │
  │   result = ...         │ → Local! Không conflict!    │
  │   temp = ...           │ → Local! Tự do đặt tên!    │
  │                                                      │
  │ Bên trong renderUI:                                  │
  │   result = ...         │ → Local! "result" KHÁC!     │
  │   temp = ...           │ → Không sợ trùng!           │
  └──────────────────────────────────────────────────────┘
```

## §8. Tự Implement: Return Function Patterns

```javascript
// PATTERN 1: Factory Function — Tạo function tuỳ chỉnh

function createMultiplier(factor) {
  // factor = data từ outer → closure (sẽ thấy ở bài sau!)
  function multiplier(num) {
    return num * factor;
  }
  return multiplier;
}

const double = createMultiplier(2);
const triple = createMultiplier(3);
const tenX = createMultiplier(10);

double(5); // 10
triple(5); // 15
tenX(5); // 50

// → MỖI lần gọi createMultiplier = function MỚI!
// → Mỗi function "nhớ" factor khác nhau (closure!)
```

```javascript
// PATTERN 2: Return function — NO closure data (like the lesson)

function createGreeter() {
  function greet(name) {
    return `Hello, ${name}!`;
  }
  return greet;
}

const myGreeter = createGreeter(); // ← chạy 1 lần, lấy ƒ ra!
myGreeter("Will"); // "Hello, Will!"
myGreeter("Chris"); // "Hello, Chris!"

// → myGreeter = ƒ(name) { return `Hello, ${name}!`; }
// → KHÔNG CÒN liên kết với createGreeter!
// → Giống ví dụ trong bài: chưa có closure data
```

```javascript
// PATTERN 3: Chứng minh "KHÔNG CÒN LIÊN KẾT"

function createFunction() {
  console.log("createFunction ĐANG CHẠY!");
  function inner() {
    return "I'm the inner function";
  }
  return inner;
}

const myFunc = createFunction();
// Console: "createFunction ĐANG CHẠY!" ← CHẠY 1 LẦN!

myFunc(); // "I'm the inner function" ← KHÔNG log!

// → createFunction() chỉ chạy ĐÚng 1 LẦN (ở dòng const)!
// → Mỗi lần gọi myFunc → KHÔNG quay lại createFunction!
// → Nếu có liên kết → phải thấy console.log mỗi lần!
// → Nhưng KHÔNG! → CHỨNG MINH không còn liên kết!
```

```javascript
// PATTERN 4: Trace bằng tay — function EVALUATE vs function REFERENCE

function outer() {
  return function (x) {
    return x + 1;
  };
}

// CASE A: Gọi outer() → nhận FUNCTION DEFINITION:
const fn = outer(); // fn = ƒ(x) { return x + 1; }
fn(5); // 6

// CASE B: KHÔNG gọi outer → nhận FUNCTION REFERENCE:
const ref = outer; // ref = ƒ outer() { ... } ← CHÍNH outer!
ref(); // ƒ(x) { return x + 1; } ← phải gọi 2 lần!
ref()(5); // 6 ← outer()... (5) = 2 bước!

// SỰ KHÁC BIỆT:
//   outer()  → CHẠY outer → return code bên trong
//   outer    → CHỈ THAM CHIẾU đến outer, KHÔNG chạy!
//   () = PARENS = "đi chạy ngay!" (tạo EC!)
```

## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Return Function

```
5 WHYS: TẠI SAO RETURN FUNCTION TỪ FUNCTION?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao cần tạo function bên trong function?
  └→ Để function bên trong TRUY CẬP biến local
     của function bên ngoài. (Chưa thấy ở bài này,
     sẽ thấy ở closure bài sau!)

  WHY ②: Tại sao return ra thay vì gọi bên trong?
  └→ Vì muốn function SỐNG SÓT sau khi outer kết thúc!
     Return ra → lưu vào label mới → dùng mãi!

  WHY ③: Tại sao không khai báo ở global luôn?
  └→ Vì khi khai báo bên trong outer, inner function
     "BẮT" (close over) biến local của outer!
     Đây là CƠ CHẾ CỐT LÕI = CLOSURE!

  WHY ④: Tại sao JS cho phép return function?
  └→ Vì first-class objects! Functions = values!
     Giống return số 6 hay string "hello",
     return ƒ(num){return num*2} HOÀN TOÀN hợp lệ!

  WHY ⑤: Tại sao bài này chưa có closure?
  └→ Vì Will dạy FOUNDATION trước! Bước 1: hiểu
     cơ chế return function + KHÔNG CÒN liên kết.
     Bước 2: thêm data → closure!
     ⭐ "The most distorting behavior" = phải hiểu
        trước khi học closure!
```

### 9.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — RETURN FUNCTION = GÌ?
═══════════════════════════════════════════════════════════════

  RETURN trong JS = EVALUATE expression → gửi value ra:

  return 5;              → evaluate "5" → gửi số 5
  return "hello";        → evaluate "hello" → gửi string
  return [1, 2, 3];      → evaluate "[1,2,3]" → gửi array
  return multiplyBy2;    → evaluate "multiplyBy2" → gửi ???

  multiplyBy2 = LABEL trỏ đến FUNCTION OBJECT trong memory!

  Khi evaluate:
  ┌──────────────────────────────────────────────────────┐
  │ return multiplyBy2;                                  │
  │        ↓                                             │
  │ Tìm "multiplyBy2" trong local memory                 │
  │        ↓                                             │
  │ Tìm thấy: ƒ(num) { return num * 2; }                │
  │        ↓                                             │
  │ GỬI function object ĐÓ ra ngoài!                    │
  │        ↓                                             │
  │ Label "multiplyBy2" BỊ XOÁ (cùng local memory)      │
  │ NHƯNG function object SỐNG SÓT (được gán label mới!)│
  └──────────────────────────────────────────────────────┘

  → return = "copy value, send out, forget label"
  → Function object = VALUE → có thể return!
```

### 9.3 Pattern ③: Trade-off Analysis

```
TRADE-OFFS: RETURN FUNCTION vs DEFINE GLOBALLY
═══════════════════════════════════════════════════════════════

  ┌──────────────────┬──────────────────┬──────────────────┐
  │ Tiêu chí          │ Define globally  │ Return from fn   │
  ├──────────────────┼──────────────────┼──────────────────┤
  │ Simplicity       │ ✅ Dễ đọc       │ ❌ Phức tạp hơn │
  ├──────────────────┼──────────────────┼──────────────────┤
  │ Closure access   │ ❌ Không có     │ ✅ Có persistent │
  │                  │ persistent mem  │ memory!          │
  ├──────────────────┼──────────────────┼──────────────────┤
  │ Namespace        │ ❌ Pollute      │ ✅ Clean global  │
  │                  │ global          │ namespace        │
  ├──────────────────┼──────────────────┼──────────────────┤
  │ Customization    │ ❌ Fixed        │ ✅ Factory       │
  │                  │ behavior        │ → custom per call│
  ├──────────────────┼──────────────────┼──────────────────┤
  │ Encapsulation    │ ❌ All public   │ ✅ Private data  │
  └──────────────────┴──────────────────┴──────────────────┘

  Khi nào dùng return function:
  → Cần closure (persistent data) → LUÔN
  → Cần factory (tạo nhiều variants) → LUÔN
  → Cần encapsulation (private state) → LUÔN
  → Simple utility function → define globally = OK!
```

### 9.5 Pattern ⑤: Reverse Engineering & Implementation

```javascript
// TỰ BUILD: Chứng minh function = OBJECT, có thể inspect

function createFunction() {
  function multiplyBy2(num) {
    return num * 2;
  }
  return multiplyBy2;
}

const generatedFunc = createFunction();

// INSPECT function object:
console.log(typeof generatedFunc); // "function"
console.log(generatedFunc.name); // "multiplyBy2" ← JS nhớ tên GỐC!
console.log(generatedFunc.length); // 1 (số params)
console.log(generatedFunc.toString()); // "function multiplyBy2(num) {..."

// → Dù label = "generatedFunc", JS vẫn nhớ .name = "multiplyBy2"!
// → .name là METADATA, KHÔNG phải "liên kết" về createFunction!
// → generatedFunc vẫn KHÔNG quay lại createFunction khi chạy!

// CHỨNG MINH: function = object, có thể thêm properties:
generatedFunc.customProp = "hello";
console.log(generatedFunc.customProp); // "hello"
// → Functions = objects + [[Call]] = có properties!

// CHỨNG MINH: generatedFunc COMPLETELY INDEPENDENT:
// Xoá createFunction → generatedFunc VẪN CHẠY!
```
