## §1. Hint Parameter — "number" / "string" / "default"

### 1.1 Bài Viết: JS Engine Tự Truyền Hint!

> Will: _"Our JavaScript engine, when it kicks off and adds parens — it also automatically inserts some information about what the primitive specifics are. In this case the string 'number'. It's a parameter name we can call whatever we want."_

```
HINT — JS ENGINE TỰ TRUYỀN:
═══════════════════════════════════════════════════════════════

  function coerce(hint) {  // "hint" = tên tùy ý!
    // hint ĐƯỢC JS ENGINE TỰ TRUYỀN VÀO!
    // Developer KHÔNG gọi coerce("number")!
    // JS engine calls: coerce("number") automatically!
  }

  userStored[Symbol.toPrimitive] = coerce;

  +userStored  →  JS engine chạy: coerce("number")
                                         ↑
                                    AUTO-INSERTED!

  OPERATOR → HINT:
  ┌──────────────────────────────────────────────────┐
  │ OPERATOR/CONTEXT         → HINT INSERTED         │
  ├──────────────────────────────────────────────────┤
  │ - * / % ** (math)        → "number"              │
  │ < > <= >= (comparison)   → "number"              │
  │ unary +                  → "number"              │
  │ Number(obj)              → "number"              │
  ├──────────────────────────────────────────────────┤
  │ `${obj}` (template)      → "string"              │
  │ String(obj)              → "string"              │
  │ [].join(obj)             → "string"              │
  ├──────────────────────────────────────────────────┤
  │ + (binary, with string)  → "default"             │
  │ == (loose equality)      → "default"             │
  └──────────────────────────────────────────────────┘

  Will: "It literally inserts the string 'number'.
  As long as we put a PLACEHOLDER in our function,
  which you can call anything — could be 'hello my
  name is Chris' — and then go: if that equals
  'number', return 105."
```

    }
    if (hint === "string") {
      return "User";     // cho template literals!

## §3. Symbols = Semi-Hidden Properties — ES6 Feature

### 3.1 Bài Viết: "Introducing Symbols"

> Will: _"Introducing Symbols. ES6 feature for adding SEMI-HIDDEN properties to objects. These are labels. Unique identifiers that we cannot write out directly and so will not override developers' existing code."_

```
SYMBOLS — SEMI-HIDDEN PROPERTIES:
═══════════════════════════════════════════════════════════════

  "SEMI-HIDDEN" — tại sao?
  ┌──────────────────────────────────────────────────┐
  │ HIDDEN:                                          │
  │ ✅ Object.keys(obj) → KHÔNG bao gồm Symbols!    │
  │ ✅ for...in loop → KHÔNG bao gồm Symbols!       │
  │ ✅ JSON.stringify(obj) → KHÔNG bao gồm Symbols! │
  │ ✅ Không thể viết trực tiếp (@@toPrimitive)     │
  │                                                  │
  │ ACCESSIBLE (semi!):                              │
  │ ✅ Object.getOwnPropertySymbols(obj) → CÓ!      │
  │ ✅ obj[Symbol.toPrimitive] → CÓ THỂ access!     │
  │ ✅ Reflect.ownKeys(obj) → CÓ (strings + symbols)│
  └──────────────────────────────────────────────────┘

  → KHÔNG PHẢI fully hidden (như [[DateValue]])!
  → KHÔNG PHẢI fully visible (như "name", "id")!
  → = SEMI-HIDDEN! Perfect middle ground!

  const obj = { name: "Will", id: 105 };
  obj[Symbol.toPrimitive] = function(hint) { return 105; };

  VISIBLE LAYER:
  → Object.keys(obj) → ["name", "id"]
  → Đây là DATA USER quan tâm!

  HIDDEN LAYER:
  → Object.getOwnPropertySymbols(obj) → [Symbol(Symbol.toPrimitive)]
  → Đây là META BEHAVIOR!
  → Separation of concerns!
```

## §4. Tại Sao Symbols? — 30 Năm Backward Compatibility!

### 4.1 Bài Viết: "30 Years of History"

> Will: _"When you're dealing with a language that has to deal with 30 years of history, it can only make changes that are largely non-breaking backwards. So how do you do that? You go: brand new data type that doesn't clash with all prior ones."_

```
TẠI SAO SYMBOLS? — BACKWARD COMPATIBILITY:
═══════════════════════════════════════════════════════════════

  VẤN ĐỀ — nếu dùng STRING "toPrimitive":
  ┌──────────────────────────────────────────────────┐
  │ // Developer viết code năm 2005:                 │
  │ const myObj = {                                  │
  │   toPrimitive: function() {                      │
  │     return "This is MY function!";               │
  │   }                                              │
  │ };                                               │
  │                                                  │
  │ // Năm 2015, JS thêm feature mới:               │
  │ // "Nếu object có .toPrimitive → chạy nó!"      │
  │                                                  │
  │ +myObj → JS grabs .toPrimitive → CHẠY NÓ!       │
  │ → Developer: "WTF?! Tôi KHÔNG MUỐN điều này!"  │
  │ → CODE BỊ BREAK!                                │
  │ → TRIỆU websites bị ảnh hưởng!                 │
  └──────────────────────────────────────────────────┘

  GIẢI PHÁP — SYMBOL (brand new type!):
  ┌──────────────────────────────────────────────────┐
  │ // Developer code năm 2005 → KHÔNG BỊ ẢNH HƯỞNG│
  │ const myObj = {                                  │
  │   toPrimitive: function() { ... }  // STRING key │
  │ };                                               │
  │                                                  │
  │ // JS 2015 checks: @@toPrimitive (SYMBOL key!)  │
  │ // @@toPrimitive ≠ "toPrimitive"!               │
  │ // KHÔNG CLASH!                                  │
  │ // Old code KHÔNG BỊ BREAK!                     │
  │                                                  │
  │ // Nếu muốn opt-in:                             │
  │ myObj[Symbol.toPrimitive] = function(hint) {     │
  │   return 42; // EXPLICITLY added!                │
  │ };                                               │
  └──────────────────────────────────────────────────┘

  Will: "@@toPrimitive on a Symbol WON'T override
  developers' existing toPrimitive. JavaScript won't
  go and use their existing toPrimitive property
  that was written as a string."

  → Symbol = GUARANTEED không clash!
  → Unique data type = safe upgrade path!
  → 30 years of code = PROTECTED!
```

## §7. Explicit Control > Implicit Coercion — Tổng Kết Triết Lý

### 7.1 Bài Viết: Will's Philosophy

> Will: _"What we have to default to is EXPLICIT CONTROL. Whether that's a call to Number to ensure the data off the DOM is definitely a number, or controlling explicitly our coercion from object to primitive. We want CONTROL — yes, we benefit from readable code, but it's not unpredictable."_

```
TRIẾT LÝ CỦA WILL — EXPLICIT > IMPLICIT:
═══════════════════════════════════════════════════════════════

  ① FLEXIBILITY:
  → Coercion CÓ ÍCH: "3" * 7 = 21 (tiện!)
  → Data tự động chuyển đổi khi cần!

  ② PREDICTABILITY:
  → Coercion NGUY HIỂM: "3" + 7 = "37" (gotcha!)
  → Implicit = unpredictable!

  ③ SOLUTION = EXPLICIT CONTROL:
  → Number(input) trước khi math!
  → === thay vì ==!
  → Symbol.toPrimitive cho objects!

  Will: "Every API call, every operator triggers
  coercions. But we default to EXPLICIT CONTROL."

  HIERARCHY CỦA CONTROL:
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  BEST:  TypeScript (compile-time type safety!)  │
  │    │                                             │
  │    ├── Explicit: Number(), String(), ===          │
  │    │                                             │
  │    ├── Semi-explicit: Symbol.toPrimitive          │
  │    │                                             │
  │    ├── Implicit but known: - * / (toNumber)      │
  │    │                                             │
  │  WORST: Implicit + surprising: + (dual role!)    │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

// CHỨNG MINH 1: Hint-aware coercion — full conditional logic

const user = {
  name: "Will",
  id: 105,
  email: "will@example.com",

  [Symbol.toPrimitive](hint) {
    console.log(`  [toPrimitive] hint = "${hint}"`);

    switch (hint) {
      case "number":
        return this.id; // cho math!
      case "string":
        return `${this.name} (${this.email})`; // cho display!
      default:
        return this.id; // cho == và +
    }
  },
};

// Number context:
console.log(+user); // 105 (hint = "number")
console.log(user - 100); // 5  (hint = "number")
console.log(user > 100); // true (hint = "number")

// String context:
console.log(`${user}`); // "Will (will@example.com)" (hint = "string")
console.log(String(user)); // "Will (will@example.com)" (hint = "string")

// Default context:
console.log(user + "!"); // "105!" (hint = "default" → 105 → "105!")
console.log(user == 105); // true (hint = "default")

// CHỨNG MINH 2: Symbol-based "private" properties

const _balance = Symbol("balance");
const _pin = Symbol("pin");

function createBankAccount(owner, initialBalance, pin) {
  return {
    owner,
    [_balance]: initialBalance,
    [_pin]: pin,

    [Symbol.toPrimitive](hint) {
      if (hint === "number") return this[_balance];
      if (hint === "string") return `Account(${this.owner})`;
      return this[_balance];
    },

    deposit(amount) {
      this[_balance] += amount;
      return this;
    },

    withdraw(amount, enteredPin) {
      if (enteredPin !== this[_pin]) {
        throw new Error("Wrong PIN!");
      }
      this[_balance] -= amount;
      return this;
    },
  };
}

// Symbol.iterator — control for...of!
const range = {
  from: 1,
  to: 5,

  [Symbol.iterator]() {
    let current = this.from;
    const last = this.to;
    return {
      next() {
        if (current <= last) {
          return { value: current++, done: false };
        }
        return { done: true };
      },
    };
  },
};

for (const num of range) {
  console.log(num); // 1, 2, 3, 4, 5
}
console.log([...range]); // [1, 2, 3, 4, 5]

// Symbol.hasInstance — control instanceof!
const EvenNumber = {
  [Symbol.hasInstance](num) {
    return typeof num === "number" && num % 2 === 0;
  },
};

console.log(5 instanceof EvenNumber); // false!

  [Symbol.toStringTag]: "MyAwesomeLibrary",
};

// CHỨNG MINH 4: Full authentication system — Part 4 finale!

function createAuthSystem() {
  // Symbol-based private fields
  const _timestamp = Symbol("timestamp");
  const _hash = Symbol("hash");

  function simpleHash(name, id) {
    // Simple hash for demo (real auth uses bcrypt etc.)
    let h = 0;
    const str = `${name}:${id}`;
    for (let i = 0; i < str.length; i++) {
      h = (h * 31 + str.charCodeAt(i)) | 0;
    }
    return h;
  }

  function createUserRecord(name, id) {
    const record = { name, id };

    record[_timestamp] = Date.now();
    record[_hash] = simpleHash(name, id);

    // Metaprogramming: control coercion!
    record[Symbol.toPrimitive] = function (hint) {
      if (hint === "number") return this[_hash];
      if (hint === "string") return `User(${this.name}#${this.id})`;
      return this[_hash]; // default → hash for comparison!
    };

    return record;
  }

  function authenticate(stored, submitted) {
    // Using our coercion pipeline!
    if (+stored === +submitted) {
      // Check timestamp — prevent double submit!
      const timeDiff = submitted[_timestamp] - stored[_timestamp];
      if (timeDiff < 2000) {
        return { success: false, reason: "Too fast! Double submit?" };
      }
      return { success: true, user: String(stored) };
    }
    return { success: false, reason: "Credentials don't match!" };
  }

  return { createUserRecord, authenticate };
}

// Usage:
const auth = createAuthSystem();
const stored = auth.createUserRecord("Will", 105);
const submitted = auth.createUserRecord("Will", 105);

// Visible:
console.log(Object.keys(stored)); // ["name", "id"]
// Hidden (Symbol) properties = metaprogramming!

// Authenticate:
const result = auth.authenticate(stored, submitted);
console.log(result);
// { success: true, user: "User(Will#105)" }
// or { success: false, reason: "Too fast!" } if < 2s

// Wrong user:
const hacker = auth.createUserRecord("Evil", 666);
console.log(auth.authenticate(stored, hacker));
// { success: false, reason: "Credentials don't match!" }

## §10. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 10.1 Pattern ①: 5 Whys — Symbols & Metaprogramming

```
5 WHYS: TẠI SAO METAPROGRAMMING QUAN TRỌNG?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao cần override language behavior?
  └→ Vì default behavior KHÔNG phù hợp mọi trường hợp!
     +{id:105} → NaN (useless!) thay vì 105 (useful!)

  WHY ②: Tại sao Symbols thay vì strings?
  └→ Vì 30 năm code history = KHÔNG THỂ thêm
     string property mà không break existing code!
     Symbol = new type = GUARANTEED no clash!

  WHY ③: Tại sao "semi-hidden" thay vì fully hidden?
  └→ Vì developers CẦN debug và inspect!
     Object.getOwnPropertySymbols() = accessible!
     Nhưng for...in, Object.keys = hidden = neat API!

  WHY ④: Tại sao hint thay vì separate functions?
  └→ Vì 1 function + hint = SIMPLER than 3 functions!
     obj[Symbol.toNumber], obj[Symbol.toString]... = messy!
     1 central function + conditional = CLEAN!

  WHY ⑤: Tại sao TypeScript vẫn cần hiểu coercion?
  └→ Vì TypeScript compiles → JavaScript!
     Runtime = vẫn JS! DOM = vẫn strings!
     3rd-party libraries = vẫn có implicit coercion!
     Understanding WHY = better debugging!
```

// TỰ BUILD: Complete metaprogramming toolkit

// 1. Type-safe smart objects with full coercion control
function SmartNumber(value, label) {
  return {
    value,
    label,
    [Symbol.toPrimitive](hint) {
      if (hint === "number") return this.value;
      if (hint === "string") return `${this.label}: ${this.value}`;
      return this.value;
    },
    [Symbol.iterator]() {
      let i = 0;
      const max = this.value;
      return {
        next() {
          return i < max ? { value: i++, done: false } : { done: true };
        },
      };
    },
    [Symbol.toStringTag]: "SmartNumber",
  };
}

const price = SmartNumber(42, "Price");

// Coercion:
console.log(price + 8); // 50 (number!)
console.log(`${price}`); // "Price: 42" (string!)

// Iteration:
console.log([...price]); // [0,1,2,...,41] (iterator!)

// toString tag:
console.log(Object.prototype.toString.call(price));
// "[object SmartNumber]"

// 2. Simulate Proxy-like metaprogramming
function createValidatedObject(schema) {
  const data = {};
  const handler = {
    set(target, prop, value) {
      if (schema[prop]) {
        const { type, validate } = schema[prop];
        if (typeof value !== type) {
          throw new TypeError(`${prop} must be ${type}, got ${typeof value}`);
        }
        if (validate && !validate(value)) {
          throw new RangeError(`${prop} failed validation!`);
        }
      }
      target[prop] = value;
      return true;
    },
  };
  return new Proxy(data, handler);
}

const user = createValidatedObject({
  name: { type: "string", validate: (s) => s.length > 0 },
  id: { type: "number", validate: (n) => n > 0 },
  email: { type: "string", validate: (s) => s.includes("@") },
});

user.name = "Will"; // ✅
user.id = 105; // ✅
