```javascript
// CHỨNG MINH 2: .prototype vs __proto__

function UserCreator(name, score) {
  this.name = name;
  this.score = score;
}

// Add methods to .prototype:
UserCreator.prototype.increment = function () {
  this.score++;

// Create with new:
const user1 = new UserCreator("Ari", 3);

// The bond:
console.log(user1.__proto__ === UserCreator.prototype); // true!
// → user1's __proto__ LINKS to UserCreator.prototype!

```

```javascript
// CHỨNG MINH 3: Simulate what new does behind the scenes!

function simulateNew(Constructor, ...args) {
  // ① Create empty object:
  const obj = {};

  // ② Bond to Constructor.prototype:
  Object.setPrototypeOf(obj, Constructor.prototype);
  // same as: obj.__proto__ = Constructor.prototype

  // ③ Run constructor with this = obj:
  const result = Constructor.apply(obj, args);

  // ④ Return obj (unless constructor returns an object):
  return typeof result === "object" && result !== null ? result : obj;
}

function UserCreator(name, score) {
  this.name = name;
  this.score = score;
}
UserCreator.prototype.increment = function () {
  this.score++;
};

// These two are IDENTICAL:
const userNew = new UserCreator("Ari", 3);
const userSim = simulateNew(UserCreator, "Ari", 3);

console.log(userNew.name); // "Ari"
console.log(userSim.name); // "Ari"

userNew.increment();
console.log(userNew.score); // 4
userSim.increment();
console.log(userSim.score); // 4

// Same bond:
console.log(userNew.__proto__ === UserCreator.prototype); // true
console.log(userSim.__proto__ === UserCreator.prototype); // true
```

```javascript
// CHỨNG MINH 5: Solution 1 vs 2 vs 3

// Solution 1: Factory function
function createUser1(name, score) {
  const newUser = {};
  newUser.name = name;
  newUser.score = score;
  newUser.increment = function () {
    newUser.score++;
  };
  return newUser;
}

// Solution 2: Object.create + prototype chain
const store2 = {
  increment() {
    this.score++;
  },
};
function createUser2(name, score) {
  const newUser = Object.create(store2);
  newUser.name = name;
  newUser.score = score;
  return newUser;
}

// Solution 3: new keyword
function UserCreator3(name, score) {
  this.name = name;
  this.score = score;
}
UserCreator3.prototype.increment = function () {
  this.score++;
};

const u1 = createUser1("A", 1); // factory
const u2 = createUser2("B", 2); // Object.create
const u3 = new UserCreator3("C", 3); // new!

console.log(u1.score); // 2 ✅
u2.increment();
console.log(u2.score); // 3 ✅
u3.increment();
console.log(u3.score); // 4 ✅

const u1b = createUser1("X", 0);

const u3b = new UserCreator3("X", 0);
console.log(u3.increment === u3b.increment); // true! (shared) ✅
```

## §8. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 8.1 Pattern ①: 5 Whys — The new Keyword

```
5 WHYS: TẠI SAO CẦN new?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao cần keyword mới?
  └→ Vì Solution 2 quá verbose! 6 bước mỗi lần!
     "Do I REALLY need to do all that stuff?"
     Other languages automate this!

  WHY ②: Tại sao new tự tạo object?
  └→ Vì mọi constructor function đều cần tạo object!
     100% of the time → automate!

  WHY ③: Tại sao dùng .prototype cho function store?
  └→ Vì mọi function đã CÓ SẴN .prototype!
     Không cần tạo separate store!
     new tự biết link __proto__ → .prototype!

  WHY ④: Tại sao reuse this keyword?
  └→ Vì... design choice (không phải tốt nhất!)
     Will (sarcastic): "Genius design."
     Same word, completely different meanings!

  WHY ⑤: Tại sao nguy hiểm nếu quên new?
  └→ Vì this = window thay vì new object!
     Properties leak to global scope!
     Function returns undefined!
     "It ain't going to work the way you want!"
```

### 8.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — WHAT new ACTUALLY DOES:
═══════════════════════════════════════════════════════════════

  ECMAScript spec — [[Construct]](args):

  1. Let obj = OrdinaryCreateFromConstructor(F)
     → Create new ordinary object
     → Set obj.[[Prototype]] = F.prototype

  2. Let result = F.[[Call]](obj, args)
     → Run F with this = obj
     → this.name = name assigns to obj!

  3. If result is an Object → return result!
     If result is NOT Object → return obj!
     → Auto-return the created object!

  FUNCTION-OBJECT DUALITY:
  → Every function = callable object
  → Has [[Call]] (function behavior)
  → Has properties (object behavior)
  → .prototype = default property (empty {})
  → Parens = access [[Call]]
  → Dot = access object properties
```

### 8.5 Pattern ⑤: Reverse Engineering

```javascript
// PROVE: Every function has .prototype by default

function foo() {}

const baz = () => {}; // arrow!
```
