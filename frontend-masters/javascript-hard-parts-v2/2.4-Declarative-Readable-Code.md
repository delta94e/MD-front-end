## §1. Functions Are First-Class Objects — Nền Tảng Mọi Thứ

### 1.1 Bài Viết: Tại Sao Truyền Function Được?

Will Sentance đặt câu hỏi then chốt: _"How am I able to pass in function code as an input to another function?"_

Câu trả lời: **"In JavaScript, functions are first-class objects."**

Điều này nghĩa là gì? Functions trong JavaScript **LÀ objects** — chúng không phải syntax đặc biệt, không phải keyword đặc biệt. Chúng là **DỮ LIỆU**, giống như số, chuỗi, mảng.

> Will: _"They are just objects. Now they have extra ability — the ability to be called, to be invoked — but they are just objects."_

```
FUNCTIONS = FIRST-CLASS OBJECTS:
═══════════════════════════════════════════════════════════════

  Trong JavaScript, FUNCTIONS = OBJECTS + khả năng GỌI:

  ┌──────────────────────────────────────────────────────┐
  │                    OBJECT                             │
  │                                                      │
  │  ┌────────────────────────────────────────────────┐  │
  │  │ ✅ Lưu vào biến                                │  │
  │  │ ✅ Truyền làm argument                         │  │
  │  │ ✅ Return từ function                           │  │
  │  │ ✅ Gán làm property trên object                │  │
  │  │ ✅ Lưu trong array                              │  │
  │  │ ✅ Có properties và methods riêng               │  │
  │  └────────────────────────────────────────────────┘  │
  │                      +                                │
  │  ┌────────────────────────────────────────────────┐  │
  │  │ ⭐ EXTRA: Khả năng được GỌI bằng ()            │  │
  │  │   → Đây là điều DUY NHẤT khác biệt!            │  │
  │  └────────────────────────────────────────────────┘  │
  │                      =                                │
  │                   FUNCTION                            │
  └──────────────────────────────────────────────────────┘

  So sánh:
  ┌──────────────┬─────────────┬──────────────┐
  │ Khả năng      │ Object {}   │ Function ƒ   │
  ├──────────────┼─────────────┼──────────────┤
  │ Lưu vào biến │ ✅ const a  │ ✅ const f   │
  │ Truyền arg   │ ✅ fn(obj)  │ ✅ fn(func)  │
  │ Return       │ ✅ return o │ ✅ return f  │
  │ Property     │ ✅ o.key    │ ✅ f.name    │
  │ Gọi bằng () │ ❌          │ ✅ f()       │
  └──────────────┴─────────────┴──────────────┘

  → Function = Object mà có thể GỌI!
  → Không có gì "magic" — chỉ là value!
```

### 1.2 Tại Sao Điều Này Không Hiển Nhiên?

Trong nhiều ngôn ngữ khác, functions **KHÔNG phải** objects. Will nói: _"In many languages, this violates our idea of what declaring or saving a function even is doing."_

```
SO SÁNH: JS vs NGÔN NGỮ KHÁC:
═══════════════════════════════════════════════════════════════

  JAVA (trước Java 8):
  ┌──────────────────────────────────────────────────────┐
  │ // Function KHÔNG phải value → KHÔNG truyền được!    │
  │ // Phải bọc trong anonymous class:                   │
  │ button.addActionListener(new ActionListener() {      │
  │   public void actionPerformed(ActionEvent e) {       │
  │     System.out.println("Clicked!");                  │
  │   }                                                  │
  │ });                                                  │
  │ // → 5 DÒNG chỉ để truyền 1 dòng functionality!     │
  └──────────────────────────────────────────────────────┘

  JAVASCRIPT:
  ┌──────────────────────────────────────────────────────┐
  │ // Function = value → truyền trực tiếp!              │
  │ button.addEventListener("click", () => {             │
  │   console.log("Clicked!");                           │
  │ });                                                  │
  │ // → GỌN, TRỰC TIẾP, KHÔNG cần wrapper class!       │
  └──────────────────────────────────────────────────────┘
```

## §2. 4 Khả Năng Của First-Class Functions

### 2.1 Bài Viết: Functions Có Thể Làm Mọi Thứ Objects Làm

Will liệt kê 4 khả năng cốt lõi:

```
4 KHẢ NĂNG CỦA FIRST-CLASS FUNCTIONS:
═══════════════════════════════════════════════════════════════

  ① ASSIGN to variables (Gán vào biến):
  ┌──────────────────────────────────────────────────────┐
  │ const double = function(x) { return x * 2; };       │
  │ const triple = (x) => x * 3;  // arrow function     │
  │                                                      │
  │ → Will: "the ever more popular style of declaring    │
  │   functions that involves assigning to a label"      │
  └──────────────────────────────────────────────────────┘

  ② PROPERTIES on objects (Gán làm property → gọi là METHOD):
  ┌──────────────────────────────────────────────────────┐
  │ const obj = {                                        │
  │   name: "Jun",                                       │
  │   greet: function() { return "Hi " + this.name; }   │
  │ };                                                   │
  │ obj.greet();  // "Hi Jun"                            │
  │                                                      │
  │ → Chris trả lời Will: "A function on an object       │
  │   is known as a METHOD."                             │
  └──────────────────────────────────────────────────────┘

  ③ PASS as input (Truyền làm argument → HOF pattern):
  ┌──────────────────────────────────────────────────────┐
  │ copyArrayAndManipulate([1,2,3], multiplyBy2);        │
  │                                  ^^^^^^^^^^^^        │
  │                              truyền function vào!    │
  │                                                      │
  │ → Will: "We can pass functions in as inputs.          │
  │   Nobody's worried about passing an object in.       │
  │   Functions are just like objects!"                   │
  └──────────────────────────────────────────────────────┘

  ④ RETURN as output (Trả từ function → CLOSURE!):
  ┌──────────────────────────────────────────────────────┐
  │ function createMultiplier(factor) {                  │
  │   return function(x) { return x * factor; };         │
  │ }                ↑ return FUNCTION!                   │
  │                                                      │
  │ const double = createMultiplier(2);                  │
  │ double(5); // 10                                     │
  │                                                      │
  │ → Will: "We can also return out a function as an     │
  │   output value. When we do that, we get CLOSURE."    │
  │ → PHẦN 3 sẽ học chi tiết!                            │
  └──────────────────────────────────────────────────────┘
```

## §3. Higher-Order Function vs Callback — Thuật Ngữ Chính Xác

### 3.1 Bài Viết: Ai Là HOF, Ai Là Callback?

Will hỏi Austin: _"Which was our higher-order function?"_ → `copyArrayAndManipulate`. _"Which was our callback?"_ → `multiplyBy2`.

**Định nghĩa chính thức:**

- **Higher-Order Function (HOF)**: function **NHẬN** function khác làm argument, hoặc **RETURN** function
- **Callback**: function được **TRUYỀN VÀO** HOF

```
HOF vs CALLBACK — AI LÀ AI?
═══════════════════════════════════════════════════════════════

  copyArrayAndManipulate([1,2,3], multiplyBy2);
  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
  │                                │
  │                                └── CALLBACK
  │                                    = function được truyền VÀO
  │                                    = "hàm con", "hàm phụ"
  │                                    = multiplyBy2
  │
  └── HIGHER-ORDER FUNCTION
      = function NHẬN function khác
      = "hàm cha", "hàm điều phối"
      = copyArrayAndManipulate

  ·················································

  QUAN TRỌNG — Will nhấn mạnh:
  ┌──────────────────────────────────────────────────────┐
  │ "Surely we had to declare it with a special keyword  │
  │  to say 'this is a special higher-order function'?"  │
  │                                                      │
  │ → KHÔNG! Không có keyword đặc biệt!                 │
  │ → copyArrayAndManipulate khai báo BÌNH THƯỜNG        │
  │ → Nó "trở thành" HOF vì NHẬN function làm argument  │
  │ → "Nothing intrinsically different about them."      │
  │               — Will Sentance                        │
  └──────────────────────────────────────────────────────┘
```

### 3.2 HOF = Nhận HOẶC Return Function

```
HAI LOẠI HIGHER-ORDER FUNCTION:
═══════════════════════════════════════════════════════════════

  LOẠI 1: NHẬN function làm argument (đã học)
  ┌──────────────────────────────────────────────────────┐
  │ function doSomething(data, callback) {               │
  │   return callback(data);                             │
  │ }                                                    │
  │                                                      │
  │ Ví dụ: map, filter, reduce, forEach, addEventListener│
  └──────────────────────────────────────────────────────┘

  LOẠI 2: RETURN function (sẽ học ở Closure!)
  ┌──────────────────────────────────────────────────────┐
  │ function createGreeter(greeting) {                   │
  │   return function(name) {                            │
  │     return greeting + ", " + name;                   │
  │   };       ↑ RETURN function!                        │
  │ }                                                    │
  │                                                      │
  │ const sayHi = createGreeter("Hi");                   │
  │ sayHi("Jun");  // "Hi, Jun"                          │
  │                                                      │
  │ → createGreeter = HOF (return function)              │
  │ → sayHi = function được RETURN ra                    │
  │ → Cơ chế: CLOSURE (phần 3!)                          │
  └──────────────────────────────────────────────────────┘

  LOẠI 3: CẢ HAI — Nhận VÀ Return function
  ┌──────────────────────────────────────────────────────┐
  │ function compose(f, g) {                             │
  │   return function(x) {                               │
  │     return f(g(x));  ← NHẬN f, g + RETURN function   │
  │   };                                                 │
  │ }                                                    │
  │ const doubleAndAddOne = compose(addOne, double);     │
  │ doubleAndAddOne(3); // addOne(double(3)) = 7         │
  └──────────────────────────────────────────────────────┘
```

## §4. Callback vs Handler vs Lambda — Nhiều Tên, Một Bản Chất

### 4.1 Bài Viết: Tên Gọi Phụ Thuộc Ngữ Cảnh

Will thừa nhận: _"I don't know if I love the term callback function here."_ Tại sao? Vì "callback" gợi ý function sẽ được **gọi lại sau** (call back later) — nhưng ở đây `multiplyBy2` chạy **NGAY LẬP TỨC**.

```
CÙNG MỘT THỨ — NHIỀU TÊN GỌI:
═══════════════════════════════════════════════════════════════

  ┌───────────────┬────────────────────────────────────────┐
  │ Tên gọi        │ Khi nào dùng                          │
  ├───────────────┼────────────────────────────────────────┤
  │ CALLBACK      │ Thuật ngữ CHUNG cho function truyền   │
  │               │ vào HOF. Phổ biến nhất.                │
  │               │ Will: "They are still technically      │
  │               │ called callback functions."            │
  ├───────────────┼────────────────────────────────────────┤
  │ HANDLER       │ Khi callback XỬ LÝ mỗi phần tử.      │
  │               │ Will: "It's handling what to do to     │
  │               │ each element."                         │
  │               │ VD: event handler, request handler     │
  ├───────────────┼────────────────────────────────────────┤
  │ TRANSFORMATION│ Khi callback BIẾN ĐỔI data.           │
  │ FUNCTION      │ Will: "It's doing the transformation." │
  │               │ VD: map callback = transformer         │
  ├───────────────┼────────────────────────────────────────┤
  │ PREDICATE     │ Khi callback return true/false.        │
  │               │ VD: filter callback = test function    │
  ├───────────────┼────────────────────────────────────────┤
  │ REDUCER       │ Khi callback gộp accumulator + element.│
  │               │ VD: reduce callback                    │
  ├───────────────┼────────────────────────────────────────┤
  │ LAMBDA        │ Khi callback KHÔNG CÓ TÊN (anonymous).│
  │               │ Will: "especially where we pass them   │
  │               │ in without a label"                    │
  │               │ VD: arr.map(x => x * 2) ← lambda      │
  ├───────────────┼────────────────────────────────────────┤
  │ ARGUMENT      │ Will: "You could call it an argument   │
  │ FUNCTION      │ function" — function dùng như argument. │
  └───────────────┴────────────────────────────────────────┘

  ⚠️ Will's WARNING about "callback":
  "I think it's a STRANGE NAME because it suggests that they
   come back later. No no no, they run RIGHT THERE."

  → Ở HOF: callback chạy NGAY (synchronous)
  → Ở Async: callback chạy SAU (asynchronous) → đúng nghĩa "call back"!
```

## §5. Declarative vs Imperative — Code Đọc Như Tiếng Anh

### 5.1 Bài Viết: Declarative = "Cái Gì", Imperative = "Như Thế Nào"

Will so sánh 2 styles viết code:

> _"Declarative code is far more readable. In under the hood of any declarative code, there has to be some imperative code of how actually to do it."_

**Imperative** = bạn viết TỪNG BƯỚC cách làm (how to do it)
**Declarative** = bạn mô tả KẾT QUẢ mong muốn (what to do)

```
IMPERATIVE vs DECLARATIVE:
═══════════════════════════════════════════════════════════════

  ❌ IMPERATIVE — Mô tả CÁCH LÀM (how):
  ┌──────────────────────────────────────────────────────┐
  │ const output = [];                                   │
  │ for (let i = 0; i < array.length; i++) {             │
  │   output.push(array[i] * 2);                         │
  │ }                                                    │
  │                                                      │
  │ Đọc: "Tạo mảng rỗng. Duyệt từ 0 đến length.        │
  │       Lấy phần tử tại vị trí i. Nhân đôi.           │
  │       Push vào mảng output."                         │
  │                                                      │
  │ → Phải đọc 4 dòng → hiểu → mới biết "à, nhân đôi!" │
  └──────────────────────────────────────────────────────┘

  ✅ DECLARATIVE — Mô tả KẾT QUẢ (what):
  ┌──────────────────────────────────────────────────────┐
  │ const result = copyArrayAndManipulate(                │
  │   [1, 2, 3],                                         │
  │   multiplyBy2                                        │
  │ );                                                   │
  │                                                      │
  │ Đọc: "Copy mảng [1,2,3] và manipulate bằng           │
  │       multiplyBy2."                                  │
  │                                                      │
  │ → ĐỌC 1 DÒNG → biết ngay: nhân đôi mảng!           │
  └──────────────────────────────────────────────────────┘

  Will: "What do you want the output to be?
  I want you to take this data, copy array, and
  MANIPULATE it via this TRANSFORMATION."
  → Code ĐỌC NHƯ TIẾNG ANH!
```

### 5.2 Insight: Declarative = Abstractions TRÊN Imperative

```
DECLARATIVE LUÔN ẨN IMPERATIVE BÊN DƯỚI:
═══════════════════════════════════════════════════════════════

  Bạn viết (DECLARATIVE):
  ┌──────────────────────────────────────────────────────┐
  │ [1, 2, 3].map(x => x * 2);   // [2, 4, 6]          │
  │                                                      │
  │ → Đọc: "Map mảng bằng cách nhân đôi mỗi phần tử."  │
  │ → KHÔNG thấy for loop, KHÔNG thấy push!             │
  └──────────────────────────────────────────────────────┘

  Bên dưới .map() (IMPERATIVE):
  ┌──────────────────────────────────────────────────────┐
  │ Array.prototype.map = function(callback) {           │
  │   const output = [];                                 │
  │   for (let i = 0; i < this.length; i++) {            │
  │     output.push(callback(this[i], i, this));         │
  │   }                                                  │
  │   return output;                                     │
  │ };                                                   │
  │                                                      │
  │ → Tất cả imperative code VẪN CHẠY — chỉ ẨN đi!     │
  │ → Ai đó đã viết for loop 1 LẦN                      │
  │ → Bạn dùng API declarative MÃI MÃI                  │
  └──────────────────────────────────────────────────────┘

  Phép ẩn dụ:
  ┌──────────────────────────────────────────────────────┐
  │ Imperative = Nấu ăn từ nguyên liệu thô              │
  │ → "Cắt hành, phi dầu, cho thịt vào, xào 5 phút..." │
  │                                                      │
  │ Declarative = Gọi món tại nhà hàng                   │
  │ → "Cho tôi phở bò tái!" ← KHÔNG cần biết cách nấu  │
  │ → Bên bếp (bên dưới) VẪN phải cắt hành, phi dầu... │
  └──────────────────────────────────────────────────────┘
```

### 5.3 So Sánh Đọc Code

```
VÍ DỤ THỰC TẾ — DECLARATIVE READABLE HƠN:
═══════════════════════════════════════════════════════════════

  BÀI TOÁN: Từ danh sách users, lấy email của adult active users

  ❌ IMPERATIVE:
  const emails = [];
  for (let i = 0; i < users.length; i++) {
    if (users[i].age >= 18) {
      if (users[i].active) {
        emails.push(users[i].email);
      }
    }
  }
  // → Phải đọc 7 dòng, 2 nested if! 😩

  ✅ DECLARATIVE (HOF chain):
  const emails = users
    .filter(user => user.age >= 18)
    .filter(user => user.active)
    .map(user => user.email);
  // → Đọc: "Filter adults, filter active, map to email." ✅
  // → ĐỌC NHƯ TIẾNG ANH!

  ⭐ Will: "These are very popular in INTERVIEWS."
     → Interviewer thích declarative code!
     → Dễ đọc, dễ maintain, dễ review!
```

## §6. HOF = Xương Sống Của Async JavaScript

### 6.1 Bài Viết: Callbacks Ở Khắp Nơi

Will teaser: _"Callbacks are a vital part of the asynchronous JavaScript pattern. They sit under promises, under async/await."_

```
HOF/CALLBACK = NỀN TẢNG CỦA MODERN JS:
═══════════════════════════════════════════════════════════════

  ① ARRAY METHODS (đã học):
  → map, filter, reduce, forEach, find, some, every
  → Callback chạy NGAY (synchronous)

  ② EVENT HANDLING:
  → button.addEventListener("click", handleClick)
  → HOF = addEventListener, Callback = handleClick
  → Callback chạy KHI event xảy ra (asynchronous)

  ③ TIMERS:
  → setTimeout(callback, 1000)
  → Callback chạy SAU 1 giây (asynchronous)
  → "Called BACK" → đúng nghĩa callback!

  ④ PROMISES:
  → fetch(url).then(callback)
  → Callback chạy KHI data về (asynchronous)

  ⑤ ASYNC/AWAIT:
  → Syntax sugar cho Promises
  → BÊN DƯỚI vẫn là callback pattern!

  ┌─────────────────────────────────────────────────────┐
  │ Will: "These are the BACKBONE of asynchronous        │
  │ JavaScript, which is what makes JavaScript so        │
  │ powerful and popular, why it's so important."         │
  └─────────────────────────────────────────────────────┘
```

## §7. Teaser: Return Function → Closure

Will giới thiệu: _"When we return out a function as an output value from another function, we get the most obscure feature of JavaScript — known as CLOSURE."_

```
CLOSURE PREVIEW (Phần 3 sẽ học):
═══════════════════════════════════════════════════════════════

  function createCounter() {
    let count = 0;
    return function() {      ← Return FUNCTION!
      count++;               ← Function "nhớ" count!
      return count;
    };
  }

  const counter = createCounter();
  counter(); // 1
  counter(); // 2
  counter(); // 3

  ❓ Tại sao count không bị xoá khi createCounter() return?
  → Vì function được return "mang theo" local memory!
  → Đó là CLOSURE — phần 3 của Hard Parts!

  ┌─────────────────────────────────────────────────────┐
  │ First-class functions cho phép:                     │
  │ ① Truyền function VÀO    → HOF pattern (đã học)    │
  │ ② Return function RA      → CLOSURE (sắp học!)     │
  │                                                     │
  │ Will: "The MOST OBSCURE feature of JavaScript."     │
  │ → "The most elegant and beautiful feature" (từ P0)  │
  └─────────────────────────────────────────────────────┘
```

## §8. Tự Implement: First-Class Functions In Action

```javascript
// CHỨNG MINH: Functions = Objects — có properties!

function greet(name) {
  return "Hello, " + name;
}

// Functions CÓ properties (vì là objects):
console.log(greet.name); // "greet" ← tên function
console.log(greet.length); // 1       ← số parameters
console.log(typeof greet); // "function" (nhưng thực ra là object!)

// Thêm property tuỳ ý:
greet.callCount = 0;
greet.description = "Greets a person by name";

console.log(greet.callCount); // 0
console.log(greet.description); // "Greets a person by name"
// → Functions lưu data NHƯ objects! ✅
```

```javascript
// TỰ IMPLEMENT: 4 KHẢ NĂNG CỦA FIRST-CLASS FUNCTIONS

// ① Assign to variable:
const double = (x) => x * 2;

// ② Property on object (METHOD):
const mathUtils = {
  double: (x) => x * 2,
  square: (x) => x * x,
  negate: (x) => -x,
};
mathUtils.double(5); // 10
mathUtils.square(5); // 25

// ③ Pass as argument (HOF + CALLBACK):
function applyToAll(array, transform) {
  const result = [];
  for (let i = 0; i < array.length; i++) {
    result.push(transform(array[i]));
  }
  return result;
}

  return function (x) {

const quadruple = createMultiplier(4);

quadruple(5); // 20
// → createMultiplier = HOF (return function)
// → triple, quadruple = CLOSURES!
```

## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — First-Class Functions

```
5 WHYS: TẠI SAO FUNCTIONS LÀ FIRST-CLASS OBJECTS?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao JS cho functions là first-class?
  └→ Vì Brendan Eich lấy cảm hứng từ Scheme (1975),
     nơi functions = values từ ngày đầu.

  WHY ②: Tại sao Scheme coi functions = values?
  └→ Vì Scheme dựa trên Lambda Calculus (Alonzo Church, 1936)
     → toán học coi function = đối tượng thao tác.

  WHY ③: Tại sao Java/C++ KHÔNG coi functions = values?
  └→ Vì chúng thiết kế theo OOP paradigm — mọi thứ phải
     gắn với CLASS. Function = method, thuộc về class.
     Không tồn tại độc lập → không phải value.

  WHY ④: Tại sao first-class functions quan trọng cho JS?
  └→ Vì JS cần HOFs cho event handling (DOM), async (AJAX),
     và callbacks. Web browser = event-driven → CẦN khả
     năng truyền function đi khắp nơi.

  WHY ⑤: Tại sao HOFs thay đổi cách viết code?
  └→ Vì chúng cho phép DECLARATIVE style:
     → Code đọc được (what, not how)
     → DRY (1 HOF = vô hạn behaviors)
     → Composable (pipe, compose, chain)
     ⭐ Đây là nền tảng của Functional Programming!
```

```javascript
// TỰ BUILD: DECLARATIVE QUERY LANGUAGE

// Bước 1: Imperative primitives
function where(array, predicate) {
  const result = [];
  for (let i = 0; i < array.length; i++) {
    if (predicate(array[i])) result.push(array[i]);
  }
  return result;
}

function select(array, mapper) {
  const result = [];
  for (let i = 0; i < array.length; i++) {
    result.push(mapper(array[i]));
  }
  return result;
}

// Bước 2: Declarative pipeline (compose HOFs)
function pipe(initialValue, ...operations) {
  let result = initialValue;
  for (const op of operations) {
    result = op(result);
    // Mỗi operation = HOF nhận data, return data mới
  }
  return result;
}

// Bước 3: Sử dụng — ĐỌC NHƯ SQL!
const products = [
  { name: "Laptop", price: 1000, inStock: true },
  { name: "Mouse", price: 25, inStock: true },
  { name: "Monitor", price: 500, inStock: false },
  { name: "Keyboard", price: 75, inStock: true },
];

const result = pipe(
  products,
  (data) => where(data, (p) => p.inStock), // WHERE inStock
  (data) => where(data, (p) => p.price < 200), // WHERE price < 200
  (data) => select(data, (p) => p.name), // SELECT name
);

console.log(result); // ["Mouse", "Keyboard"]
// → Đọc: WHERE inStock AND price < 200, SELECT name
// → Declarative! Mọi step = HOF + callback!
```
