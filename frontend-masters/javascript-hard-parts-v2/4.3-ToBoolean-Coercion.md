## §2. toBoolean Coercion — Falsy Values

### 2.1 Bài Viết: Coercion Thứ Ba

> Will: _"We can actually, in our conditional, rely on the final coercion — the toBoolean. That's going to allow us to immediately coerce to a boolean."_

```
TO BOOLEAN — FALSY VALUES:
═══════════════════════════════════════════════════════════════

  JavaScript có ĐÚNG 7 giá trị FALSY:

  ┌──────────────────┬──────────────┬──────────────────────┐
  │ Giá trị          │ toBoolean    │ Giải thích           │
  ├──────────────────┼──────────────┼──────────────────────┤
  │ false            │ false        │ Đã là false!         │
  │ 0                │ false        │ Number zero          │
  │ -0               │ false        │ Negative zero        │
  │ 0n               │ false        │ BigInt zero          │
  │ "" (empty string)│ false        │ Chuỗi rỗng          │
  │ null             │ false        │ Không có giá trị     │
  │ undefined        │ false        │ Chưa được gán        │
  │ NaN              │ false        │ Not a Number         │
  └──────────────────┴──────────────┴──────────────────────┘

  TẤT CẢ GIÁ TRỊ KHÁC = TRUTHY:

  ┌──────────────────┬──────────────┬──────────────────────┐
  │ Giá trị          │ toBoolean    │ Bất ngờ?             │
  ├──────────────────┼──────────────┼──────────────────────┤
  │ "0"              │ true ⚠️      │ STRING "0" ≠ NUMBER 0│
  │ "false"          │ true ⚠️      │ STRING "false" ≠ false│
  │ [] (empty array) │ true ⚠️      │ Object = always true │
  │ {} (empty object)│ true ⚠️      │ Object = always true │
  │ function(){}     │ true         │ Function = object    │
  │ " " (space)      │ true ⚠️      │ NOT empty!           │
  │ "abc"            │ true         │ Non-empty string     │
  │ 42               │ true         │ Non-zero number      │
  │ -1               │ true         │ Non-zero number      │
  │ Infinity         │ true         │ Non-zero number      │
  └──────────────────┴──────────────┴──────────────────────┘

  QUY TẮC ĐƠN GIẢN:
```

## §5. == Loose Equality — CŨNG Coerce! "" == 0 → true! 🔥

### 5.1 Bài Viết: Equality Cũng Coerce!

> Will: _"Even our equality — our check 'is this thing the same' — even IT kicks off, in this case, toNumber coercion! It coerced our empty string to 0. The very thing we want to say 'yeah no problem' — but not allow us to reach 'want to donate'."_

```
== LOOSE EQUALITY — "" == 0 → true! 💀
═══════════════════════════════════════════════════════════════

  donation = ""  (user chưa gõ)

  if (donation == 0) {
    console.log("No donation, no problem."); // ← ĐẾN ĐÂY!
  } else {
    console.log("Want to donate?"); // ← KHÔNG BAO GIỜ ĐẾN!
  }

  TRACE:
  ┌──────────────────────────────────────────────────┐
  │ "" == 0                                          │
  │                                                  │
  │ == thấy: LEFT = "" (string), RIGHT = 0 (number) │
  │ → COERCE! toNumber("") = 0                      │
  │ → 0 == 0 → true!                                │
  │                                                  │
  │ → "No donation, no problem." ← SAI!             │
  │ → User CHƯA gõ gì nhưng app nói "no problem"!  │
  │ → Không bao giờ hỏi "Want to donate?" 💀        │
  └──────────────────────────────────────────────────┘

  ⚠️ VẤN ĐỀ:
  ┌──────────────────────────────────────────────────┐
  │ donation = 0  → 0 == 0   → true → "No problem" │ ✅
  │ donation = "" → "" == 0  → true → "No problem" │ ❌
  │                                                  │
  │ → KHÔNG THỂ phân biệt 0 vs "" bằng ==!          │
  │ → == coerce "" → 0 → tưởng giống nhau!          │
  │ → "There's no way here for us to reach the       │
  │    else: Want to donate?"                        │
  └──────────────────────────────────────────────────┘

  Will: "To have an equality — a check of 'are these
  things the same' — which kicks off a COMPLEX WEB
  of coercions depending on what's either side — now
  THAT IS WILD."
```

## §6. === Strict Equality — Cứu Tinh! KHÔNG Coerce!

### 6.1 Bài Viết: "At Least One Bloody Operator"

> Will: _"Fortunately, people, we do have at least ONE BLOODY OPERATOR in JavaScript — the triple equals — that only compares what it actually sees and does NOT try and coerce either side before it does a comparison."_

```
=== STRICT EQUALITY — ZERO COERCION!
═══════════════════════════════════════════════════════════════

  if (donation === 0) {
    console.log("No donation, no problem.");
  } else {
    console.log("Want to donate?");
  }

  VỚI donation = 0:
  ┌──────────────────────────────────────────────────┐
  │ 0 === 0                                          │
  │ → CẢ HAI đều number? ✅                          │
  │ → CẢ HAI đều = 0? ✅                             │
  │ → true! → "No donation, no problem." ✅          │
  └──────────────────────────────────────────────────┘

  VỚI donation = "":
  ┌──────────────────────────────────────────────────┐
  │ "" === 0                                         │
  │ → LEFT = string, RIGHT = number                  │
  │ → KHÁC TYPE! → false NGAY! (NO coercion!)        │
  │ → "Want to donate?" ✅                            │
  └──────────────────────────────────────────────────┘

  SO SÁNH == vs ===:
  ┌────────────────────┬──────────┬──────────┐
  │ Expression         │ == result│ === result│
  ├────────────────────┼──────────┼──────────┤
  │ 0 vs 0             │ true ✅  │ true ✅   │
  │ "" vs 0            │ true 💀  │ false ✅  │
  │ "0" vs 0           │ true 💀  │ false ✅  │
  │ false vs 0         │ true 💀  │ false ✅  │
  │ null vs undefined  │ true ⚠️  │ false ✅  │
  │ null vs 0          │ false ⚠️ │ false ✅  │
  │ NaN vs NaN         │ false ⚠️ │ false ⚠️ │
  └────────────────────┴──────────┴──────────┘

  → === = "compares what it ACTUALLY SEES"
  → === = KHÔNG coerce → predictable!
  → === = standard practice!
```

## §8. Bảng Falsy Values Hoàn Chỉnh & == Coercion Rules

```
== COERCION RULES — BẢNG HOÀN CHỈNH:
═══════════════════════════════════════════════════════════════

  QUY TẮC == (ECMAScript spec):

  ① CÙNG TYPE → so sánh TRỰC TIẾP (giống ===)

  ② null == undefined → true (special case!)
     undefined == null → true
     null == BẤT KỲ (trừ undefined) → false
     undefined == BẤT KỲ (trừ null) → false

  ③ NUMBER vs STRING → toNumber(string)
     42 == "42" → 42 == 42 → true

  ④ BOOLEAN vs BẤT KỲ → toNumber(boolean)
     true == 1 → 1 == 1 → true
     false == 0 → 0 == 0 → true
     true == "1" → 1 == "1" → 1 == 1 → true

  ⑤ OBJECT vs PRIMITIVE → ToPrimitive(object)
     [1] == 1 → "1" == 1 → 1 == 1 → true

  BẢNG == KINH ĐIỂN (KẾT QUẢ):
  ┌────────────┬───────┬───┬────┬───┬────┬──────┬─────┬─────┐
  │ ==         │ false │ 0 │ "" │"0"│ [] │[0]   │null │undef│
  ├────────────┼───────┼───┼────┼───┼────┼──────┼─────┼─────┤
  │ false      │ T     │ T │ T  │ T │ T  │ T    │ F   │ F   │
  │ 0          │ T     │ T │ T  │ T │ T  │ T    │ F   │ F   │
  │ ""         │ T     │ T │ T  │ F │ T  │ F    │ F   │ F   │
  │ "0"        │ T     │ T │ F  │ T │ F  │ T    │ F   │ F   │
  │ []         │ T     │ T │ T  │ F │ F  │ F    │ F   │ F   │
  │ [0]        │ T     │ T │ F  │ T │ F  │ F    │ F   │ F   │
  │ null       │ F     │ F │ F  │ F │ F  │ F    │ T   │ T   │
  │ undefined  │ F     │ F │ F  │ F │ F  │ F    │ T   │ T   │
  └────────────┴───────┴───┴────┴───┴────┴──────┴─────┴─────┘
  T=true, F=false

  → Bảng này = NIGHTMARE to memorize!
  → GIẢI PHÁP: LUÔN dùng === !
```

## §9. Tự Implement: toBoolean, ==, === Từ Đầu

```javascript
// CHỨNG MINH 1: toBoolean — 7 falsy values

function toBoolean(value) {
  // 7 falsy values:
  if (value === false) return false;
  if (value === 0) return false;
  if (value === -0) return false;
  if (value === 0n) return false; // BigInt zero
  if (value === "") return false;
  if (value === null) return false;
  if (value === undefined) return false;
  if (Number.isNaN(value)) return false; // NaN

  // TẤT CẢ CÒN LẠI = truthy!
  return true;
}

// Test:
console.log(toBoolean(0)); // false
console.log(toBoolean("")); // false
console.log(toBoolean(null)); // false
console.log(toBoolean(undefined)); // false
console.log(toBoolean(NaN)); // false
console.log(toBoolean(false)); // false

console.log(toBoolean("0")); // true ← ⚠️ string!
console.log(toBoolean("false")); // true ← ⚠️ string!
console.log(toBoolean([])); // true ← ⚠️ object!
console.log(toBoolean({})); // true ← ⚠️ object!
console.log(toBoolean(1)); // true
console.log(toBoolean(" ")); // true ← ⚠️ space!
```

```javascript
// CHỨNG MINH 3: Simulate == operator (simplified)

function looseEqual(a, b) {
  // Rule 1: Same type → compare directly
  if (typeof a === typeof b) {
    // Special: NaN ≠ NaN
    if (typeof a === "number" && Number.isNaN(a) && Number.isNaN(b))
      return false;
    return a === b;
  }

  // Rule 2: null == undefined (and vice versa)
  if ((a === null && b === undefined) || (a === undefined && b === null)) {
    return true;
  }

  // Rule 3: null/undefined vs anything else → false
  if (a === null || a === undefined || b === null || b === undefined) {
    return false;
  }

  // Rule 4: number vs string → toNumber(string)
  if (typeof a === "number" && typeof b === "string") {
    return looseEqual(a, Number(b));
  }
  if (typeof a === "string" && typeof b === "number") {
    return looseEqual(Number(a), b);
  }

  // Rule 5: boolean vs anything → toNumber(boolean)
  if (typeof a === "boolean") return looseEqual(Number(a), b);
  if (typeof b === "boolean") return looseEqual(a, Number(b));

  // Rule 6: object vs primitive → ToPrimitive(object)
  if (typeof a === "object") return looseEqual(a.valueOf(), b);
  if (typeof b === "object") return looseEqual(a, b.valueOf());

  return false;
}

// Test:
console.log(looseEqual("", 0)); // true! (toNumber("") = 0)
console.log(looseEqual(0, "")); // true!
console.log(looseEqual("0", 0)); // true! (toNumber("0") = 0)
console.log(looseEqual(false, 0)); // true! (toNumber(false) = 0)
console.log(looseEqual(null, undefined)); // true! (special case)
console.log(looseEqual(null, 0)); // false! (null only == undefined)
```

```javascript
function validateForm(formData) {
  const errors = [];

  // Quantity: 0 AND "" đều invalid → dùng if (!quantity) OK!
  if (!formData.quantity) {
    errors.push("Quantity is required and must be > 0");
  }

  // Donation: 0 is valid, "" is "ask user"
  // → PHẢI dùng === !
  if (formData.donation === "") {
    errors.push("Would you like to add a donation?");
  }
  // donation === 0 → NO error! (valid choice)

  // Name: check empty string AND whitespace
  if (!formData.name || !formData.name.trim()) {
    errors.push("Name is required");

  return {
    valid: errors.length === 0,
    errors,
  };
}
```

## §10. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 10.1 Pattern ①: 5 Whys — == vs ===

```
5 WHYS: TẠI SAO == COERCE NHƯNG === KHÔNG?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao "" == 0 trả true?
  └→ Vì == coerce string → number: toNumber("") = 0!
     → 0 == 0 → true!

  WHY ②: Tại sao == coerce?
  └→ Vì JS designers nghĩ: "type không quan trọng,
     VALUE mới quan trọng!" (loose comparison)
     → "42" == 42 → "đúng rồi, cùng value 42!"

  WHY ③: Tại sao cần === nếu đã có ==?
  └→ Vì == rules QUÁ PHỨC TẠP & surprising!
     → false == "0" → true?! (cả hai → 0!)
     → Cần 1 operator PREDICTABLE!
     → === ra đời!

  WHY ④: Tại sao === không phải default ban đầu?
  └→ Vì JS 1.0 (1995) chỉ có ==!
     === được thêm vào sau (ECMAScript 3, 1999)!
     → Backward compatibility → == vẫn tồn tại!

  WHY ⑤: Tại sao null == undefined nhưng null != 0?
  └→ Spec SPECIAL CASE!
     null và undefined = "nothing values" → tương đương!
     NHƯNG null ≠ 0 vì 0 = "something" (zero)!
     → ⭐ Đây là 1 trong ít trường hợp == hữu ích:
       if (val == null) catch cả null VÀ undefined!
```

### 10.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — ECMASCRIPT SPEC:
═══════════════════════════════════════════════════════════════

  7.2.14 IsLooselyEqual(x, y):

  1. If Type(x) = Type(y) → IsStrictlyEqual(x, y)
  2. If x = null AND y = undefined → true
  3. If x = undefined AND y = null → true
  4. If Type(x) = Number AND Type(y) = String
     → IsLooselyEqual(x, ToNumber(y))
  5. If Type(x) = String AND Type(y) = Number
     → IsLooselyEqual(ToNumber(x), y)
  6. If Type(x) = Boolean
     → IsLooselyEqual(ToNumber(x), y)
  7. If Type(y) = Boolean
     → IsLooselyEqual(x, ToNumber(y))
  8. If Type(x) = Object
     → IsLooselyEqual(ToPrimitive(x), y)
  ...
  default → false

  7.2.15 IsStrictlyEqual(x, y):

  1. If Type(x) ≠ Type(y) → false  ← ĐƠN GIẢN!
  2. If x = NaN → false
  3. If x = y → true
  4. If x = +0 AND y = -0 → true
  5. Otherwise → false

  → === = 5 rules ĐƠN GIẢN!
  → == = 10+ rules PHỨC TẠP + recursive calls!
```

### 10.5 Pattern ⑤: Reverse Engineering

```javascript
// TỰ BUILD: Full equality comparison engine

function strictEqual(a, b) {
  // 1. Different types → false
  if (typeof a !== typeof b) return false;
  // 2. NaN ≠ NaN
  if (typeof a === "number" && Number.isNaN(a)) return false;
  // 3. Same value
  return a === b; // native ===
}

function looseEqual(a, b) {
  // 1. Same type → strict compare
  if (typeof a === typeof b) return strictEqual(a, b);

  // 2. null == undefined
  if (a === null && b === undefined) return true;
  if (a === undefined && b === null) return true;
  if (a === null || b === null) return false;
  if (a === undefined || b === undefined) return false;

  // 3. Boolean → toNumber first
  if (typeof a === "boolean") return looseEqual(Number(a), b);
  if (typeof b === "boolean") return looseEqual(a, Number(b));

  // 4. Number vs String → toNumber(string)
  if (typeof a === "number" && typeof b === "string") {
    return looseEqual(a, Number(b));
  }
  if (typeof a === "string" && typeof b === "number") {
    return looseEqual(Number(a), b);
  }

  return false;
}

// Full test suite:
const tests = [
  // [a, b, expected ==, expected ===]
  [0, "", true, false],
  [0, "0", true, false],
  [false, 0, true, false],
  [false, "", true, false],
  [null, undefined, true, false],
  [null, 0, false, false],
  ["42", 42, true, false],
  [0, 0, true, true],
  ["abc", "abc", true, true],
];

tests.forEach(([a, b, expectedLoose, expectedStrict]) => {
  const loose = looseEqual(a, b);
  const strict = strictEqual(a, b);
  const looseOk = loose === expectedLoose ? "✅" : "❌";
  const strictOk = strict === expectedStrict ? "✅" : "❌";
  console.log(
    `${looseOk} ${JSON.stringify(a)} == ${JSON.stringify(b)} → ${loose} | ` +
      `${strictOk} === → ${strict}`,
  );
});
```

### 10.6 Pattern ⑥: Lịch Sử

```
LỊCH SỬ: == vs ===:
═══════════════════════════════════════════════════════════════

  1995: JavaScript 1.0 — CHỈ CÓ == !
  │     → Brendan Eich: "Be forgiving with types"
  │     → == = "abstract comparison" = coerce!
  │     → KHÔNG CÓ === ban đầu!
  │
  ↓
  1999: ECMAScript 3 — === ĐƯỢC THÊM VÀO!
  │     → "Strict equality" operator
  │     → 4 năm sau JS 1.0 mới có ===!
  │     → Nhưng == VẪN tồn tại (backward compat)!
  │
  ↓
  2008: Crockford — "JavaScript: The Good Parts"
  │     → "My advice is to NEVER use == and !="
  │     → "Always use === and !=="
  │     → Bắt đầu === movement!
  │
  ↓
  2011: ESLint rule eqeqeq = "always"
  │     → Warn khi dùng == thay ===
  │     → Industry standard!
  │
  ↓
  2013: Kyle Simpson — phản bác Crockford!
  │     → "== is not evil, it's useful!"
  │     → val == null (catch null + undefined) = OK!
  │     → "Know the rules, then decide!"
  │
  ↓
  2024: Consensus:
        → === = DEFAULT cho hầu hết code!
        → == null = ACCEPTABLE exception!
        → TypeScript = NO == issues at compile time!
        → ⭐ "Always ===, except val == null"
```
