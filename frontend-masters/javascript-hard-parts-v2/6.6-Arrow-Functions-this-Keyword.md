## §2. THE TRAP: this Inside add1() = WINDOW! 😱

> Will: _"What is this going to be assigned inside of add1? I HOPE it's user2, right? Unfortunately, it points to a not very useful, unused object automatically in global called WINDOW."_

```

═══════════════════════════════════════════════════════════════

  add1() is called WITHOUT a dot!
  → add1()     ← no object left of dot!
  → NOT user2.add1()

  JS RULE: "this is assigned to whatever is
  to the LEFT of the dot on which that function
  is being run."

  add1() ← nothing left of dot!
  → Default: this = window (browser) / global (Node)!

  EC — add1() (regular function):
  ┌──────────────────────────────────────────────────┐
  │ LOCAL MEMORY:                                    │
  │ ┌──────────────────────────────────────────┐     │
  │ │ this: window  ← NOT USER2! 😱            │     │
  │ └──────────────────────────────────────────┘     │
  │                                                  │
  │ this.score++                                     │
  │ → window.score++                                │

  │                                                  │
```

## §4. Arrow Functions Save The Day — Lexical this!

> Will: _"We have a different way of declaring a function. Declare it using the arrow function style."_

```
ARROW FUNCTION — THE SOLUTION:
═══════════════════════════════════════════════════════════════

  const userFunctionStore = {
    increment: function() {
      const add1 = () => {   // ← ARROW FUNCTION!
        this.score++;
      };
      add1();
    }
  };

  ARROW FUNCTION RULE:
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │ Arrow functions DON'T get their own `this`!     │
  │                                                  │
  │ Instead, `this` inside an arrow function        │
  │ is determined by WHERE IT WAS DEFINED!          │
  │                                                  │
  │ = LEXICAL SCOPING for `this`!                   │
  │ = Same concept as closures!                     │
  │ = Look UP to the enclosing scope!               │
  │                                                  │
  │ Michael: "It would refer to the context          │
  │ of increment."                                   │
  │                                                  │
  │ Will: "It will actually act a bit like our       │
  │ LEXICAL SCOPING from before. Add1 is now        │
  │ LEXICALLY SCOPED!"                               │
  │                                                  │
  └──────────────────────────────────────────────────┘

  COMPARISON:
  ┌──────────────────┬───────────────────────────────┐
  │ Regular function │ Arrow function                │
  ├──────────────────┼───────────────────────────────┤
  │ function add1()  │ const add1 = () =>            │
  │ this = window 😱 │ this = user2 ✅              │
  │ "left of dot"    │ "where it was DEFINED"       │
  │ Dynamic this     │ Lexical this                  │
  │ THE TRAP!        │ THE FIX!                      │
  └──────────────────┴───────────────────────────────┘
```

## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Arrow Functions & this

```
5 WHYS: TẠI SAO ARROW FUNCTION FIX this?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao this trong add1() = window?
  └→ Vì JS rule: this = left of dot!
     add1() has NO DOT! → default = window!
     "add1 is NOT being called as a method!"

  WHY ②: Tại sao that = this worked?
  └→ Vì CLOSURE! "that" saved in parent scope!
     add1 looks up via scope chain → finds "that"!
     "that" = this = user2! Problem solved!

  WHY ③: Tại sao that = this "gave JS a bad name"?
  └→ Vì nó là HACK! Not intuitive! Ugly!
     Every OOP method needs this boilerplate!
     Other languages don't have this problem!

  WHY ④: Tại sao arrow functions fix it?
  └→ Vì arrow functions have NO OWN this!
     They use this from WHERE THEY WERE DEFINED!
     = Lexical this! Same concept as closure!

  WHY ⑤: Tại sao không dùng arrow cho method chính?
  └→ Vì method chính CẦN dynamic this!
     user1.increment → this = user1 (dynamic!)
     user2.increment → this = user2 (dynamic!)
     Arrow would lock this to outer scope (wrong!)
```

### 9.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — this BINDING RULES:
═══════════════════════════════════════════════════════════════

  4 RULES (priority order!):

  ① new keyword:
  → this = newly created object
  → const obj = new MyFunc()

  ② Explicit binding:
  → this = specified object
  → func.call(obj) / func.apply(obj) / func.bind(obj)

  ③ Implicit binding (method call):
  → this = object left of dot!
  → obj.method() → this = obj

  ④ Default binding (plain call):
  → this = window (sloppy) / undefined (strict)
  → func() → this = window

  ARROW FUNCTION = SPECIAL:
  → Doesn't follow ANY of these rules!
  → Uses this from LEXICAL scope (parent function)!
  → Determined at DEFINE time, not call time!
  → Cannot be overridden by call/apply/bind!
```

### 9.3 Pattern ③: Trade-off Analysis

```
TRADE-OFFS — FIXING this:
═══════════════════════════════════════════════════════════════

  ┌──────────────┬─────────┬──────────┬──────────┐
  │              │ that=   │ .bind()  │ Arrow    │
  │              │ this    │          │ function │
  ├──────────────┼─────────┼──────────┼──────────┤
  │ Readability  │ ❌ Ugly │ ⚠️ Extra │ ✅ Clean │
  │ Performance  │ ✅ Fast │ ⚠️ New fn│ ✅ Fast  │
  │ Flexibility  │ ✅ Any  │ ✅ Any   │ ❌ Fixed │
  │ Modern?      │ ❌ Old  │ ⚠️ OK    │ ✅ ES6!  │
  │ Boilerplate  │ ❌ Extra│ ❌ Extra │ ✅ None! │
  └──────────────┴─────────┴──────────┴──────────┘

  Arrow function wins for INNER functions! 🏆
  But NOT for methods themselves!
```

### 9.5 Pattern ⑤: Reverse Engineering

```javascript
// PROVE: Arrow function this cannot be overridden!

const arrowFn = () => {
  console.log("this:", this);
};

const obj = { name: "test" };

// Try to change this:
arrowFn.call(obj); // this: window (NOT obj!)
arrowFn.apply(obj); // this: window (NOT obj!)
const bound = arrowFn.bind(obj);
bound(); // this: window (STILL NOT obj!)

// Arrow function's this is LOCKED to lexical scope!
// Cannot be overridden by ANY mechanism!
// This is by design — it's the whole point! 🔒
```
