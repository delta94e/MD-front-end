## §1. [[Scope]] — Hidden Property Lưu Backpack

### 1.1 Bài Viết: Double Bracket = Hidden Link

> Will: _"It is stored as a hidden property on this function. As with many of the hidden links in JavaScript, it's [[scope]] — square bracket, square bracket — which is our way that the JavaScript spec refers to its hidden links."_

```
[[Scope]] — HIDDEN PROPERTY:
═══════════════════════════════════════════════════════════════

  JS dùng DOUBLE BRACKETS [[...]] cho hidden properties:
  → [[Scope]]         — closure data (backpack!)
  → [[Prototype]]     — prototype chain link
  → [[Call]]          — function thật sự gọi được
  → [[Construct]]     — new keyword handler
  → [[Environment]]   — ECMAScript spec term cho scope

  ⚠️ [[...]] = INTERNAL SLOT!
  → Developer KHÔNG truy cập trực tiếp được!
  → Chỉ JS engine truy cập!
  → console.dir() hiển thị để DEBUG!

  function outer() {
    let counter = 0;
    function add1() { counter++; }
    return add1;
  }
  const newFunc = outer();

  newFunc có:
  ┌─────────────────────────────────────────────────────┐
  │ newFunc (function object):                          │
  │                                                     │
  │ Visible properties:                                 │
  │   .name = "add1"                                    │
  │   .length = 0                                       │
  │   .prototype = {}                                   │
  │                                                     │
  │ HIDDEN properties:                                  │
  │   [[Scope]] = [                      ← BACKPACK!    │
  │     { counter: 0 },                  ← Closure data │
  │     { /* Script scope */ },                         │
  │     { /* Global scope */ }                          │
  │   ]                                                 │
  │   [[Call]] = internal                               │
  │   [[Prototype]] = Function.prototype                │
  └─────────────────────────────────────────────────────┘

  Will: "This one is [[scope]] — and that word 'scope'
  makes sense here."
```

## §2. "Scope" Là Gì? — Dữ Liệu Nào Available?

> Will: _"Scope is nothing profound besides: at any given line of our code, what data do I have available to me?"_

```
SCOPE = "DỮ LIỆU NÀO TÔI CÓ THỂ DÙNG?"
═══════════════════════════════════════════════════════════════

  function outer() {
    let counter = 0;
    function add1() {
      counter++;               // ← SCOPE TẠI DÒNG NÀY?
    }
    return add1;
  }

  const newFunc = outer();
  newFunc();                    // ← SCOPE TẠI DÒNG NÀY?

  SCOPE = "What data do I have available to me?"

  ① Bên trong add1() (khi chạy TRONG outer):
  ┌──────────────────────────────────────────────────────┐
  │ Available data:                                      │
  │   Local: (trống)                                     │
  │   Outer: counter = 0, add1 = ƒ                      │
  │   Global: outer = ƒ, newFunc, ...                    │
  └──────────────────────────────────────────────────────┘

  ② Bên trong newFunc() (khi chạy Ở GLOBAL, VỚI CLOSURE):
  ┌──────────────────────────────────────────────────────┐
  │ Available data:                                      │
  │   Local: (trống — tạo mới mỗi call)                 │
  │   [[Scope]]: counter = 0 → 1 ← BACKPACK!            │
  │   Global: outer = ƒ, newFunc = ƒ, ...               │
  │                                                      │
  │ → Scope KHÔNG thay đổi dù gọi ở đâu!               │
  │ → Vì [[Scope]] được set KHI DEFINE!                 │
  │ → Đây = LEXICAL SCOPE!                               │
  └──────────────────────────────────────────────────────┘

  Will: "Inside the execution context, the data available
  is here. When I return out the function and run newFunc,
  there is further available data on newFunc's hidden
  [[scope]] stored data."
```

## §3. Variable Environment → Closed Over → Backpack

### 3.1 Bài Viết: Từ "Local Memory" Đến "Backpack" — Chuỗi Thuật Ngữ

Will xây dựng thuật ngữ từng bước:

```
CHUỖI THUẬT NGỮ — TỪ ĐƠN GIẢN ĐẾN CHÍNH THỨC:
═══════════════════════════════════════════════════════════════

  Step 1: LOCAL MEMORY
  → Tên đơn giản nhất cho vùng nhớ bên trong function
  → "Nơi lưu data khi function chạy"

          ↓ tên chính thức hơn:

  Step 2: VARIABLE ENVIRONMENT
  → "The environment of variables around me at that moment"
  → "The data around me at that moment"
  → Will: "Old school word 'variable', but the data
    around me at that moment"

          ↓ khi data đó bị "đóng lại" và kéo ra:

  Step 3: CLOSED OVER Variable Environment
  → Variable Environment bị "đóng nắp" (closed over)
  → Kéo ra theo function khi return!
  → Will: "Like a lid is closed on it and it's pulled out.
    The backpack zip is closed."

          ↓ viết tắt:

  Step 4: C.O.V.E.
  → Closed Over Variable Environment
  → Thuật ngữ "fancy" đầu tiên!

          ↓ muốn "fancier" nữa:

  Step 5: P.L.S.R.D.
  → Persistent Lexically Scoped Referenced Data
  → Thuật ngữ "fancier"!

          ↓ thực tế mọi người dùng:

  Step 6: CLOSURE
  → Thuật ngữ phổ biến nhất
  → NHƯNG bị "overloaded" — dùng cho nhiều thứ!

          ↓ Will thích:

  Step 7: BACKPACK 🎒
  → Thuật ngữ trực quan nhất!
  → "You can see why I prefer the term backpack"
```

## §4. C.O.V.E. — Closed Over Variable Environment

```
C.O.V.E. = CLOSED OVER VARIABLE ENVIRONMENT:
═══════════════════════════════════════════════════════════════

  Phân tích từng từ:

  CLOSED OVER:
  → "Đóng lại" → giống đậy nắp hộp
  → Data bị "nhốt" lại theo function
  → Will: "The lid gets closed on it and we pull it out"

  VARIABLE:
  → Biến (data) — let, const, function declarations
  → "Old school word" cho data

  ENVIRONMENT:
  → Môi trường — nơi biến tồn tại
  → "The stuff around me at that moment"

  GỘP LẠI:
  → C.O.V.E. = "Môi trường biến bị đóng lại"
  → = Vùng nhớ local bị "đóng gói" và đi theo function
  → = BACKPACK 🎒!

  Ví dụ:
  function outer() {
    let counter = 0;        ┐
    let name = "Will";      ├── Variable Environment
    function add1() {       │   CỦA outer
      counter++;            │
    }                       │
    return add1;            │
  }                         ┘

  return add1 → CLOSE OVER variable environment:
  → counter: 0 → ĐI THEO (referenced) ✅
  → name: "Will" → ❌ (not referenced → GC)
  → add1: ƒ → ❌ (chính nó, không cần reference)

  C.O.V.E. của add1 = { counter: 0 }
```

## §5. P.L.S.R.D. — Persistent Lexically Scoped Referenced Data

> Will xây dựng từng chữ một — rất methodical:

```
P.L.S.R.D. = PERSISTENT LEXICALLY SCOPED REFERENCED DATA:
═══════════════════════════════════════════════════════════════

  PERSISTENT:
  → Tồn tại lâu dài, KHÔNG bị xoá
  → "Data that STICKS AROUND between invocations"
  → Khác với local memory (temporary!)

  LEXICALLY:
  → Liên quan đến LEXICAL (vị trí trong code)
  → Will: "Lexical means in order of words on the page"
  → "You SAVED add1 inside the call to outer, therefore
     it has the surrounding data from that SAVING MOMENT"

  SCOPED:
  → Thuộc về SCOPE (vùng dữ liệu available)
  → Data được giữ theo scope chain
  → Referenced bởi [[Scope]] property

  REFERENCED:
  → Data được REFERENCE (tham chiếu) bởi inner function
  → CHỈ giữ data mà inner function THỰC SỰ dùng!
  → → Selective closure! (V8 optimization)

  DATA:
  → Dữ liệu thực tế: numbers, strings, objects, functions

  GỘP LẠI:
  ┌──────────────────────────────────────────────────────┐
  │ P.L.S.R.D. =                                        │
  │   Dữ liệu (Data)                                    │
  │   được tham chiếu (Referenced)                       │
  │   theo quy tắc vị trí code (Lexically Scoped)       │
  │   và tồn tại lâu dài (Persistent)                   │
  │                                                      │
  │ = BACKPACK 🎒!                                       │
  │                                                      │
  │ Will: "Persistent, sticks around. Lexical scoped,    │
  │ or lexically scoped. Referenced by the scope          │
  │ property. Data."                                     │
  └──────────────────────────────────────────────────────┘
```

## §6. Lexical Scoping vs Dynamic Scoping — Full Breakdown

```
LEXICAL vs DYNAMIC — FULL COMPARISON:
═══════════════════════════════════════════════════════════════

  LEXICAL SCOPING (= Static Scoping): JS DÙNG CÁI NÀY! ✅
  ┌──────────────────────────────────────────────────────┐
  │ → Scope xác định bởi NƠI FUNCTION ĐƯỢC LƯU (SAVED) │
  │ → "Based on where the function was saved"            │
  │ → "Position on the page in the code"                 │
  │ → CỐ ĐỊNH — không thay đổi theo runtime!            │
  │                                                      │
  │ Will: "Lexical means in order of words on the page.  │
  │ You saved add1 inside of the call to outer, therefore│
  │ it has the surrounding data from that saving moment.  │
  │ That is LEXICAL SCOPING."                            │
  │                                                      │
  │ Kết quả: closures HOẠT ĐỘNG!                        │
  │ → Function mang theo data từ nơi SINH RA!           │
  └──────────────────────────────────────────────────────┘

  DYNAMIC SCOPING: JS KHÔNG DÙNG!  ❌
  ┌──────────────────────────────────────────────────────┐
  │ → Scope xác định bởi NƠI FUNCTION ĐƯỢC GỌI (CALLED)│
  │ → "Where I call my function determines the only data │
  │    available to me — from where I RAN it"            │
  │ → THAY ĐỔI theo runtime context!                    │
  │                                                      │
  │ Will: "If I ran newFunc, I'd only have the data      │
  │ inside there or in global. That would be DYNAMIC     │
  │ SCOPING."                                            │
  │                                                      │
  │ Kết quả: closures KHÔNG hoạt động!                   │
  │ → Function CHỈ thấy data ở nơi ĐANG CHẠY!          │
  └──────────────────────────────────────────────────────┘

  VÍ DỤ SO SÁNH:
  function outer() {
    let x = "from outer";
    function inner() { return x; }
    return inner;
  }
  let x = "from global";
  const fn = outer();
  fn(); // ???

  LEXICAL (JS thực tế):  fn() → "from outer"
  → inner SAVED trong outer → tìm x ở outer!

  DYNAMIC (nếu JS dùng):  fn() → "from global"
  → inner CALLED ở global → tìm x ở global!

  Will: "Lexical or static scoping — those are the
  two terms for it."
```

## §7. "Closure" — Thuật Ngữ Overloaded

### 7.1 Bài Viết: Tại Sao Will Không "Love" Thuật Ngữ "Closure"

> Will: _"The reason I don't love that term is — we talk about the overall CONCEPT as closure, people talk about storing stuff IN the closure, they talk about the REFERENCE as the closure. It's too OVERLOADED a term."_

```
"CLOSURE" — NHIỀU NGHĨA QUÁ!
═══════════════════════════════════════════════════════════════

  Cùng từ "closure" nhưng dùng cho KHÁC NHAU:

  ① CLOSURE = CONCEPT (khái niệm tổng thể):
  "Let's use closure to enable persistent memory."
  → Ý: dùng CƠ CHẾ closure

  ② CLOSURE = DATA STORE (backpack):
  "Store the counter in the closure."
  → Ý: lưu counter vào BACKPACK

  ③ CLOSURE = FUNCTION + DATA (tổ hợp):
  "That function is a closure."
  → Ý: function có kèm [[Scope]] data

  ④ CLOSURE = REFERENCE/LINK:
  "The closure gives access to outer variables."
  → Ý: LIÊN KẾT đến outer scope

  ⑤ CLOSURE = SCOPE:
  "Check the closure scope in DevTools."
  → Ý: vùng nhớ closure trong debugger

  Will: "You absolutely can and will say: let's make
  sure that this function, let's use CLOSURE to enable
  our function to have a persistent memory. But you
  can see how that term ends up being OVERLOADED."

  → Đó là lý do Will thích "BACKPACK" hơn!
  → BACKPACK = chỉ 1 nghĩa: vùng data đi theo function!
  → Không mơ hồ!
```

## §9. Tự Implement: Inspect [[Scope]] Trong Chrome DevTools

```javascript
// CÁCH XEM [[Scope]] / BACKPACK TRONG THỰC TẾ

function outer() {
  let counter = 0;
  let secret = "hidden";
  function add1() {
    counter++;
    return counter;
  }
  return add1;
}

const newFunc = outer();

// CÁCH 1: console.dir — xem toàn bộ function object
console.dir(newFunc);
// Output:
// ƒ add1()
//   arguments: null
//   caller: null
//   length: 0
//   name: "add1"
//   prototype: {constructor: ƒ}
//   [[FunctionLocation]]: script.js:3
//   [[Prototype]]: ƒ ()
//   [[Scopes]]: Scopes[3]          ← ĐÂY!
//     0: Closure (outer)            ← BACKPACK!
//       counter: 0                  ← Data!
//     1: Script                     ← Module/script scope
//     2: Global {window, ...}       ← Global scope

newFunc();
newFunc();
console.dir(newFunc);
// [[Scopes]]:
//   0: Closure (outer)
//     counter: 2                    ← CẬP NHẬT!
//   1: Script
//   2: Global

// CÁCH 2: Debugger trong Chrome DevTools
// → Đặt breakpoint bên trong add1
// → Khi dừng, xem panel "Scope" bên phải
// → Sẽ thấy: Local | Closure (outer) | Script | Global

// CÁCH 3: Dùng biến trung gian để "inspect"
function createInspectable() {
  let privateCount = 0;

  return {
    increment() {
      privateCount++;
    },
    // Method đặc biệt để "nhìn" vào closure:
    inspect() {
      return { privateCount }; // Copy giá trị ra ngoài
    },
  };
}

const obj = createInspectable();
obj.increment();
obj.increment();
console.log(obj.inspect()); // { privateCount: 2 }
// → Chỉ "inspect" được nếu ta TỰ VIẾT method cho phép!
// → Không có method inspect → HOÀN TOÀN PRIVATE!
```

## §10. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 10.1 Pattern ①: 5 Whys — Tại Sao Nhiều Tên?

```
5 WHYS: TẠI SAO CLOSURE CÓ 7 TÊN?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao có nhiều tên cho cùng 1 thứ?
  └→ Vì khác CONTEXT cần khác cách diễn đạt!
     Spec dùng [[Environment]], teacher dùng "backpack",
     developer dùng "closure"!

  WHY ②: Tại sao spec dùng [[Scope]] / [[Environment]]?
  └→ Vì spec cần CHÍNH XÁC, không mơ hồ!
     Hidden slot = rõ ràng là internal, không public.
     Double bracket = convention cho internal properties.

  WHY ③: Tại sao "closure" bị overloaded?
  └→ Vì thuật ngữ quá phổ biến! Mỗi người dùng
     theo cách riêng: concept, data, function, mechanism.
     Thiếu convention thống nhất.

  WHY ④: Tại sao "lexical" trong P.L.S.R.D.?
  └→ "Lexical" = "liên quan đến text". Code = text!
     Vị trí function TRONG text quyết định scope!
     "Lexical analysis" = bước đầu compiler đọc text!

  WHY ⑤: Tại sao "backpack" effective hơn?
  └→ Vì metaphor CỤ THỂ! "Backpack" = hình ảnh rõ ràng.
     Ai cũng hiểu "mang ba lô" = mang data theo.
     C.O.V.E. hoặc P.L.S.R.D. = quá trừu tượng cho beginner!
     ⭐ Nhưng phỏng vấn → dùng "closure" + giải thích!
```

### 10.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — [[Scope]] TRONG ECMASCRIPT SPEC:
═══════════════════════════════════════════════════════════════

  ECMAScript 2024 Spec — Function Object Internal Slots:

  ① Mỗi function có internal slot [[Environment]]:
     "The Lexical Environment that the function was
      closed over at the time of its creation."

  ② Khi function được CREATE (FunctionCreate/OrdinaryFunctionCreate):
     → step: "Set F.[[Environment]] to env"
     → env = Lexical Environment hiện tại!

  ③ Khi function được CALL (OrdinaryCallEvaluateBody):
     → Tạo new Lexical Environment
     → Set outer reference = F.[[Environment]]
     → ĐÂY là scope chain!

  ④ Khi tìm biến (ResolveBinding):
     → Tìm trong current Lexical Environment
     → Nếu không có → đi theo outer reference
     → Lặp lại cho đến khi tìm thấy hoặc null

  MAPPING thuật ngữ:
  ┌──────────────────────┬──────────────────────┐
  │ Will / Colloquial    │ ECMAScript Spec      │
  ├──────────────────────┼──────────────────────┤
  │ Backpack             │ [[Environment]]      │
  │ Scope chain          │ outer reference chain│
  │ Local memory         │ Environment Record   │
  │ Closure scope        │ Lexical Environment  │

  └──────────────────────┴──────────────────────┘
```

### 10.5 Pattern ⑤: Reverse Engineering

```javascript
// TỰ BUILD: Map TỪNG thuật ngữ vào code

function createTracker(name) {
  // ┌─── VARIABLE ENVIRONMENT (local memory) ───┐
  let visits = 0; // Variable
  const startTime = Date.now(); // Variable
  // └──────────────────────────────────────────────┘

  // Function DEFINED ở đây → [[Environment]] = scope này!
  function track() {
    // ┌─── TEMPORARY MEMORY (mới mỗi call) ───┐
    const now = Date.now();
    // └─────────────────────────────────────────┘

    // ┌─── [[Scope]] / BACKPACK lookup ───┐
    visits++; // P.L.S.R.D.!
    const elapsed = now - startTime; // C.O.V.E.!
    return `${name}: ${visits} visits, ${elapsed}ms`;
    //       ↑         ↑                   ↑
    //    backpack   backpack           backpack
    // └────────────────────────────────────┘
  }

  // CLOSED OVER: visits, startTime, name
  // khi return → đi theo track trong BACKPACK!
  return track;
}

const pageTracker = createTracker("homepage");
// pageTracker.[[Scopes]] =
//   Closure (createTracker):
//     visits: 0, startTime: 170..., name: "homepage"

pageTracker(); // "homepage: 1 visits, 5ms"
pageTracker(); // "homepage: 2 visits, 1003ms"

// Verify:
console.dir(pageTracker);
// [[Scopes]]: Closure(createTracker)
//   name: "homepage"    ← P (persistent) ✅
//   visits: 2           ← L (lexically scoped) ✅
//   startTime: 170...   ← R (referenced) ✅
//   = P.L.S.R.D. ✅     ← D (data) ✅
```
