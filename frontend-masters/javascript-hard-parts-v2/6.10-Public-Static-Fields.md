## §4. static describe() — "Just Add To The Object Bit!"

> Will: _"That fancy word 'static' there — this is ALL only going to add our describe function to UserCreator, the object portion. Not the prototype bit, but to the OBJECT DIRECTLY."_

```
static — DIRECT OBJECT PROPERTY:
═══════════════════════════════════════════════════════════════

  static describe() {
    console.log("Creates users");
  }

  WHERE DOES IT GO?
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │ UserCreator:                                     │
  │ ┌────────────────────────────────────────┐       │
  │ │ OBJECT BIT:                            │       │
  │ │                                        │       │
  │ │ describe: ƒ  ← HERE! On object itself!│       │
  │ │                     NOT on .prototype! │       │
  │ │                                        │       │
  │ │ prototype: {                           │       │
  │ │   increment: ƒ  ← regular methods     │       │
  │ │   login: ƒ      ← regular methods     │       │
  │ │ }                                      │       │
  │ └────────────────────────────────────────┘       │
  │                                                  │
  │ WHY?                                             │
  │ "It's a function we want to have RELATED to      │
  │ the overall class. It will describe what this    │
  │ class actually DOES."                            │
  │                                                  │
  │ Will: "This is some PROPER syntactic sugar.      │
  │ That fancy word 'static' is just going to add    │

  │                                                  │

  ┌──────────────────────────────────────────────────┐
```

## §6. Public Instance Fields — "loggedIn = false" (2022!)

> Will: _"These were added only in 2022. We're now able to add static, private, class, instance FIELDS."_

```
PUBLIC INSTANCE FIELDS — NEW (2022)!:
═══════════════════════════════════════════════════════════════

  class UserCreator {
    loggedIn = false;    // ← PUBLIC INSTANCE FIELD!

    constructor(name, score) {
      this.name = name;
      this.score = score;
    }
    increment() { this.score++; }
    login() { console.log("Logged in!"); }
  }

  WHAT DOES loggedIn = false DO?
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │ Will: "This is going to save onto our function   │
  │ object a new HIDDEN PROPERTY called [[fields]]   │
  │ into which we can store things that we want       │
  │ to have ADDED TO EVERY OBJECT that gets           │
  │ returned out."                                   │
  │                                                  │
  │ WHEN new UserCreator("Ari", 3) runs:             │
  │ → this.name = "Ari"      ← from constructor!   │
  │ → this.score = 3         ← from constructor!   │
  │ → this.loggedIn = false  ← AUTO INSERTED! 🔵   │
  │                                                  │
  │ "Automatically inserted — another line!"         │
  │                                                  │
  └──────────────────────────────────────────────────┘

  THIS IS DIFFERENT FROM static:
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │ static describe() → on UserCreator OBJECT itself │
  │ → UserCreator.describe() ✅                      │
  │ → user1.describe() ❌                            │
  │                                                  │
  │ loggedIn = false → on EVERY instance!            │
  │ → user1.loggedIn ✅                              │
  │ → user2.loggedIn ✅                              │
  │ → UserCreator.loggedIn ❌                        │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

## §8. Tự Implement: static + Instance Fields

```javascript
// CHỨNG MINH 1: static = on function object, not prototype

class UserCreator {
  constructor(name, score) {
    this.name = name;
    this.score = score;
  }
  increment() {
    this.score++;
  }

  static describe() {
    console.log("Creates users");
  }
}

// static is on the object itself:
console.log(typeof UserCreator.describe); // "function" ✅
console.log(typeof UserCreator.prototype.describe); // "undefined"!

// Same as doing:
// UserCreator.describe = function() { ... };

// Instances don't have it:
const user1 = new UserCreator("Ari", 3);
console.log(typeof user1.describe); // "undefined"!
// → static = class-level, not instance-level!
```

```javascript
// CHỨNG MINH 2: Public Instance Fields

class UserCreator {
  loggedIn = false; // instance field!
  attempts = 0; // another instance field!

  constructor(name, score) {
    this.name = name;
    this.score = score;
  }
  increment() {
    this.score++;
  }
}

const user1 = new UserCreator("Ari", 3);
const user2 = new UserCreator("J", 5);

// Both have default fields:
console.log(user1.loggedIn); // false ✅
console.log(user2.loggedIn); // false ✅
console.log(user1.attempts); // 0 ✅

// They're OWN properties (not shared via prototype!):
console.log(user1.hasOwnProperty("loggedIn")); // true!
console.log(user1.hasOwnProperty("name")); // true!

// Each instance gets its own copy:
user1.loggedIn = true;
console.log(user1.loggedIn); // true
console.log(user2.loggedIn); // false (independent!)
```

```javascript
// CHỨNG MINH 3: Instance fields = auto-inserted this.X = value

// With class instance field:
class WithField {
  loggedIn = false;
  constructor(name) {
    this.name = name;
  }
}

// Equivalent WITHOUT instance field:
function WithoutField(name) {
  this.loggedIn = false; // ← manually inserted!
  this.name = name;
}

const a = new WithField("Ari");
const b = new WithoutField("Ari");

// IDENTICAL result:
console.log(JSON.stringify(a)); // {"loggedIn":false,"name":"Ari"}
console.log(JSON.stringify(b)); // {"loggedIn":false,"name":"Ari"}

// Instance field just auto-inserts the assignment!
```

```javascript
// CHỨNG MINH 4: static fields (not just methods!)

class Config {
  static version = "2.0"; // static field! (2022)
  static maxUsers = 1000; // static field!
  static describe() {
    return "Config class";
  } // static method

  constructor(name) {
    this.name = name;
  }
}

// Static fields/methods = on function object:
console.log(Config.version); // "2.0" ✅
console.log(Config.maxUsers); // 1000 ✅
console.log(Config.describe()); // "Config class" ✅

// NOT on instances:
const c = new Config("test");
console.log(c.version); // undefined!
console.log(c.maxUsers); // undefined!
```

```javascript
// CHỨNG MINH 5: Complete class with ALL features

class UserCreator {
  // Public instance fields (2022):
  loggedIn = false;
  loginCount = 0;

  // Constructor (function bit):
  constructor(name, score) {
    this.name = name;
    this.score = score;
  }

  // Regular methods (→ .prototype):
  increment() {
    this.score++;
  }
  login() {
    this.loggedIn = true;
    this.loginCount++;
    console.log(`${this.name} logged in (#${this.loginCount})`);
  }

  // Static method (→ object directly):
  static describe() {
    console.log("UserCreator: makes user objects");
  }

  // Static field (→ object directly):
  static version = "1.0";
}

// Class-level:
UserCreator.describe(); // "UserCreator: makes user objects"
console.log(UserCreator.version); // "1.0"

// Instance-level:
const user1 = new UserCreator("Ari", 3);
console.log(user1.loggedIn); // false (from instance field)
console.log(user1.loginCount); // 0 (from instance field)
console.log(user1.name); // "Ari" (from constructor)

user1.login(); // "Ari logged in (#1)"
console.log(user1.loggedIn); // true ✅
console.log(user1.loginCount); // 1 ✅

// WHERE everything lives:
console.log(user1.hasOwnProperty("loggedIn")); // true (own!)
console.log(user1.hasOwnProperty("name")); // true (own!)
console.log(user1.hasOwnProperty("increment")); // false (prototype!)
console.log(UserCreator.prototype.increment); // ƒ (on prototype!)
console.log(UserCreator.describe); // ƒ (on object!)
```

## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Static & Instance Fields

```
5 WHYS: TẠI SAO CẦN FIELDS?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao cần static fields?
  └→ Vì cần properties THUỘC VỀ class, không thuộc instances!
     Version, config, factory methods!
     UserCreator.version, not user1.version!

  WHY ②: Tại sao cần instance fields?
  └→ Vì cần DEFAULT values cho every instance!
     loggedIn = false cho MỌI user khi tạo!
     Trước đây phải viết trong constructor!

  WHY ③: Tại sao instance fields không trên .prototype?
  └→ Vì mỗi instance cần RIÊNG copy!
     user1.loggedIn = true ≠ user2.loggedIn!
     Shared prototype = sai logic cho data!

  WHY ④: Tại sao thêm hidden [[Fields]]?
  └→ Vì cần store field definitions TRƯỚC constructor!
     JS auto-insert chúng khi new runs!
     "Actually changes stuff under the hood!"

  WHY ⑤: Tại sao added 2022 mà không trước?
  └→ Vì proposal process! TC39 takes years!
     Stage 0 → 1 → 2 → 3 → 4!
     Class fields proposal started 2015, finished 2022!
```

### 9.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — WHERE THINGS GO:
═══════════════════════════════════════════════════════════════

  class Foo {
    instanceField = value;        // → EACH instance (own!)
    constructor() { ... }         // → function bit
    regularMethod() { ... }       // → Foo.prototype
    static staticMethod() { ... } // → Foo object directly
    static staticField = value;   // → Foo object directly
  }

  RULE:
  ┌──────────────────────────────────────────────────┐
  │ No keyword         → .prototype (shared!)        │
  │ static keyword     → function object (class-level)│
  │ Field declaration  → each instance (own copy!)   │
  │ constructor        → function bit (runs w/ new!) │
  └──────────────────────────────────────────────────┘
```

```
TRADE-OFFS — EXPLICIT vs SUGAR:
═══════════════════════════════════════════════════════════════

  ┌──────────────────┬───────────────┬───────────────────┐
```

### 9.5 Pattern ⑤: Reverse Engineering

```javascript
// PROVE: Instance fields are own properties, not prototype

class User {
  loggedIn = false;
  constructor(name) {
    this.name = name;
  }
}

const u = new User("Ari");

// Instance field = own property:
console.log(Object.keys(u)); // ["loggedIn", "name"]

// NOT on prototype:
console.log(User.prototype.loggedIn); // undefined!

// Compare with methods:
console.log(Object.keys(User.prototype)); // [] (non-enumerable!)
console.log(Object.getOwnPropertyNames(User.prototype));
// ["constructor"] — methods are non-enumerable!

// Prove fields are inserted BEFORE constructor body:
class Order {
  status = "pending";
  constructor() {
    console.log(this.status); // "pending" ← already set!
    this.status = "created";
  }
}
new Order(); // logs "pending" → field set BEFORE constructor!
```
