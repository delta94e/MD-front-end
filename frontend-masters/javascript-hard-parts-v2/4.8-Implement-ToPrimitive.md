## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Symbol.toPrimitive

```

═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao không dùng string "toPrimitive"?
  └→ Vì BACKWARD COMPATIBILITY!
     Nếu dùng string → objects CŨ có thể ĐÃ CÓ
     property "toPrimitive" → CLASH! BREAK code!

  WHY ②: Tại sao Symbol guaranteed unique?
  └→ Vì Symbol = primitive type, mỗi Symbol()
     tạo giá trị MỚI, KHÁC BIỆT!
     → KHÔNG THỂ clash với BẤT KỲ string nào!
     → KHÔNG THỂ clash với BẤT KỲ Symbol khác!

  WHY ③: Tại sao well-known Symbols trên Symbol object?
  └→ Vì cần 1 CENTRAL REFERENCE cho all developers!
     Symbol.toPrimitive = CÙNG 1 Symbol cho MỌI NGƯỜI!
     → Nếu ai cũng tạo Symbol("toPrimitive") riêng
       → KHÔNG AI sẽ match! Vô dụng!

  WHY ④: Tại sao hidden properties cần thiết?
  └→ Vì SEPARATION OF CONCERNS!
     → User data (name, id) ≠ meta behavior (coercion)!
     → Object.keys() chỉ trả user data!
     → JSON.stringify() chỉ serialize user data!
     → Symbol properties = "meta layer"!

  WHY ⑤: Tại sao metaprogramming quan trọng?
```

### 9.5 Pattern ⑤: Reverse Engineering

```javascript
// TỰ BUILD: Full metaprogramming system

// Mini-framework: Smart Objects with auto-coercion
function createSmartObject(data, primaryKey) {
  const obj = { ...data };

  // Add toPrimitive based on primaryKey
  obj[Symbol.toPrimitive] = function (hint) {
    if (hint === "number") {
      // Return numeric primary key
      return Number(this[primaryKey]);
    }
    if (hint === "string") {
      // Return descriptive string
      return Object.entries(this)
        .filter(([k]) => typeof k === "string") // skip Symbol keys
        .map(([k, v]) => `${k}:${v}`)
        .join(", ");
    }
    // Default: return primary key value
    return this[primaryKey];
  };

  return obj;
}

// Create smart users:
const stored = createSmartObject({ name: "Will", id: 105 }, "id");
const submitted = createSmartObject({ name: "Will", id: 105 }, "id");
const hacker = createSmartObject({ name: "Evil", id: 666 }, "id");

// Math comparison:
console.log(+stored === +submitted); // true! (105 === 105)
console.log(+stored === +hacker); // false! (105 !== 666)

// String context:
console.log(`${stored}`); // "name:Will, id:105"
console.log(`${hacker}`); // "name:Evil, id:666"

// Arithmetic:
console.log(stored - 100); // 5 (105 - 100)
console.log(stored * 2); // 210 (105 * 2)
console.log(stored > 100); // true (105 > 100)
```
