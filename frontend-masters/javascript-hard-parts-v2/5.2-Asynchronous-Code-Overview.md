## §1. Vấn Đề: Single-Threaded = BLOCKING!

### 1.1 Bài Viết: "Slow Function Blocks EVERYTHING"

> Will: _"What if we had a task accessing TikTok's server to get new video links? That could take a long time — maybe 300 milliseconds. But we also want to run code in the meantime. Challenge: we want to wait for the data, but that would mean no code could run!"_

```
VẤN ĐỀ — BLOCKING:
═══════════════════════════════════════════════════════════════

  // Pretend code:
  function getVideos() {
    // Request to TikTok server...
    // Takes 350ms to come back!
  }

  getVideos();                    // ← BLOCKS for 350ms!
  displayVideos(videos);          // ← Phải chờ!
  console.log("I wanna run!");   // ← CŨNG phải chờ!

  TIMELINE (BLOCKING):
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │ TIME →                                           │
  │                                                  │
  │ ├─── getVideos() called                          │
  │ │                                                │
  │ │    🔴 WAITING... (350ms)                       │
  │ │    🔴 BLOCKED! Nothing can run!                │
  │ │    🔴 User can't click anything!               │
  │ │    🔴 UI frozen!                               │
  │ │                                                │
  │ ├─── displayVideos() ← FINALLY!                  │
  │ │                                                │
  │ ├─── console.log("I wanna run!") ← AT LAST!     │
  │ │                                                │
  │ └─── Done (but user waited 350ms doing NOTHING!) │
  │                                                  │
  └──────────────────────────────────────────────────┘

  350ms = "remarkably short" cho mạng...
  NHƯNG = CỰC KỲ DÀI cho JavaScript!
  → Toàn bộ app có thể chạy xong trong < 10ms!
  → Chờ 350ms = UI ĐÓNG BĂNG = terrible UX! 💀
```

## §6. "Facade Functions" — Labels Cho Features Bên Ngoài!

### 6.1 Bài Viết: Will's Term

> Will: _"I call them FACADE FUNCTIONS — because they're a facade. They LOOK like they do JavaScript work, but they're actually a facade for stuff OUTSIDE of JavaScript entirely, that we're accessing from within JavaScript."_

```
FACADE FUNCTIONS — CONCEPT:
═══════════════════════════════════════════════════════════════

  FACADE = Mặt tiền, bề ngoài!

  ┌──────────────────────────────────────────────────┐
  │ NHÌN BỀ NGOÀI:                                  │
  │                                                  │
  │   setTimeout(printHello, 1000);                  │
  │                                                  │
  │   → Trông như gọi 1 JS function!                │
  │   → Trông như JS tự đếm 1000ms!                 │
  │   → Trông như JS tự gọi printHello!              │
  │                                                  │
  │ THỰC TẾ BÊN TRONG:                              │
  │                                                  │
  │   setTimeout → NÓI VỚI BROWSER:                 │
  │   "Hey browser, bật timer 1000ms!                │
  │    Khi xong, gọi printHello giùm!"              │
  │                                                  │
  │   → JS KHÔNG đếm thời gian!                     │
  │   → BROWSER đếm bằng timer thread riêng!        │
  │   → JS tiếp tục chạy dòng tiếp theo!            │
  │   → = NON-BLOCKING!                              │
  └──────────────────────────────────────────────────┘

  DIAGRAM:
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  JavaScript:          Browser:                   │
  │                                                  │
  │  setTimeout(fn,1000)  ─────→  Timer Thread       │
  │       │                        Start counting!   │
  │       │ (returns immediately!)                   │
  │       ▼                                          │
  │  console.log("Me first!");                       │
  │       │                        Timer: 999...     │
  │       ▼                        Timer: 1000! ✅   │
  │  (idle, waiting)      ←─────  "fn is ready!"    │
  │       │                                          │
  │       ▼                                          │
  │  fn() runs!                                      │
  │                                                  │
  └──────────────────────────────────────────────────┘

  → setTimeout = FACADE cho Browser Timer!
```

## §8. Tự Implement: Facade Function Simulator

```javascript
// CHỨNG MINH 1: setTimeout puzzle — verify behavior

function printHello() {
  console.log("Hello");
}

// Puzzle 1: 1000ms delay
setTimeout(printHello, 1000);
console.log("Me first!");
// Output:
// 1. "Me first!"     ← FIRST! (synchronous!)
// 2. "Hello"          ← AFTER 1000ms! (async!)

// Puzzle 2: 0ms delay
setTimeout(() => console.log("Zero delay"), 0);
console.log("Still first!");
// Output:
// 1. "Still first!"  ← FIRST! (synchronous!)
// 2. "Zero delay"    ← AFTER! (even with 0ms!)

// WHY? → setTimeout = facade function!
// → Goes to browser → callback queue → event loop
// → Must wait for ALL synchronous code to finish!
```

```javascript
// CHỨNG MINH 2: Facade function concept — simulate

class WebBrowser {
  constructor() {
    this.timers = [];
    this.callbackQueue = [];
    this.console = {
      _output: [],
      log: (message) => {
        this.console._output.push(message);
        console.log(`  [BROWSER CONSOLE] ${message}`);
      },
    };
  }

  // Facade for browser timer:
  setTimeout(callback, delay) {
    console.log(`  [BROWSER] Timer started! ${delay}ms`);
    console.log(`  [BROWSER] JS thread continues IMMEDIATELY!`);

    // Browser counts time (separate thread!):
    const timer = {
      callback,
      delay,
      startTime: Date.now(),
      done: false,
    };
    this.timers.push(timer);

    // Simulate browser timer in background:
    const realTimeout = globalThis.setTimeout(() => {
      timer.done = true;
      this.callbackQueue.push(callback);
      console.log(`  [BROWSER] Timer DONE! Callback → queue!`);
    }, delay);

    // Return immediately — NO BLOCKING!
    return timer;
  }
}

const browser = new WebBrowser();

console.log("=== setTimeout IS A FACADE ===");
console.log("1. JS calls setTimeout...");
browser.setTimeout(() => console.log("Hello!"), 1000);
console.log("2. JS continues IMMEDIATELY (not blocked!)");
console.log("3. 'Me first!' runs while browser counts...");
```

## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Facade Functions

```
5 WHYS: TẠI SAO CẦN FACADE FUNCTIONS?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao JS cần browser features?
  └→ Vì JS chỉ có thread + memory + call stack!
     Không có timer, network, DOM, storage!
     = Cực kỳ hạn chế nếu đứng một mình!

  WHY ②: Tại sao browser features nằm ngoài JS?
  └→ Vì chúng CẦN THỜI GIAN! Network = 300ms!
     Nếu trong JS → BLOCK thread!
     Ngoài JS → browser xử lý parallel!

  WHY ③: Tại sao gọi là "facade"?
  └→ Vì nhìn từ JS → giống function bình thường!
     setTimeout() = trông như JS function!
     Nhưng BÊN TRONG → nói với browser!
     = Facade = mặt tiền giả!

  WHY ④: Tại sao tên không trùng feature name?
  └→ Vì lịch sử! JS ra đời 1995, naming conventions
     thay đổi theo thời gian!
     DOM → "document" (old naming)
     Network → "fetch" (new, better naming)

  WHY ⑤: Tại sao model này successful?
  └→ Vì BEST OF BOTH WORLDS!
     JS = simple, single-threaded, predictable!
     Browser = multi-threaded, parallel, powerful!
     Combine = non-blocking, efficient!
```

### 9.3 Pattern ③: Trade-off Analysis

```
TRADE-OFFS: CÁC APPROACHES CHO ASYNC:
═══════════════════════════════════════════════════════════════

  ┌──────────────────┬──────────────────┬──────────────────┐
  │ Approach         │ Ưu điểm          │ Nhược điểm       │
  ├──────────────────┼──────────────────┼──────────────────┤
  │ Blocking         │ Simple!          │ UI frozen! 💀    │
  │ (busy-wait)      │ Predictable      │ CPU wasted       │
  ├──────────────────┼──────────────────┼──────────────────┤
  │ Multi-threaded   │ True parallel!   │ Race conditions  │
  │ (Java, C++)      │ Use all cores    │ Complex locks    │
  ├──────────────────┼──────────────────┼──────────────────┤
```

### 9.5 Pattern ⑤: Reverse Engineering

```javascript
// TỰ BUILD: Mini browser runtime model

class MiniBrowser {
  constructor() {
    // JS Engine components:
    this.callStack = ["global()"];
    this.globalMemory = {};

    // Browser components (NEW!):
    this.timerThread = [];
    this.networkThread = [];
    this.callbackQueue = [];
    this.microtaskQueue = [];

    // Console (browser feature!):
    this.consoleOutput = [];
  }

  // === FACADE FUNCTIONS ===

  // setTimeout → browser timer!
  setTimeout(callback, delay) {
    console.log(`[JS→BROWSER] setTimeout: start timer ${delay}ms`);
    console.log(`[JS] Returns immediately — NOT blocking!`);

    // Timer runs in BROWSER (separate thread!):
    const timerId = globalThis.setTimeout(() => {
      console.log(`[BROWSER] Timer done! → callback queue`);
      this.callbackQueue.push(callback);
      this._checkEventLoop();
    }, delay);

    return timerId;
  }

  // console.log → browser console!
  console_log(message) {
    console.log(`[JS→BROWSER] console.log: "${message}"`);
    this.consoleOutput.push(message);
  }

  // === EVENT LOOP (simplified) ===
  _checkEventLoop() {
    // Only run callbacks when call stack is empty (only global)!
    if (this.callStack.length <= 1) {
      if (this.callbackQueue.length > 0) {
        const cb = this.callbackQueue.shift();
        console.log(`[EVENT LOOP] Call stack empty! Running callback...`);
        this.callStack.push("callback()");
        cb();
        this.callStack.pop();
      }
    }
  }
}

// Simulate the puzzle:
```
