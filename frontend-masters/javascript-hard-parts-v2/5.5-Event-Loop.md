## §1. Event Loop — The Missing Piece!

### 1.1 Bài Viết: "How Does It Know?"

> Will: _"How does JavaScript KNOW that the call stack is empty? How does it know that global's finished running? How does it know that there's stuff waiting in the queue? It has a KEY FEATURE that sits between our queue and our call stack — checking, checking, checking, checking on repeat. It's LOOPING."_

```
EVENT LOOP — VỊ TRÍ:
═══════════════════════════════════════════════════════════════

  ┌─────────────────────────────────────────────────────────┐
  │                                                         │
  │  CALL STACK          EVENT LOOP         CALLBACK QUEUE  │
  │  ┌──────────┐        🔄                ┌─────────────┐  │
  │  │          │   ←── checking ──→      │ [printHello] │  │
  │  │  global  │        checking          │             │  │
  │  └──────────┘        checking          └─────────────┘  │
  │                      checking                           │
  │                      checking...                        │
  │                                                         │
  │  Event loop = BRIDGE between queue and stack!           │
  │  Sits BETWEEN the two!                                  │
  │  Checks BEFORE every single bit of code!               │
  │                                                         │
  └─────────────────────────────────────────────────────────┘

  EVENT LOOP LÀM GÌ?
  → Chỉ MỘT việc duy nhất!
  → CHECK, CHECK, CHECK on repeat!
  → "Is there something to dequeue?"
  → = JavaScript's TRIAGE system!
```

## §2. 3 Câu Hỏi — Checking on Repeat!

```
EVENT LOOP — 3 CÂU HỎI (mỗi tick!):
═══════════════════════════════════════════════════════════════

  🔄 LOOP {

    ❓ Câu hỏi 1: "Is my call stack EMPTY?"
    │  → Có function nào đang chạy không?
    │  → Có execution context nào trên stack không?
    │
    ❓ Câu hỏi 2: "Have I finished running ALL global code?"
    │  → Còn dòng nào trong global chưa chạy không?
    │  → TẤT CẢ code phải xong!
    │
    ❓ Câu hỏi 3: "Is there something in my callback queue?"
    │  → Có callback nào đang chờ không?
    │  → Timer done? Network done? Click event?

    IF (tất cả 3 = YES) {
      → DEQUEUE callback!
      → ADD to call stack!
      → EXECUTE!
    } ELSE {
      → Tiếp tục checking...
      → "Uh uh, sorry callback queue,
         you've just gotta sit there WAITING!"
    }

  } // repeat forever!

  Will: "Before EVERY SINGLE bit of code is run —
  checking, checking, checking, checking on repeat.
  It's LOOPING, checking all the time."
```

## §4. 2 Strict Rules — Dequeue Conditions

```
2 STRICT RULES — EVENT LOOP:
═══════════════════════════════════════════════════════════════

  CÓ THỂ DEQUEUE callback CHỈ KHI:

  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  RULE 1: Call stack FULLY EMPTY!                 │
  │  → Không có function nào đang chạy!             │
  │  → Chỉ còn global (hoặc global cũng xong!)     │
  │                                                  │
  │  RULE 2: ALL global code FINISHED running!       │
  │  → Tất cả dòng code trong global đã execute!    │
  │  → Không còn dòng nào chưa chạy!                │
  │                                                  │
  │  CẢ HAI rules phải TRUE → mới dequeue!           │
  │                                                  │
  └──────────────────────────────────────────────────┘

  Will: "Not until TWO STRICT RULES — my call stack
  is fully empty AND all my global code has finished
  running. At which point I can GRAB my printHello
  function and put it into the call stack."

  VÍ DỤ: RULE 1 TRUE nhưng RULE 2 FALSE:
  ┌──────────────────────────────────────────────────┐
  │ setTimeout(fn, 0);  // → callback queued!        │
  │ // ← call stack = chỉ global (empty-ish!)       │
  │ // ← nhưng vẫn còn code phía dưới!              │
  │ console.log("1");   // ← chưa chạy!             │
  │ console.log("2");   // ← chưa chạy!             │
  │ console.log("3");   // ← chưa chạy!             │
  │                                                  │
  │ → Call stack trống nhưng global code CHƯA XONG! │
  │ → KHÔNG dequeue!                                 │
  │ → Phải chạy hết 3 console.log trước!            │
```

## §7. Tự Implement: Event Loop Từ Scratch

```javascript
// CHỨNG MINH 1: Event loop = just checking!

class EventLoop {
  constructor() {
    this.callStack = [];
    this.callbackQueue = [];
    this.globalCodeDone = false;
    this.checkCount = 0;
  }

  // THE EVENT LOOP — pure checking!
  check() {
    this.checkCount++;
    const stackEmpty = this.callStack.length === 0;
    const queueHasItem = this.callbackQueue.length > 0;

    console.log(`\n🔄 Event Loop Check #${this.checkCount}:`);
    console.log(
      `  ❓ Call stack empty?     → ${stackEmpty ? "✅ YES" : "❌ NO"}`,
    );
    console.log(
      `  ❓ Global code done?     → ${this.globalCodeDone ? "✅ YES" : "❌ NO"}`,
    );
    console.log(
      `  ❓ Queue has callbacks?  → ${queueHasItem ? "✅ YES" : "❌ NO"}`,
    );

    if (stackEmpty && this.globalCodeDone && queueHasItem) {
      const cb = this.callbackQueue.shift();
      console.log(`  → ✅ DEQUEUE: ${cb.name}() → call stack!`);
      this.callStack.push(cb.name);
      cb.fn();
      this.callStack.pop();
      console.log(`  → POP: ${cb.name}() done!`);

      // Check again (might be more in queue!):
      this.check();
    } else if (!stackEmpty || !this.globalCodeDone) {
      console.log(`  → ❌ Cannot dequeue yet! Waiting...`);
    } else {
      console.log(`  → Queue empty! Nothing to do.`);
    }
  }

  // Simulate facade function:
  setTimeout(callback, delay) {
    console.log(`[setTimeout] ${callback.name}(${delay}ms) → browser timer!`);
    if (delay === 0) {
      // Immediately complete → queue:
      this.callbackQueue.push(callback);
      console.log(`  Timer done instantly → ${callback.name} → QUEUE!`);
    }
    // Non-zero delays would use real timers
  }
}

// SIMULATE:
const loop = new EventLoop();

// Register callback:
function printHello() {
  console.log('  >>> "Hello"');
}
loop.setTimeout(printHello, 0);

// Event loop checks — but global code not done!
loop.check(); // ❌ global code not done!

// blockFor1Sec:
console.log("\n[JS] blockFor1Sec() running...(1000ms)...");
loop.callStack.push("blockFor1Sec()");
loop.check(); // ❌ stack not empty!
loop.callStack.pop();

// console.log("Me first!"):
console.log('\n[JS] console.log("Me first!")');
console.log('  >>> "Me first!"');

// Global code done!
loop.globalCodeDone = true;
loop.check(); // ✅ DEQUEUE!
```

## §8. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 8.1 Pattern ①: 5 Whys — Event Loop

```
5 WHYS: TẠI SAO CẦN EVENT LOOP?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao cần "checking" mechanism?
  └→ Vì callbacks KHÔNG THỂ tự nhảy vào call stack!
     Cần "ai đó" kiểm tra và quyết định!
     Event loop = traffic controller!

  WHY ②: Tại sao check liên tục (loop)?
  └→ Vì browser events bất kỳ lúc nào!
     Timer xong, network response, user click...
     PHẢI check thường xuyên để responsive!

  WHY ③: Tại sao check TRƯỚC mỗi task?
  └→ Vì callback nào đến trước = chạy trước!
     FIFO order phải maintained!
     Check thường xuyên = fair scheduling!

  WHY ④: Tại sao 2 rules (stack + global)?
  └→ Vì chỉ "stack empty" thì CHƯA ĐỦ!
     Có thể stack empty nhưng vẫn còn global code!
     CẢ HAI phải true = truly safe to dequeue!

  WHY ⑤: Tại sao gọi là "event" loop?
  └→ Vì browser tạo EVENTS (timer done, click, etc.)!
     Loop processes events from queue!
     = Event-driven architecture!
```

### 8.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — EVENT LOOP SPEC DETAIL:
═══════════════════════════════════════════════════════════════

  HTML Living Standard — 8.1.7.3 Processing Model:

  Step-by-step:
  ┌──────────────────────────────────────────────────┐
  │ 1. Pick oldest RUNNABLE task from task queue     │
  │    (or null if empty)                            │
  │                                                  │
  │ 2. If task found:                                │
  │    → Set "currently running task" to that task   │
  │    → Run the task's steps (callback executes!)   │
  │    → Set "currently running task" to null        │
  │                                                  │
  │ 3. Perform MICROTASK CHECKPOINT                  │
  │    → Run ALL microtasks! (coming next lesson!)   │
  │                                                  │
  │ 4. Update rendering (if time for next frame):    │
  │    → Run requestAnimationFrame callbacks         │
  │    → Layout, paint, composite!                   │
  │                                                  │
  │ 5. GOTO step 1 (loop!)                           │
  └──────────────────────────────────────────────────┘

  KEY: step 3 — MICROTASK CHECKPOINT
  → This is WHY microtask queue has HIGHER priority!
  → After EVERY task → drain ALL microtasks!
  → SẼ HỌC chi tiết ở bài tiếp theo!
```

```
  ┌──────────────────┬──────────────────┬──────────────────┐

  THỰC TẾ:
  → Browser optimize: KHÔNG busy-loop!
  → Dùng OS-level event notification!
  → (epoll, kqueue, IOCP)

  → = Efficient + responsive!

  → "Checking on repeat" = conceptual model!
  → Thực tế = OS notifies browser khi timer/IO ready!
```

```javascript
// TỰ BUILD: Complete event loop with visualization

class VisualEventLoop {
  constructor() {
    this.callStack = [];
    this.callbackQueue = [];
    this.microtaskQueue = []; // preview!
    this.globalDone = false;
    this.ticks = 0;
  }

  // Core event loop logic:
  tick() {
    this.ticks++;
    const canDequeue =
      this.callStack.length === 0 &&
      this.globalDone &&
      this.callbackQueue.length > 0;

    // Visualize:
    this._render();

    if (canDequeue) {
      const cb = this.callbackQueue.shift();
      console.log(`  ✅ DEQUEUE: ${cb.name}`);
      return cb;
    }
    return null;
  }

  _render() {
    const stack =
      this.callStack.length > 0 ? this.callStack.join(" → ") : "(empty)";
    const queue =
      this.callbackQueue.length > 0
        ? this.callbackQueue.map((c) => c.name).join(" → ")
        : "(empty)";

    console.log(`\n═══ Tick #${this.ticks} ═══`);
    console.log(`  Stack: [${stack}]`);
    console.log(`  Queue: [${queue}]`);
    console.log(`  Global done: ${this.globalDone}`);
  }
}
```
