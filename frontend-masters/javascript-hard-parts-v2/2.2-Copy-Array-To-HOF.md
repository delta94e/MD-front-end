## §1. Walkthrough #1: copyArrayAndMultiplyBy2 — Full Trace

### 1.1 Code Gốc

```javascript
function copyArrayAndMultiplyBy2(array) {
  const output = [];
  for (let i = 0; i < array.length; i++) {
    output.push(array[i] * 2);
  }
  return output;
}

const myArray = [1, 2, 3];
const result = copyArrayAndMultiplyBy2(myArray);
```

### 1.2 Bài Viết: Đọc Code Như JavaScript Engine

Will Sentance nhắc lại: khi JavaScript bật lên, 3 thứ được tạo ngay — **Global Execution Context** (memory + thread), và **Call Stack** (push Global). Mọi khai báo chỉ **lưu vào memory**, không chạy code bên trong function.

Khi gặp `copyArrayAndMultiplyBy2(myArray)`:

- Dấu `()` = **GỌI** function = tạo **Execution Context mới**
- Push `copyArrayAndMultiplyBy2` lên **Call Stack**
- Thread dệt VÀO function, chạy từng dòng bên trong

## §5. Không Thể Truyền Operator — Nhưng Có Thể Truyền Function!

### 5.1 Bài Viết: Tại Sao Không Thể Truyền Operator?

Will hỏi: _"Can I pass in `+ 3` as an argument?"_ → **"No, you can't pass little bits of functionality."**

Trong JavaScript, operators (`+`, `-`, `*`, `/`) KHÔNG phải là values — bạn không thể lưu chúng vào biến, truyền vào function, hay return. Chúng là **syntax**, không phải **data**.

Nhưng **FUNCTIONS** là values! Functions có thể lưu, truyền, return. Vậy nếu ta **bọc** operator bên trong function, ta có thể truyền function đó vào!

```
TẠI SAO KHÔNG THỂ TRUYỀN OPERATOR:
═══════════════════════════════════════════════════════════════

  ❌ KHÔNG THỂ:
  copyArray(myArray, * 2)         // SyntaxError!
  copyArray(myArray, + 3)         // SyntaxError!
  copyArray(myArray, / 2)         // SyntaxError!
  → Operators là SYNTAX, không phải VALUES!

  ⚠️ HACK XẤU (đừng làm!):
  copyArray(myArray, "* 2")       // Truyền STRING
  // Bên trong: eval("element " + "* 2")
  → Will: "very high risk" — eval = security nightmare!

  ✅ GIẢI PHÁP: BỌC operator trong FUNCTION!
  ┌──────────────────────────────────────────────────────┐
  │                                                      │
  │  Operator * 2    →  function(x) { return x * 2; }   │
  │  Operator / 2    →  function(x) { return x / 2; }   │
  │  Operator + 3    →  function(x) { return x + 3; }   │
  │                                                      │
  │  → Function = "hộp chứa functionality"               │
  │  → Function = VALUE → có thể truyền làm argument!   │
  └──────────────────────────────────────────────────────┘

  Tenzin trả lời Will: "Wrap it in a FUNCTION."
  → Chính xác! Function = cách duy nhất bọc functionality!
```

## §6. Side Effects & Pure Functions — Tại Sao Tạo Mảng Mới?

### 6.1 Bài Viết: "Consequences Outside" — Side Effects

Will giải thích rất kỹ: tại sao trong function ta tạo mảng `output` **MỚI** thay vì sửa trực tiếp `array`?

> _"We ideally want our functions to not have what we call side effects. That is to say, things happening inside of them that are going to have consequences outside of them that are not defined by the return value."_

**Pure Function** = function mà:

1. Kết quả **CHỈ** phụ thuộc vào input (không đọc global state)
2. **KHÔNG** thay đổi gì bên ngoài (không sửa global data)
3. **Mọi hậu quả** đều thông qua **return value**

```
SIDE EFFECTS vs PURE FUNCTIONS:
═══════════════════════════════════════════════════════════════

  ❌ IMPURE — CÓ SIDE EFFECTS:
  ┌──────────────────────────────────────────────────────┐
  │ function dangerousMultiply(array) {                  │
  │   for (let i = 0; i < array.length; i++) {           │
  │     array[i] = array[i] * 2;   ← SỬA array GỐC!    │
  │   }                                                  │
  │   return array;                                      │
  │ }                                                    │
  │                                                      │
  │ const myArray = [1, 2, 3];                           │
  │ dangerousMultiply(myArray);                          │
  │ console.log(myArray); // [2, 4, 6] 😱               │
  │ → myArray BỊ THAY ĐỔI! Global data bị ảnh hưởng!   │
  │ → Side effect: hậu quả NGOÀI return value!          │
  └──────────────────────────────────────────────────────┘

  ✅ PURE — KHÔNG CÓ SIDE EFFECTS:
  ┌──────────────────────────────────────────────────────┐
  │ function safeMultiply(array) {                       │
  │   const output = [];                ← MẢNG MỚI!     │
  │   for (let i = 0; i < array.length; i++) {           │
  │     output.push(array[i] * 2);     ← Push vào MỚI   │
  │   }                                                  │
  │   return output;                   ← Return MỚI     │
  │ }                                                    │
  │                                                      │
  │ const myArray = [1, 2, 3];                           │
  │ const result = safeMultiply(myArray);                │
  │ console.log(myArray); // [1, 2, 3] ✅ KHÔNG ĐỔI!    │
  │ console.log(result);  // [2, 4, 6] ✅ Mảng mới!     │
  │ → Mọi hậu quả = return value ONLY!                  │
  └──────────────────────────────────────────────────────┘

  Will: "We want our function's behavior to be ENTIRELY
  defined by its return value, not that each time it runs
  it can have UNEXPECTED CONSEQUENCES to our global data."
```

### 6.2 Tại Sao Pure Functions Quan Trọng?


## §7. Pass By Reference vs Pass By Value

### 7.1 Bài Viết: Tại Sao array Trỏ Về Cùng Mảng?

Will nhấn mạnh: _"Were we passing in the original array... or were we making a copy? We were passing in a link back to our originally defined myArray."_

Khi truyền array vào function, JavaScript KHÔNG copy mảng — nó truyền **reference** (link). Parameter `array` bên trong function và `myArray` bên ngoài **trỏ đến CÙNG MỘT mảng** trên heap!

```
PASS BY REFERENCE — ARRAYS:
═══════════════════════════════════════════════════════════════

  const myArray = [1, 2, 3];
  copyArrayAndMultiplyBy2(myArray);

  MEMORY (thực tế):
```

```
  │ → Sửa arr = sửa myArr! (SIDE EFFECT!)               │
```

### 9.1 Pattern ①: 5 Whys — Tại Sao Cần copyArrayAndManipulate?

```
5 WHYS: TỪ COPY-PASTE ĐẾN HOF
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao tạo copyArrayAndManipulate?
  └→ Vì 3 functions (multiplyBy2, divideBy2, add3) vi phạm DRY
     — 85% code GIỐNG NHAU, chỉ khác 1 dòng!

  WHY ②: Tại sao DRY violation là vấn đề?
  └→ Vì mỗi bug phải sửa ở N nơi. Thêm 1 behavior =
     viết LẠI toàn bộ skeleton. Maintenance = ác mộng.

  WHY ③: Tại sao không truyền OPERATOR thay vì function?
  └→ Vì operators (+, -, *) là SYNTAX, không phải VALUES.
     Chỉ VALUES mới có thể lưu/truyền/return.
     Function = cách duy nhất "bọc" functionality thành value.

  WHY ④: Tại sao JS cho phép truyền function làm argument?
  └→ Vì functions là FIRST-CLASS CITIZENS — là values giống
     numbers, strings. Thiết kế lấy từ Scheme (1975).

  WHY ⑤: Tại sao tạo mảng mới thay vì sửa mảng gốc?
  └→ Vì arrays truyền bằng REFERENCE — sửa param = sửa gốc!
     → Side effect = hậu quả ngoài return value = UNPREDICTABLE.
     → Pure function = TẠO MỚI + return → safe, testable.
     ⭐ Đây là nền tảng của Immutability trong FP!
```

### 9.2 Pattern ②: First Principles

```javascript
// PHÂN RÃ copyArrayAndManipulate THÀNH CÁC THÀNH PHẦN TỐI THIỂU:

// Component 1: ITERATE — đi qua collection
function iterate(array, callback) {
  for (let i = 0; i < array.length; i++) {
    callback(array[i], i, array);
  }
}
// → Tách riêng LOGIC DUYỆT khỏi LOGIC XỬ LÝ

// Component 2: TRANSFORM — biến đổi 1 element
const double = (x) => x * 2;
// → Tách riêng LOẠI BIẾN ĐỔI

// Component 3: COLLECT — thu gom kết quả
function collect(array, transform) {
  const results = [];
  iterate(array, (element) => {
    results.push(transform(element));
  });
  return results;
}
// → Kết hợp iterate + transform + accumulator

// = copyArrayAndManipulate! Nhưng phân rã rõ ràng hơn:
collect([1, 2, 3], double); // [2, 4, 6]

// INSIGHT: Mọi HOF đều gồm 3 phần:
// ① Cách DUYỆT (for, while, recursion)
// ② Cách XỬ LÝ (callback)
// ③ Cách THU GOM (push, accumulate, filter)
```

### 9.5 Pattern ⑤: Reverse Engineering & Implementation

```javascript
// TỰ BUILD: IMMUTABILITY HELPER — chứng minh pure function hoạt động

class ImmutableArray {
  constructor(items = []) {
    // Freeze = KHÔNG cho sửa!
    this._items = Object.freeze([...items]);
  }

  // map — trả ImmutableArray MỚI
  map(callback) {
    const newItems = [];
    for (let i = 0; i < this._items.length; i++) {
      newItems.push(callback(this._items[i], i));
    }
    return new ImmutableArray(newItems); // MỚI!
  }

  // filter — trả ImmutableArray MỚI
  filter(predicate) {
    const newItems = [];
    for (let i = 0; i < this._items.length; i++) {
      if (predicate(this._items[i], i)) {
        newItems.push(this._items[i]);
      }
    }
    return new ImmutableArray(newItems);
  }

  // reduce — trả single value
  reduce(combiner, initial) {
    let acc = initial;
    for (let i = 0; i < this._items.length; i++) {
      acc = combiner(acc, this._items[i], i);
    }
    return acc;
  }

  // Xem nội dung
  toArray() {
    return [...this._items]; // Trả COPY, không trả gốc!
  }

  get length() {
    return this._items.length;
  }
}

// Sử dụng:
const arr = new ImmutableArray([1, 2, 3, 4, 5]);

const doubled = arr.map((x) => x * 2);
console.log(arr.toArray()); // [1, 2, 3, 4, 5] ← KHÔNG ĐỔI!
console.log(doubled.toArray()); // [2, 4, 6, 8, 10]

const evens = arr.filter((x) => x % 2 === 0);
console.log(evens.toArray()); // [2, 4]

const sum = arr.reduce((acc, x) => acc + x, 0);
console.log(sum); // 15

// Chaining:
const result = arr
  .filter((x) => x > 2) // [3, 4, 5]
  .map((x) => x * 10) // [30, 40, 50]
  .reduce((acc, x) => acc + x, 0); // 120

console.log(result); // 120
console.log(arr.toArray()); // [1, 2, 3, 4, 5] ← VẪN KHÔNG ĐỔI! ✅

// Thử sửa trực tiếp:
try {
  arr._items[0] = 99; // TypeError: Cannot assign to read-only property!
} catch (e) {
  console.log("✅ Object.freeze ngăn mutation!");
}
```

```

  │     → Quá academic, ít áp dụng thực tế
  │
  ↓
  1995: JavaScript — Chiến trường giữa 2 thế giới
  │     → Array.push/pop = MUTATION (từ C)
  │     → map/filter = IMMUTABLE (từ Scheme)
  │     → Developer tự chọn style
  │
  ↓
  2009: ES5 — Array.prototype.map, filter, reduce
  │     → HOFs = cách CHÍNH THỐNG xử lý arrays
  │     → "Functional JavaScript" phong trào bắt đầu
  │
  ↓
  2013: React — "Treat state as IMMUTABLE"
  │     → setState(newState), không mutate trực tiếp
  │     → Reconciliation dựa trên reference equality
  │     → Mutation → React KHÔNG detect thay đổi!
  │
  ↓

        → RAM không còn là bottleneck
```
