## §1. new Date() — Object Với Hidden Property

### 1.1 Bài Viết: Date = Built-In Function

> Will: _"Date is a built-in function in JavaScript, available to us as soon as we switch on JavaScript's engine. When we call it with the `new` keyword, we get an object out that has a hidden property called [[DateValue]]."_

```
NEW DATE() — TRẢ VỀ OBJECT:
═══════════════════════════════════════════════════════════════

  const time1 = new Date();

  GLOBAL MEMORY (trước khi chạy):
  ┌──────────────────────────────────────────────────┐
  │ Date: ƒ (built-in!) ← có sẵn khi JS engine bật │
  │ time1: ???                                       │
  └──────────────────────────────────────────────────┘

  new Date() trả về:
  ┌──────────────────────────────────────────────────┐
  │ OBJECT (trông rỗng từ bên ngoài!):              │
  │ {                                                │
  │   // KHÔNG CÓ visible properties!               │
  │                                                  │
  │   [[DateValue]]: 1800000000000                   │
  │   ↑ HIDDEN PROPERTY!                             │
  │   ↑ Không thể access bằng time1.DateValue!      │
  │   ↑ Không thể access bằng time1.[[DateValue]]!  │
  │ }                                                │
  └──────────────────────────────────────────────────┘

  Will: "This object is going to have a HIDDEN property
  on it — [[DateValue]] — that has a number of
  milliseconds since January 1st, 1970 at midnight."

  → new keyword → sẽ học chi tiết ở phần Classes!
  → Bây giờ chỉ cần biết: new Date() → object + hidden data!
```

## §5. Square Bracket Notation — Gán Property Động

### 5.1 Bài Viết: [] vs Dot Notation

> Will: _"I want us to be reminded of square bracket access and property setting, because we're going to REALLY rely on using square brackets to add a property — that's very important for metaprogramming very soon."_

```
SQUARE BRACKET NOTATION — DYNAMIC PROPERTY ACCESS:
═══════════════════════════════════════════════════════════════

  const month = "Jan";

  // Dot notation — LITERAL name:
  time1.month = true;
  // → Tạo property "month" (chữ "month"!)
  // → { month: true }

  // Square bracket — EVALUATED expression:
  time1[month] = true;
  // → month = "Jan" → evaluate!
  // → time1["Jan"] = true
  // → { Jan: true }

  ⚠️ KHÁC BIỆT QUAN TRỌNG:
  ┌──────────────────────────────────────────────────┐
  │ time1.month = true                               │
  │ → key = "month" (literal string "month")         │
  │ → KHÔNG evaluate!                                │
  │                                                  │
  │ time1[month] = true                              │
  │ → month = variable → evaluate → "Jan"            │
  │ → key = "Jan"                                    │
  │ → HAS evaluate!                                  │
  └──────────────────────────────────────────────────┘

  Will: "Square brackets say: go find the time1
  object, and assign whatever that thing inside
  evaluates to. Month is NOT your property name!
  Month is a thing to go LOOK UP. Whatever you
  find stored in month in memory — Jan — THAT'S
  your property name."

  TRACE:
  ┌──────────────────────────────────────────────────┐
  │ time1[month] = true;                             │
  │                                                  │
  │ Step 1: tìm time1 → @1001 (object in heap)      │
  │ Step 2: evaluate month → "Jan"                   │
  │ Step 3: time1["Jan"] = true                      │
  │ Step 4: thêm property Jan: true vào object       │
  │                                                  │
  │ time1 = {                                        │
  │   [[DateValue]]: 1800000000000,  ← hidden        │
  │   Jan: true                      ← visible!      │
  │ }                                                │
  └──────────────────────────────────────────────────┘

  time2[month] = true;
  → Tương tự: time2.Jan = true

  → time1 và time2 = REGULAR OBJECTS!
  → Có thể thêm properties bình thường!
  → Chứng minh: Date object = chỉ là object thường
    + 1 hidden property!
```

### 5.2 Tại Sao Square Brackets Quan Trọng Cho Metaprogramming?

```
SQUARE BRACKETS → METAPROGRAMMING:
═══════════════════════════════════════════════════════════════

  VỚI DOT NOTATION:
  → time1.name = "Will"
  → Property name = PHẢI là string literal!
  → KHÔNG THỂ dùng Symbol! (vì Symbol ≠ string!)

  VỚI SQUARE BRACKETS:
  → const sym = Symbol("myProp");
  → time1[sym] = "secret";
  → Property name = SYMBOL! ← key cho metaprogramming!

  Will đang setup cho:
  → Symbol.toPrimitive = hidden Symbol key!
  → obj[Symbol.toPrimitive] = function(hint) { ... }
  → = CÁch để control coercion pipeline!
  → = METAPROGRAMMING!

  → Sẽ học ở bài tiếp theo! 🔮
```

```javascript
const time1 = new Date();
```

```javascript
// CHỨNG MINH 2: Square bracket = evaluate, dot = literal

const key = "name";

const obj = {};

console.log(obj);
// { key: "dot", name: "bracket" }
//   ↑ literal     ↑ evaluated!

// Với nhiều keys:
const props = ["a", "b", "c"];
const dynamicObj = {};
props.forEach((p, i) => {
  dynamicObj[p] = i; // dynamic property names!

// → Dot notation KHÔNG THỂ làm điều này!
```

```javascript
// CHỨNG MINH 3: Date comparison — THE REAL WAY

const time1 = new Date();
// Simulate 3 seconds passing:
const time2 = new Date(time1.getTime() + 3000);

// SURPRISE: time2 - time1 ACTUALLY WORKS!
console.log(time2 - time1); // 3000 ← milliseconds! 🤯

// HOW? Because - operator → toNumber → toPrimitive!
// Date object's toPrimitive("number") → [[DateValue]]!
// → 1800000003000 - 1800000000000 = 3000!

// BUT: time2 + time1 → STRING concat! (+ = toString!)
console.log(time2 + time1); // "Mon Jan 15 2027...Mon Jan 15 2027..."
// → toPrimitive("default") → toString for Date!

// SAFER: explicit conversion
console.log(time2.getTime() - time1.getTime()); // 3000 ✅
console.log(+time2 - +time1); // 3000 ✅
console.log(Number(time2) - Number(time1)); // 3000 ✅
```

```javascript
// CHỨNG MINH 4: Prevent double submit — FULL implementation

function createSubmitGuard(cooldownMs = 2000) {
  let lastSubmitTime = null;

  return function onSubmit(formData) {
    const now = new Date();

    if (lastSubmitTime !== null) {
      const elapsed = now - lastSubmitTime;
      // now - lastSubmitTime → toPrimitive → number math!
      // → milliseconds difference!

      if (elapsed < cooldownMs) {
        console.log(
          `⚠️ Too fast! Wait ${cooldownMs - elapsed}ms. ` +
            `(${elapsed}ms since last submit)`,
        );
        return { success: false, reason: "cooldown" };
      }
    }

    lastSubmitTime = now;
    console.log(`✅ Submitted: ${JSON.stringify(formData)}`);
    return { success: true, data: formData };
  };
}

const submitOrder = createSubmitGuard(2000);

submitOrder({ item: "Widget", qty: 3 });
// ✅ Submitted!

// Simulate rapid second click (500ms later):
setTimeout(() => {
  submitOrder({ item: "Widget", qty: 3 });
  // ⚠️ Too fast! Wait 1500ms.
}, 500);

// Simulate click after cooldown (3s later):
setTimeout(() => {
  submitOrder({ item: "Widget", qty: 3 });
  // ✅ Submitted!
}, 3000);
```

```javascript
// CHỨNG MINH 5: Hidden properties — Object.getOwnPropertyNames
```

## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Date & Hidden Properties

```
5 WHYS: TẠI SAO DATE LƯU MILLISECONDS?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao Date lưu milliseconds thay vì date string?
  └→ Vì MATH trên numbers = dễ! (3000ms - 0ms = 3000ms)
     MATH trên strings = khó! ("2027-01-15" - "2027-01-12" = ???)

  WHY ②: Tại sao đếm từ January 1, 1970?
  └→ Vì UNIX EPOCH! Unix OS (1971) chọn Jan 1, 1970
     = ngày tròn, gần thời điểm Unix ra đời
     → Trở thành TIÊU CHUẨN cho hầu hết languages!

  WHY ③: Tại sao [[DateValue]] là HIDDEN?
  └→ Vì ENCAPSULATION! User không nên trực tiếp modify
     millisecond value → có thể tạo invalid date!
     → Access qua methods: getTime(), getFullYear()...

  WHY ④: Tại sao hidden dùng [[double brackets]]?
  └→ Vì CONVENTION trong ECMAScript spec!
     [[ ]] = "internal slot" = engine-only access
     → Developer đọc spec biết đây là internal!

  WHY ⑤: Tại sao cần hiểu hidden properties?
  └→ Vì METAPROGRAMMING sắp tới!
     Symbol.toPrimitive = cách ACCESS/OVERRIDE
     hidden behavior! → CONTROL coercion pipeline!
```

### 9.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — DATE TRONG ECMASCRIPT:
═══════════════════════════════════════════════════════════════

  21.4 Date Objects (ECMAScript Spec):

  21.4.1.1 Time Values:
  → "A Number — finite or NaN"
  → "Represents a moment in time as milliseconds
     since 01 January 1970 UTC"
  → Range: ±8,640,000,000,000,000 ms
     (±100 million days = ~±273,000 years!)

  21.4.2 The Date Constructor:
  → new Date() → sets [[DateValue]] to current time
  → new Date(ms) → sets [[DateValue]] to ms
  → new Date(year, month, ...) → compute ms

  21.4.4 Properties of Date Prototype:
  → .getTime() → returns [[DateValue]] ← ACCESSOR!
  → .valueOf() → returns [[DateValue]] ← cho coercion!
  → [Symbol.toPrimitive](hint):
     → hint="number" → valueOf() → number
     → hint="string" → toString() → string
     → hint="default" → toString() → string ← !!!

  → Date = đặc biệt! hint="default" → STRING (không phải number)!
  → Hầu hết objects: hint="default" → valueOf() (number)!
  → Date override để + concat thay vì math!
```

### 9.4 Pattern ④: Mental Mapping

```
MENTAL MAP: DATE TRONG REAL-WORLD:
═══════════════════════════════════════════════════════════════

  ┌─────────────────────────────────────────────────────────┐
  │ DEBOUNCE / THROTTLE — dùng Date!                       │
  │                                                         │
  │ function throttle(fn, delay) {                          │
  │   let lastCall = 0;                                     │
  │   return function(...args) {                            │
  │     const now = Date.now(); // ms (number!)             │
  │     if (now - lastCall >= delay) {                      │
  │       lastCall = now;                                   │
  │       fn.apply(this, args);                             │
  │     }                                                   │
```

### 9.5 Pattern ⑤: Reverse Engineering

```javascript
// TỰ BUILD: Simplified Date object from scratch

function MyDate(msOrNow) {
  // Hidden property (simulate with closure!)
  let dateValue;

  if (msOrNow === undefined) {
    dateValue = Date.now(); // current time in ms
  } else {
    dateValue = Number(msOrNow);
  }

  // Return object with methods to access hidden value
  const dateObj = {
    // Accessor method (like .getTime())
    getTime() {
      return dateValue;
    },

    // valueOf → used by toNumber coercion!
    valueOf() {
      return dateValue;
    },

    // toString → used by toString coercion!
    toString() {
      return new Date(dateValue).toISOString();
    },

    // toPrimitive → controls coercion!
    [Symbol.toPrimitive](hint) {
      if (hint === "number") return dateValue;
      if (hint === "string") return this.toString();
      // "default" → Date returns STRING (unique!)
      return this.toString();
    },
  };

  return dateObj;
}

// Test:
```
