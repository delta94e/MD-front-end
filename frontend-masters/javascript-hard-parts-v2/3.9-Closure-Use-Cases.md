## §1. Tại Sao Functions KHÔNG Nhớ — Và Tại Sao Cần Nhớ

### 1.1 Bài Viết: Hai Mặt Của Đồng Xu

> Will: _"It's fantastic that functions DON'T remember their previous invocations — get started again. That's a key part of their design, modularizing our code."_

```
TẠI SAO KHÔNG NHỚ = TỐT:
═══════════════════════════════════════════════════════════════

  function multiplyBy2(input) {
    return input * 2;
  }

  multiplyBy2(7);  // 14
  multiplyBy2(10); // 20 ← KHÔNG phải 14! Fresh!

  Will: "I don't want to run multiplyBy2 on 7, and
  then run it again on 10, and it's still got the
  MEMORY of running it on 7, because it's like
  a sort of reverie from... you want to start AFRESH."

  → "PURE FUNCTION" = cùng input → cùng output!
  → KHÔNG side effects từ lần chạy trước!
  → Predictable, testable, modular!

TẠI SAO CẦN NHỚ = CŨNG TỐT:
═══════════════════════════════════════════════════════════════

  NHƯNG đôi khi BẠN CẦN nhớ:
  → Đã chạy bao nhiêu lần? (once)
  → Kết quả tính toán cũ? (memoize)
  → Đang ở vị trí nào? (iterator)
  → Đã thực hiện đến dòng nào? (generator)
  → Private state cho module? (module pattern)
  → Data từ lúc setup? (async callbacks)

  → CLOSURE = cho functions khả năng NHỚ!
  → "If we can also give them a special store of
     PERSISTENT DATA, that is powerful, and it is
     a vital part of the toolkit for writing PRO CODE."
```

## §2. Use Case ①: once() — Chỉ Chạy 1 Lần

### 2.1 Bài Viết: Oncify A Function

> Will: _"The once function — you might oncify a multiplyBy2, first time it takes in 7, you go 14 great. Next time it takes in 10, you say sorry, I've ALREADY been run, because it checks the counter in the backpack."_

```
ONCE — CƠ CHẾ:
═══════════════════════════════════════════════════════════════

  once(fn) trả về 1 function MỚI mà:
  → Lần gọi đầu tiên: CHẠY fn, lưu kết quả!
  → Lần gọi thứ 2+: KHÔNG chạy fn, trả kết quả cũ!

  BACKPACK chứa:
  🎒 { hasRun: false, result: undefined }

  Lần 1: hasRun? → false → chạy fn → lưu result → hasRun = true
  Lần 2: hasRun? → true → KHÔNG chạy → return result cũ!

  Will: "You might want to prevent a WIN function in
  a game running more than once, in order that the
  user can't accidentally win TWICE."
```

```javascript
// TỰ BUILD: once()

function once(fn) {
  let hasRun = false; // 🎒 backpack!
  let savedResult; // 🎒 backpack!

  return function (...args) {
    if (!hasRun) {
      savedResult = fn(...args);
      hasRun = true;
    }
    return savedResult;
  };
}

// SỬ DỤNG:
const onceMultiply = once((x) => {
  console.log("Computing...");
  return x * 2;
});

onceMultiply(7); // "Computing..." → 14
onceMultiply(10); // (im lặng!) → 14 ← KHÔNG chạy lại!
onceMultiply(99); // (im lặng!) → 14 ← VẪN KẾT QUẢ CŨ!

// TRACE:
// Call 1: hasRun=false → chạy fn(7) → result=14 → hasRun=true
// Call 2: hasRun=true → skip fn → return 14
// Call 3: hasRun=true → skip fn → return 14
```

## §3. Use Case ②: memoize() — Cache Kết Quả Tốn Kém

### 3.1 Bài Viết: Tránh Tính Lại

> Will: _"If you have a function nthPrime that takes in 1.2 million, and tries to calculate the 1.2 millionth prime — that's going to have to do a LOT of math. I don't want the next time that somebody passes in 1.2 million to have to REDO that math."_

```
MEMOIZE — CƠ CHẾ:
═══════════════════════════════════════════════════════════════

  memoize(fn) trả về function MỚI mà:
  → Check: input này ĐÃ TÍNH chưa?
  → Rồi: trả kết quả từ CACHE (backpack!) → O(1)!
  → Chưa: tính → LƯU vào cache → trả kết quả

  BACKPACK chứa:
  🎒 { cache: { "1200000": 19588457, "42": 181, ... } }

  Will: "Store as a pair of values: 1.2 million with
  whatever the 1.2 millionth prime number is. Where
  can I store that? In the nthPrime function's BACKPACK."

  LẦN 1: nthPrime(1200000)
  ┌──────────────────────────────────────────────────┐
  │ Check cache["1200000"] → MISS! ❌                │
  │ → Tính toán nặng... 5 giây... → 19588457        │
  │ → cache["1200000"] = 19588457                    │
  │ → return 19588457                                │
  └──────────────────────────────────────────────────┘

  LẦN 2: nthPrime(1200000)
  ┌──────────────────────────────────────────────────┐
  │ Check cache["1200000"] → HIT! ✅                 │
  │ → return 19588457 NGAY LẬP TỨC! 0ms!            │
  └──────────────────────────────────────────────────┘
```

```javascript
// TỰ BUILD: memoize()

function memoize(fn) {
  const cache = {}; // 🎒 backpack! { input → result }

  return function (...args) {
    const key = JSON.stringify(args); // Tạo cache key

    if (key in cache) {
      console.log(`Cache HIT for ${key}`);
      return cache[key]; // Trả từ backpack!
    }

    console.log(`Cache MISS for ${key} — computing...`);
    const result = fn(...args);
    cache[key] = result; // Lưu vào backpack!
    return result;
  };
}

// SỬ DỤNG:
function slowAdd(a, b) {
  // Giả lập tính toán nặng:
  let sum = 0;
  for (let i = 0; i < 1e8; i++) sum; // ~200ms
  return a + b;
}

const fastAdd = memoize(slowAdd);
fastAdd(3, 4); // "Cache MISS..." → ~200ms → 7
fastAdd(3, 4); // "Cache HIT..."  → ~0ms   → 7 ← INSTANT!
fastAdd(5, 6); // "Cache MISS..." → ~200ms → 11
fastAdd(5, 6); // "Cache HIT..."  → ~0ms   → 11 ← INSTANT!
```

## §4. Use Case ③: Iterators — Duyệt Từng Phần Tử

### 4.1 Bài Viết: Function "Nhớ" Vị Trí

> Will: _"Iterators allow us to not look at a list and use a for loop, but instead have a function that when run, returns out the first time the 0th element, the next time the next element. But a function doesn't remember the last time it was run — so it doesn't know what position it's in. Don't panic, it keeps track of it in its BACKPACK."_

```
ITERATOR — CƠ CHẾ:
═══════════════════════════════════════════════════════════════

  createIterator(array) trả về function:
  → Mỗi lần gọi → trả phần tử TIẾP THEO!
  → "Nhớ" vị trí bằng BACKPACK!

  BACKPACK chứa:
  🎒 { array: [1,2,3], index: 0 }

  Gọi lần 1: index=0 → return array[0]=1 → index=1
  Gọi lần 2: index=1 → return array[1]=2 → index=2
  Gọi lần 3: index=2 → return array[2]=3 → index=3
  Gọi lần 4: index=3 → hết! → return undefined
```

```javascript
// TỰ BUILD: createIterator()

function createIterator(array) {
  let index = 0; // 🎒 backpack! Nhớ vị trí!

  return function next() {
    if (index < array.length) {
      const value = array[index];
      index++; // Cập nhật vị trí trong backpack!
      return { value, done: false };
    }
    return { value: undefined, done: true };
  };
}

// SỬ DỤNG:
const iter = createIterator(["a", "b", "c"]);

iter(); // { value: "a", done: false } — 🎒 index: 0→1
iter(); // { value: "b", done: false } — 🎒 index: 1→2
iter(); // { value: "c", done: false } — 🎒 index: 2→3
iter(); // { value: undefined, done: true } — hết!

// → Giống Symbol.iterator trong JS!
// → for...of, spread (...), destructuring ĐỀU dùng iterators!
```

## §5. Use Case ④: Generators — Tạm Dừng & Tiếp Tục

### 5.1 Bài Viết: Pause A Function

> Will: _"Generators say that you could PAUSE the running of a function halfway through — potentially until some asynchronous work gets back the result, then you can CONTINUE ON. What do you need to continue a function on? You need its local memory, you need the line position in the code. Where's that stored? In CLOSURE, in the backpack."_

```
GENERATOR — CƠ CHẾ:
═══════════════════════════════════════════════════════════════

  Generator = function đặc biệt có thể:
  → PAUSE (yield) giữa chừng!
  → RESUME (next()) khi cần!
  → NHỚ local memory + dòng code trong BACKPACK!

  BACKPACK chứa:
  🎒 { localMemory: {...}, linePosition: 3 }

  function* gen() {
    let x = 1;         // line 1
    yield x;           // line 2 — PAUSE! 🎒 lưu x=1, line=2
    x = x + 10;        // line 3 — RESUME! đọc 🎒 x=1
    yield x;           // line 4 — PAUSE! 🎒 lưu x=11, line=4
    return x * 2;      // line 5 — RESUME! đọc 🎒 x=11
  }

  const g = gen();
  g.next(); // { value: 1,  done: false } — pause at line 2
  g.next(); // { value: 11, done: false } — pause at line 4
  g.next(); // { value: 22, done: true }  — finished!
```

```javascript
// TỰ BUILD: Simulate generator bằng closure!

function createGenerator(steps) {
  let currentStep = 0; // 🎒 line position!
  let state = {}; // 🎒 local memory!

  return {
    next(input) {
      if (currentStep >= steps.length) {
        return { value: undefined, done: true };
      }
      const result = steps[currentStep](state, input);
      currentStep++; // "Di chuyển" đến step tiếp theo
      return { value: result, done: currentStep >= steps.length };
    },
  };
}

// Giả lập: function* gen() { let x = 1; yield x; x += 10; yield x; }
const gen = createGenerator([
  (state) => {
    state.x = 1;
    return state.x;
  }, // yield x (=1)
  (state) => {
    state.x = state.x + 10;
    return state.x;
  }, // yield x (=11)
]);

console.log(gen.next()); // { value: 1,  done: false }
console.log(gen.next()); // { value: 11, done: true }

// → currentStep + state = BACKPACK!
// → Giữ vị trí + local memory qua các lần gọi!
```

## §6. Use Case ⑤: Module Pattern — Private Persistent Data

> Will: _"The module pattern — the ability for us to have PERSISTENT labels that don't pollute the global memory. You can't just have it in a function, because once that execution context closes, that label's gone. So you have to have a way of having LIFETIME memories, but NOT in global. And the answer is in BACKPACKS."_

```
MODULE PATTERN — CƠ CHẾ:
═══════════════════════════════════════════════════════════════

  VẤN ĐỀ:
  → Cần state PERSISTENT → không dùng local (xoá mỗi call!)
  → Cần state PRIVATE → không dùng global (ai cũng thấy!)
  → CLOSURE = persistent + private = SOLUTION!

  Will: "Have a look at the JavaScript module pattern.
  It implements using the backpack, using closure,
  using functions having PERSISTENT MEMORY."
```

```javascript
// TỰ BUILD: Module pattern — API service

const ApiService = (function () {
  // 🎒 PRIVATE — chỉ methods bên dưới access!
  let baseUrl = "https://api.example.com";
  let authToken = null;
  const requestLog = [];

  // 🎒 PRIVATE HELPER
  function logRequest(method, endpoint) {
    requestLog.push({
      method,
      endpoint,
      timestamp: Date.now(),
      authenticated: !!authToken,
    });
  }

  // PUBLIC API
  return {
    setAuth(token) {
      authToken = token; // Sửa backpack!
    },

    async get(endpoint) {
      logRequest("GET", endpoint);
      const headers = authToken ? { Authorization: `Bearer ${authToken}` } : {};
      // fetch(`${baseUrl}${endpoint}`, { headers });
      return `GET ${baseUrl}${endpoint}`;
    },

    getRequestCount() {
      return requestLog.length; // Đọc backpack!
    },

    getLog() {
      return [...requestLog]; // Return copy!
    },
  };
})();

ApiService.setAuth("my-secret-token");
ApiService.get("/users");
ApiService.get("/posts");
console.log(ApiService.getRequestCount()); // 2
console.log(ApiService.getLog()); // [{...}, {...}]

// PRIVATE:
console.log(ApiService.authToken); // undefined ❌
console.log(ApiService.requestLog); // undefined ❌
console.log(ApiService.baseUrl); // undefined ❌
```

## §7. Use Case ⑥: Async Callbacks & Promises

### 7.1 Bài Viết: Data Vẫn Available Khi Callback Chạy

> Will: _"Some of the code we want to run may involve OTHER DATA besides the data we got back from the API call. How can we have that data still available to us? Don't panic, the function we run will of course have a BACKPACK of the data from when it was defined."_

```
ASYNC + CLOSURE — CƠ CHẾ:
═══════════════════════════════════════════════════════════════

  TIMELINE:
  t=0ms:    setupFetch() chạy
            → userId = 42, startTime = now ← BACKPACK!
            → fetch gửi request
            → setupFetch() KẾT THÚC! EC xoá!

  t=2000ms: Response từ API đến!
            → callback chạy
            → CẦN userId → BACKPACK! ✅
            → CẦN startTime → BACKPACK! ✅
            → CLOSURE = CỨU TINH! 🎒

  Will: "We didn't go and get the latest Spotify tracks
  to not do something with them. We wanted to run some
  CODE on them. But some code may also involve OTHER DATA
  besides the data from the API call. Don't panic —
  the function has a BACKPACK."
```

```javascript
// TỰ BUILD: Async closure — real-world pattern

function createApiHandler(userId, config) {
  // 🎒 BACKPACK: userId, config, requestCount
  let requestCount = 0;
  const startTime = Date.now();

  return async function fetchData(endpoint) {
    requestCount++; // Đếm requests — backpack!

    console.log(`[User ${userId}] Request #${requestCount} to ${endpoint}`);
    // userId từ BACKPACK! Dù createApiHandler đã kết thúc!

    try {
      const response = await fetch(`${config.baseUrl}${endpoint}`, {
        headers: { Authorization: `Bearer ${config.token}` },
        // config từ BACKPACK!
      });
      const data = await response.json();

      const elapsed = Date.now() - startTime;
      // startTime từ BACKPACK!
      console.log(`Completed in ${elapsed}ms`);

      return data;
    } catch (error) {
      console.error(`[User ${userId}] Error: ${error.message}`);
      // userId VẪNTỪ backpack dù chạy 5 phút sau!
    }
  };
}

// Setup:
const handler = createApiHandler(42, {
  baseUrl: "https://api.example.com",
  token: "secret",
});
// createApiHandler ĐÃ KẾT THÚC! EC XOÁ!

// 5 phút sau... callback chạy:
handler("/users"); // "[User 42] Request #1..."
handler("/posts"); // "[User 42] Request #2..."
// userId, config, requestCount, startTime = TẤT CẢ TỪ BACKPACK!
```

```javascript
// ═══════════════════════════════════════════════
// ① ONCE — Advanced: với reset capability
// ═══════════════════════════════════════════════

function once(fn) {
  let called = false;
  let result;

  const wrapper = function (...args) {
    if (!called) {
      result = fn.apply(this, args);
      called = true;
    }
    return result;
  };

  // Bonus: reset functionality
  wrapper.reset = function () {
    called = false;
    result = undefined;
  };

  return wrapper;
}

const initialize = once(() => {
  console.log("Initializing app...");
  return { db: "connected", cache: "ready" };
});

initialize(); // "Initializing app..." → { db: "connected", cache: "ready" }
initialize(); // (im lặng) → { db: "connected", cache: "ready" }
initialize.reset();
initialize(); // "Initializing app..." → lại chạy!
```

```javascript
// ═══════════════════════════════════════════════
// ② MEMOIZE — Advanced: với LRU cache limit
// ═══════════════════════════════════════════════

function memoize(fn, maxSize = 100) {
  const cache = new Map(); // Map giữ insertion order!

  return function (...args) {
    const key = JSON.stringify(args);

    if (cache.has(key)) {
      // Move to end (most recently used):
      const value = cache.get(key);
      cache.delete(key);
      cache.set(key, value);
      return value;
    }

    const result = fn(...args);

    // Evict oldest if over limit:
    if (cache.size >= maxSize) {
      const oldestKey = cache.keys().next().value;
      cache.delete(oldestKey);
    }

    cache.set(key, result);
    return result;
  };
}

// Ví dụ: Fibonacci memoized
const fib = memoize(function (n) {
  if (n <= 1) return n;
  return fib(n - 1) + fib(n - 2);
});

console.log(fib(50)); // 12586269025 — instant! (without memo = freeze!)
```

```javascript
// ③ ITERATOR — Advanced: Symbol.iterator compatible
// ═══════════════════════════════════════════════

function createRangeIterator(start, end, step = 1) {
  let current = start; // 🎒

  return {
    next() {
      if (current <= end) {
        const value = current;
        current += step;
        return { value, done: false };
      }
      return { value: undefined, done: true };
    },
    // Make it iterable (for...of compatible):
    [Symbol.iterator]() {
      return this;
    },
  };
}

const range = createRangeIterator(1, 5);
for (const num of range) {
  console.log(num); // 1, 2, 3, 4, 5
}

// Infinite iterator:
function createIdGenerator(prefix = "id") {

  return {
    next() {
      return { value: `${prefix}_${++counter}`, done: false };
    },
    [Symbol.iterator]() {
      return this;
    },
  };
}

const ids = createIdGenerator("user");
console.log(ids.next().value); // "user_1"
console.log(ids.next().value); // "user_2"
console.log(ids.next().value); // "user_3"
```

```javascript
function simulateGenerator() {
  let step = 0; // 🎒 line position
  let localX; // 🎒 local memory

  return {
    next(input) {
      switch (step) {
        case 0:
          localX = 1;
          step = 1;
          return { value: localX, done: false }; // yield x

        case 1:
          localX = localX + (input || 0) + 10;
          step = 2;
          return { value: localX, done: false }; // yield x

        case 2:
          step = 3;
          return { value: localX * 2, done: true }; // return x * 2

        default:
          return { value: undefined, done: true };
      }
    },
  };
}

//   return x * 2;

const g = simulateGenerator();
```

```javascript
// ═══════════════════════════════════════════════
// ⑤ MODULE PATTERN — Full implementation
// ═══════════════════════════════════════════════

const EventBus = (function () {
  // 🎒 PRIVATE:
  const listeners = {}; // { eventName: [callback, ...] }

  return {
    on(event, callback) {
      if (!listeners[event]) listeners[event] = [];
      listeners[event].push(callback);
      // Return unsubscribe function (also a closure!)
      return () => {
        listeners[event] = listeners[event].filter((cb) => cb !== callback);
      };
    },

    emit(event, data) {
      if (!listeners[event]) return;
      listeners[event].forEach((cb) => cb(data));
    },

    listenerCount(event) {
      return (listeners[event] || []).length;
    },
  };
})();

const unsub = EventBus.on("click", (data) => console.log("Clicked!", data));
EventBus.emit("click", { x: 10, y: 20 }); // "Clicked! {x:10, y:20}"
console.log(EventBus.listenerCount("click")); // 1
unsub(); // Unsubscribe! (closure returns closure!)
console.log(EventBus.listenerCount("click")); // 0
```

```javascript
// TỰ BUILD: Universal closure factory — tạo BẤT KỲ pattern nào

function createClosurePattern(config) {
  // 🎒 BACKPACK = config.initialState
  const state = { ...config.initialState };

  return function execute(...args) {
    // Check guard (like once):
    if (config.guard && !config.guard(state, args)) {
      return config.guardReturn?.(state) ?? undefined;
    }

    // Check cache (like memoize):
    const key = config.cacheKey?.(args);
    if (key && state.cache?.[key] !== undefined) {
      return state.cache[key];
    }

    // Execute main logic:
    const result = config.execute(state, args);

    // Post-execute updates:
    config.afterExecute?.(state, args, result);

    // Cache result if needed:
    if (key) {
      state.cache = state.cache || {};
      state.cache[key] = result;
    }

    return result;
  };
}

// once() via factory:
const onceAdd = createClosurePattern({
  initialState: { called: false, result: null },
  guard: (state) => !state.called,
  guardReturn: (state) => state.result,
  execute: (state, [a, b]) => a + b,
  afterExecute: (state, _, result) => {
    state.called = true;
    state.result = result;
  },
});

console.log(onceAdd(3, 4)); // 7
console.log(onceAdd(5, 6)); // 7 (still!)
```
