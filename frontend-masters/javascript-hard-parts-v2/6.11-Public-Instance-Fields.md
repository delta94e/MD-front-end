## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Public Instance Fields

```
5 WHYS: TẠI SAO INSTANCE FIELDS?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao cần instance fields?
  └→ Vì muốn DEFAULT VALUES cho mọi instance!
     loggedIn = false cho EVERY user!
     Trước đây phải viết trong constructor!

  WHY ②: Tại sao không viết trong constructor?
  └→ Có thể! Nhưng instance fields TÁCH default values
     ra khỏi logic! Cleaner code!
     "loggedIn = false" là DECLARATION, không phải logic!

  WHY ③: Tại sao cần hidden [[Fields]]?
  └→ Vì fields được DEFINED ngoài constructor!
     JS cần nơi lưu trữ chúng trên function-object!
     Rồi auto-insert khi new chạy!

  WHY ④: Tại sao fields set TRƯỚC constructor?
  └→ Vì constructor có thể REFERENCE chúng!
     constructor có thể kiểm tra this.loggedIn!
     Nên fields phải đã exist trước!

  WHY ⑤: Tại sao mỗi instance gets OWN copy?
  └→ Vì instance data = PER-OBJECT!
     user1.loggedIn = true ≠ user2.loggedIn!
     Shared prototype chỉ cho methods, không cho data!
```

### 9.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — new WITH [[Fields]]:
═══════════════════════════════════════════════════════════════

  ECMAScript [[Construct]] WITH class fields:

  1. Create empty object (this = {})
  2. Set [[Prototype]] = F.prototype
  3. 🆕 InitializeInstanceElements(this)!
     → For each field in [[Fields]]:
        → DefineField(this, field.name, field.value)
  4. Run constructor with this
  5. Return this

  [[Fields]] is an INTERNAL SLOT on the function:
  → Not visible via Object.keys()!
  → Not on .prototype!
  → Defined by ClassFieldDefinitionEvaluation!
  → Auto-processed during [[Construct]]!
```

### 9.3 Pattern ③: Trade-off Analysis

```
TRADE-OFFS — FIELDS IN CONSTRUCTOR vs CLASS BODY:
═══════════════════════════════════════════════════════════════

  ┌──────────────────────┬──────────────┬──────────────┐

  ├──────────────────────┼──────────────┼──────────────┤

  │ Default values       │ ⚠️ Mixed w/  │ ✅ Separated! │
```

### 9.5 Pattern ⑤: Reverse Engineering

```javascript
// PROVE: Fields are processed BEFORE constructor body

class Test {
  x = 10;
  y = 20;

  constructor() {
    // Fields already exist!
    console.log("x:", this.x); // 10 ← already set!
    console.log("y:", this.y); // 20 ← already set!

    // We can even override them:
    this.x = 999;
  }
}

const t = new Test();
console.log(t.x); // 999 (constructor overrode field!)
console.log(t.y); // 20 (field value kept!)

// PROVE: Fields create OWN properties
// (not prototype, not inherited)
console.log(t.hasOwnProperty("x")); // true
console.log(t.hasOwnProperty("y")); // true
console.log(Test.prototype.x); // undefined (NOT on prototype!)

// PROVE: Field initializers run per-instance
class Counter {
  id = Math.random(); // different each time!
}

const a = new Counter();
const b = new Counter();
console.log(a.id === b.id); // false! Each gets unique value!
```
