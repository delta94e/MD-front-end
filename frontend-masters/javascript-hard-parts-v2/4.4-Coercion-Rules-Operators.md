## §1. 3 Coercion Flows — toNumber, toString, toBoolean

### 1.1 Bài Viết: 3 Funnels — Chỉ Cần Nhớ 3 Thứ!

> Will: _"We only have 3 things to take away: our 3 coercion journeys that can happen between our data types, and then the things that can KICK THOSE OFF."_

```
3 COERCION FLOWS — MASTER VIEW:
═══════════════════════════════════════════════════════════════

  BẤT KỲ giá trị JS nào → có thể bị đẩy qua 3 "funnels":

  ┌────────────────────────────────────────────────────────┐
  │                                                        │
  │  INPUT (bất kỳ type)                                   │
  │     │                                                  │
  │     ├──→ 🔢 toNumber  → number (hoặc NaN)             │
  │     │                                                  │
  │     ├──→ 📝 toString  → string                        │
  │     │                                                  │
  │     └──→ ✅ toBoolean → true hoặc false               │
  │                                                        │
  │  PIPELINE NÀO được chọn?                               │
  │  → Phụ thuộc vào OPERATOR nào kích hoạt!               │
  │  → * - / % ** → toNumber                               │
  │  → + (có string) → toString                            │
  │  → if () → toBoolean                                   │
  │                                                        │
  └────────────────────────────────────────────────────────┘

  Will: "We have 2 or 3 major funnels that can convert,
  coerce stuff into a number, a string, or a true/false,
  and each is kicked off automatically by a range of
  OPERATORS and/or ACTIONS."
```

## §5. Operator → Coercion Map — Ai Kích Hoạt Cái Nào?

### 5.1 Bài Viết: The Complete Map

```
OPERATOR → COERCION MAP:
═══════════════════════════════════════════════════════════════

  🔢 toNumber — KICKED OFF BY:
  ┌──────────────────────────────────────────────────────┐
  │ ① Math operators:  * - / % **                       │
  │    → "3" * 7 = 21                                   │
  │    → LUÔN toNumber, KHÔNG CÓ ngoại lệ!              │
  │                                                      │
  │ ② Unary + operator:  +value                         │
  │    → +"3" = 3                                        │
  │    → Manual coercion tool!                           │
  │                                                      │
  │ ③ Relational operators:  < > <= >=                   │
  │    → "3" < 10 → 3 < 10 → true                       │
  │    → TRƯỚC HẾT toNumber, nếu cả 2 string → so sánh  │
  │      theo Unicode position                           │
  │                                                      │
  │ ④ Loose equality ==:  (nhiều hướng!)                 │
  │    → "" == 0 → toNumber("") = 0 → true              │
  │    → PHỨC TẠP nhất! Nhiều variations!                │
  └──────────────────────────────────────────────────────┘

  📝 toString — KICKED OFF BY:
  ┌──────────────────────────────────────────────────────┐
  │ ① + operator (khi 1 bên là string):                  │
  │    → 21 + "10" = "2110"                              │
  │    → String "thắng"!                                  │
  │                                                      │
  │ ② DOM boundary (browser → JS):                       │
  │    → document.getElementById("q").value = STRING!    │
  │    → Automatic — không cần operator!                 │
  │                                                      │
  │ ③ Template literals:  `${value}`                     │
  │    → `${42}` = "42"                                  │
  │    → Manual coercion tool!                           │
  │                                                      │
  │ ④ String() function call:                            │
  │    → String(42) = "42"                               │
  │    → Manual coercion tool!                           │
  │                                                      │
  │ ⑤ .toString() method:                                │
  │    → (42).toString() = "42"                          │
  └──────────────────────────────────────────────────────┘

  ✅ toBoolean — KICKED OFF BY:
  ┌──────────────────────────────────────────────────────┐
  │ ① Conditionals:  if (value), ternary (? :)           │
  │    → if (0) → false                                  │
  │    → if ("abc") → true                               │
  │                                                      │
  │ ② Logical operators:  && || !                        │
  │    → !0 = true, !!0 = false                          │
  │    → "abc" && 42 = 42 (truthy → return right)       │
  │                                                      │
  │ ③ Boolean() function call:                           │
  │    → Boolean(0) = false                              │
  │    → Manual coercion tool!                           │
  └──────────────────────────────────────────────────────┘

  Will: "Our math to number, our unary to number, our
  plus to string unless both are numbers. Our relational
  first to number, our loose equality just goes any
  direction. Our conditionals to boolean."
```

## §6. NaN = Số Không Phải Số — ISO Standard!

### 6.1 Bài Viết: Không Phải JS Quirk!

> Will: _"Some of these may sound like JavaScript quirks — they're actually very typical of MANY programming languages. These are features of the International Standards Organisation for programming languages."_

```
NaN — KHÔNG PHẢI BUG, MÀ LÀ TIÊU CHUẨN!
═══════════════════════════════════════════════════════════════

  typeof NaN → "number"  ← Vô lý? KHÔNG!

  Will: "You want to have a thing that IS a number
  (type number) but it's NOT actually of numeric value.
  That's a USEFUL TOOL."

  TẠI SAO NaN type = "number"?
  ┌──────────────────────────────────────────────────┐
  │ IEEE 754 standard (1985):                        │
  │ → Defines: integers, floats, Infinity, -Infinity│
  │ → VÀ: NaN (Not a Number)                       │
  │ → NaN = giá trị TRONG tập số IEEE 754!          │
  │ → = KẾT QUẢ của phép toán KHÔNG HỢP LỆ!        │
  │ → 0/0 = NaN, Math.sqrt(-1) = NaN               │
  │ → NaN LÀ 1 giá trị trong number system!         │
  └──────────────────────────────────────────────────┘

  NaN PROPERTIES:
  → NaN === NaN → false ← UNIQUE! (IEEE 754 standard!)
  → NaN !== NaN → true
  → isNaN(NaN) → true ← legacy, coerces!
  → Number.isNaN(NaN) → true ← modern, NO coerce!
  → isNaN("abc") → true ⚠️ (coerce "abc"→NaN→true)
  → Number.isNaN("abc") → false ✅ (no coerce)

  NHIỀU NGÔN NGỮ CÓ NaN:
  → Python: float('nan'), math.nan
  → Java: Double.NaN, Float.NaN
  → C/C++: NAN (math.h)
  → Rust: f64::NAN
  → → KHÔNG PHẢI "JavaScript quirk"!
```

## §9. Tự Implement: Full Coercion Engine Từ Đầu

```javascript
// FULL COERCION ENGINE — 3 Pipelines

const CoercionEngine = {
  // ═══════════════════════════════════════
  // PIPELINE 1: toNumber
  // ═══════════════════════════════════════
  toNumber(value) {
    if (typeof value === "number") return value;
    if (typeof value === "bigint") throw TypeError("Cannot convert BigInt");
    if (typeof value === "symbol") throw TypeError("Cannot convert Symbol");
    if (typeof value === "boolean") return value ? 1 : 0;
    if (value === null) return 0;
    if (value === undefined) return NaN;
    if (typeof value === "string") {
      const trimmed = value.trim();
      if (trimmed === "") return 0;
      return +trimmed; // let engine parse
    }
    if (typeof value === "object") {
      return this.toNumber(this.toPrimitive(value, "number"));
    }
    return NaN;
  },

  // ═══════════════════════════════════════
  // PIPELINE 2: toString
  // ═══════════════════════════════════════
  toString(value) {
    if (typeof value === "string") return value;
    if (typeof value === "number") return String(value);
    if (typeof value === "boolean") return value ? "true" : "false";
    if (value === null) return "null";
    if (value === undefined) return "undefined";
    if (typeof value === "symbol") return value.toString();
    if (Array.isArray(value)) return value.join(",");
    if (typeof value === "object") return "[object Object]";
    return String(value);
  },

  // ═══════════════════════════════════════
  // PIPELINE 3: toBoolean
  // ═══════════════════════════════════════
  toBoolean(value) {
    // 7 falsy values:
    if (value === false) return false;
    if (value === 0 || value === -0) return false;
    if (value === 0n) return false;
    if (value === "") return false;
    if (value === null) return false;
    if (value === undefined) return false;
    if (typeof value === "number" && Number.isNaN(value)) return false;
    // Everything else = truthy:
    return true;
  },

  // ═══════════════════════════════════════
  // HELPER: toPrimitive (for objects)
  // ═══════════════════════════════════════
  toPrimitive(obj, hint = "default") {
    if (hint === "number") {
      const val = obj.valueOf();
      if (typeof val !== "object") return val;
      return obj.toString();
    }
    const str = obj.toString();
    if (typeof str !== "object") return str;
    return obj.valueOf();
  },
};

// Test toNumber:
console.log("=== toNumber ===");
console.log(CoercionEngine.toNumber("3")); // 3
console.log(CoercionEngine.toNumber("")); // 0
console.log(CoercionEngine.toNumber(true)); // 1
console.log(CoercionEngine.toNumber(null)); // 0
console.log(CoercionEngine.toNumber(undefined)); // NaN

// Test toString:

console.log(CoercionEngine.toString(42)); // "42"

console.log(CoercionEngine.toString(null)); // "null"
console.log(CoercionEngine.toString([1, 2, 3])); // "1,2,3"
console.log(CoercionEngine.toString({})); // "[object Object]"
```

```javascript
// OPERATOR DISPATCHER — simulate operator coercion behavior

function operatorDispatch(left, op, right) {
  const CE = CoercionEngine;

  switch (op) {
    // Math operators → ALWAYS toNumber
    case "*":
      return CE.toNumber(left) * CE.toNumber(right);
    case "-":
      return CE.toNumber(left) - CE.toNumber(right);
    case "/":
      return CE.toNumber(left) / CE.toNumber(right);
    case "%":
      return CE.toNumber(left) % CE.toNumber(right);
    case "**":
      return CE.toNumber(left) ** CE.toNumber(right);

    // + operator → STRING if either is string!
    case "+": {
      const lp = typeof left === "object" ? CE.toPrimitive(left) : left;
      const rp = typeof right === "object" ? CE.toPrimitive(right) : right;
      if (typeof lp === "string" || typeof rp === "string") {
        return CE.toString(lp) + CE.toString(rp); // CONCAT!
      }
      return CE.toNumber(lp) + CE.toNumber(rp); // MATH!
    }

    // Relational → toNumber
    case "<":
      return CE.toNumber(left) < CE.toNumber(right);
    case ">":
      return CE.toNumber(left) > CE.toNumber(right);
    case "<=":
      return CE.toNumber(left) <= CE.toNumber(right);
    case ">=":
      return CE.toNumber(left) >= CE.toNumber(right);

    default:
      throw new Error(`Unknown operator: ${op}`);
  }
}

// Test:
```

## §10. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 10.1 Pattern ①: 5 Whys — 3 Coercion Pipelines

```
5 WHYS: TẠI SAO CHỈ CÓ 3 PIPELINES?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao chỉ toNumber, toString, toBoolean?
  └→ Vì JS CHỈ CÓ 7 primitive types!
     number, string, boolean = 3 types "active" nhất!
     null, undefined = "empty" types — targets, not sources!

  WHY ②: Tại sao operators tự coerce?
  └→ Vì DOM → JS boundary! User input = STRING!
     Manual convert MỌI NƠI = verbose!
     Auto coerce = "less strenuous" (Will)

  WHY ③: Tại sao + khác * (dual role)?
  └→ Vì + kế thừa từ Java/C++ (concat + math)!
     * chỉ có 1 meaning → unambiguous!
     + = legacy design decision!

  WHY ④: Tại sao == coerce nhưng === không?
  └→ == = original (1995), === = added later (1999)!
     Community nhận ra == quá phức tạp!
     === = "fixed version" — no coercion!

  WHY ⑤: Tại sao không bỏ == đi?
  └→ BACKWARD COMPATIBILITY!
     Xoá == = break hàng triệu websites!
     → Thêm === thay vì sửa == = JS philosophy!
     ⭐ "Don't break the web!"
```

```javascript
function coercionQuiz() {
  const questions = [
    { expr: '"3" * 7', answer: 21, rule: "* → toNumber" },
    { expr: '"3" + 7', answer: "37", rule: "+ → toString" },
    { expr: '21 + "10"', answer: "2110", rule: "+ → toString" },

    { expr: '"" === 0', answer: false, rule: "=== → no coerce, diff type" },
    { expr: 'Boolean("")', answer: false, rule: "toBoolean → falsy" },

    { expr: "Boolean([])", answer: true, rule: "toBoolean → object = truthy!" },
    { expr: "null == undefined", answer: true, rule: "== → special case!" },
    { expr: "null === undefined", answer: false, rule: "=== → diff type" },
    { expr: '+"" ', answer: 0, rule: "unary + → toNumber" },
    { expr: "typeof NaN", answer: "number", rule: "IEEE 754 standard!" },
  ];

  let score = 0;

    const actual = eval(q.expr);

        `${correct ? "" : ` (expected: ${q.answer})`}` +
        ` | ${q.rule}`,
    );
  }
```
