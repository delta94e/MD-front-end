## §1. 4 Versions Của multiplyBy2 — Step By Step

### 1.1 Bài Viết: Từ Verbose Đến Tối Giản

Will Sentance dẫn dắt học viên qua **4 phiên bản** của cùng 1 function. Mỗi version ngắn hơn, nhưng **HÀNH VI HOÀN TOÀN GIỐNG NHAU**.

> Will: _"All of which are legitimate, to the version known as the arrow function style, that turns otherwise syntactically heavy code into something very legible, very readable."_

```
4 VERSIONS — TỪ VERBOSE → TỐI GIẢN:
═══════════════════════════════════════════════════════════════

  VERSION 1: Function Declaration (đầy đủ nhất)
  ┌──────────────────────────────────────────────────────┐
  │ function multiplyBy2(input) {                        │
  │   return input * 2;                                  │
  │ }                                                    │
  │                                                      │
  │ ✅ Keyword "function" = biết NGAY đây là hàm        │
  │ ✅ Hoisted = gọi TRƯỚC khi khai báo được            │
  │ ✅ Will dùng cho TEACHING = rõ ràng nhất             │
  └──────────────────────────────────────────────────────┘

  VERSION 2: Function Expression (gán vào const)
  ┌──────────────────────────────────────────────────────┐
  │ const multiplyBy2 = function(input) {                │
  │   return input * 2;                                  │
  │ };                                                   │
  │                                                      │
  │ → Bỏ TÊN sau "function" (vì tên = const label)      │
  │ → Gán function vào BIẾN (first-class object!)        │
  │ → Will: "the ever more popular style that does        │
  │   involve assigning to a label"                      │
  └──────────────────────────────────────────────────────┘

  VERSION 3: Arrow Function (đầy đủ body)
  ┌──────────────────────────────────────────────────────┐
  │ const multiplyBy2 = (input) => {                     │
  │   return input * 2;                                  │
  │ };                                                   │
  │                                                      │
  │ → Bỏ keyword "function"                              │
  │ → Thêm => (fat arrow) giữa params và body           │
  │ → Vẫn cần {} và return vì có body block              │
  └──────────────────────────────────────────────────────┘

  VERSION 4: Arrow Function (tối giản — implicit return)
  ┌──────────────────────────────────────────────────────┐
  │ const multiplyBy2 = input => input * 2;              │
  │                                                      │
  │ → Bỏ {} (curly braces)                               │
  │ → Bỏ "return" (implicit return!)                     │
  │ → Bỏ () quanh parameter (1 param only!)              │
  │ → 1 DÒNG thay vì 3 dòng!                            │
  │                                                      │
  │ Will: "If we only have one parameter, we can reduce  │
  │ our function definition to just our parameter name    │
  │ and then the return value."                          │
  └──────────────────────────────────────────────────────┘
```

### 1.2 So Sánh Trực Quan

```
4 VERSIONS CẠnh NHAU:
═══════════════════════════════════════════════════════════════

  V1: function multiplyBy2(input) { return input * 2; }
      ^^^^^^^^                      ^^^^^^
      keyword                       explicit return

  V2: const multiplyBy2 = function(input) { return input * 2; };
      ^^^^^                ^^^^^^^^          ^^^^^^
      const label          keyword           explicit return

  V3: const multiplyBy2 = (input) => { return input * 2; };
                           ^^^^^^^^    ^^^^^^
                           arrow       explicit return

  V4: const multiplyBy2 = input => input * 2;
                           ^^^^^^^^^^^^^^^^
                           TẤT CẢ trong 1 expression!

  Bảng so sánh:
  ┌──────────┬──────────┬──────────┬──────────┬──────────┐
  │          │ V1       │ V2       │ V3       │ V4       │
  ├──────────┼──────────┼──────────┼──────────┼──────────┤
  │ keyword  │ function │ function │ =>       │ =>       │
  │ {}       │ ✅       │ ✅       │ ✅       │ ❌       │
  │ return   │ ✅       │ ✅       │ ✅       │ ❌ auto  │
  │ ()       │ ✅       │ ✅       │ ✅       │ ❌ 1param│
  │ hoisted  │ ✅       │ ❌       │ ❌       │ ❌       │
  │ this     │ dynamic  │ dynamic  │ lexical  │ lexical  │
  │ dòng     │ 3        │ 3        │ 3        │ 1        │
  └──────────┴──────────┴──────────┴──────────┴──────────┘

  → V1-V2 = function keyword → DYNAMIC this
  → V3-V4 = arrow function  → LEXICAL this (khác biệt!)
  → Behavior chạy code giống 100%!
```

```
TRACE — ARROW FUNCTION V4:
═══════════════════════════════════════════════════════════════
```

## §3. Anonymous Functions — Truyền Code Trực Tiếp, Không Tên

### 3.1 Bài Viết: Bỏ Luôn Tên Function

Bước tiếp theo: thay vì **khai báo** function riêng rồi truyền vào, ta **truyền code trực tiếp** khi gọi HOF. Function không có tên = **anonymous function**.

> Will: _"The function we passed in without a name… is known as an anonymous function. It ain't got no name."_

```
NAMED vs ANONYMOUS — 3 CẤP ĐỘ:
═══════════════════════════════════════════════════════════════

  CẤP 1: Named + Truyền label
  ┌──────────────────────────────────────────────────────┐
  │ function multiplyBy2(input) { return input * 2; }    │
  │                                                      │
  │ copyArrayAndManipulate([1,2,3], multiplyBy2);        │
  │                                 ^^^^^^^^^^^^         │
  │                              truyền LABEL            │
  │ → Function có tên riêng trong global memory          │
  │ → Tái sử dụng được ở nhiều nơi                      │
  └──────────────────────────────────────────────────────┘

  CẤP 2: Arrow named + Truyền label
  ┌──────────────────────────────────────────────────────┐
  │ const multiplyBy2 = input => input * 2;              │
  │                                                      │
  │ copyArrayAndManipulate([1,2,3], multiplyBy2);        │
  │ → Gọn hơn, vẫn có tên, vẫn reusable                │
  └──────────────────────────────────────────────────────┘

  CẤP 3: Anonymous — Truyền code TRỰC TIẾP!
  ┌──────────────────────────────────────────────────────┐
  │ copyArrayAndManipulate([1,2,3], input => input * 2); │
  │                                 ^^^^^^^^^^^^^^^^^    │
  │                              ANONYMOUS FUNCTION!     │
  │ → KHÔNG có tên                                       │
  │ → KHÔNG lưu riêng trong global memory                │
  │ → Chỉ tồn tại KHI truyền vào HOF                    │
  │ → Will: "We're just not giving a name at all.        │
  │   We're just passing it straight in."                │
  └──────────────────────────────────────────────────────┘
```

### 3.2 Bài Viết: Đừng Nghĩ Nó "Chạy Tại Chỗ"!

Will cảnh báo hiểu sai phổ biến:

> _"My goodness, does that look a little bit like we're doing something there in the line? We ain't. We're passing it in... It is creating an execution context when instructions runs."_

```
HIỂU SAI PHỔ BIẾN VỀ ANONYMOUS FUNCTIONS:
═══════════════════════════════════════════════════════════════

  Code:
  copyArrayAndManipulate([1,2,3], input => input * 2);

  ❌ HIỂU SAI: "input => input * 2 CHẠY ngay tại dòng này"
  → KHÔNG! Nó chỉ TRUYỀN VÀO như argument!
  → Function code KHÔNG chạy cho đến khi gặp ()!

  ✅ HIỂU ĐÚNG:
  ┌──────────────────────────────────────────────────────┐
  │ Step 1: Evaluate arguments:                          │
  │   arg1 = [1, 2, 3]                                  │
  │   arg2 = ƒ(input) { return input * 2 }              │
  │          ↑ FUNCTION OBJECT, CHƯA CHẠY!              │
  │                                                      │
  │ Step 2: Gọi copyArrayAndManipulate:                  │
  │   array = [1, 2, 3]                                  │
  │   instructions = ƒ(input) { return input * 2 }      │
  │   → "instructions" thay thế anonymous code           │
  │                                                      │
  │ Step 3: BÊN TRONG for loop:                          │
  │   instructions(array[0]) → tạo EC!                   │
  │   LÚC NÀY mới chạy input * 2!                      │
  └──────────────────────────────────────────────────────┘

  → Anonymous function = CÙNG cơ chế như named function!
  → Chỉ khác: KHÔNG có label trong global memory!
```

## §4. Built-in map() vs Hand-built copyArrayAndManipulate

### 4.1 Bài Viết: map() = copyArrayAndManipulate Built-in!

Will đặt câu hỏi: _"What's our copyArrayAndManipulate called in practice?"_ → Chris trả lời: **"Map."**

`Array.prototype.map` = phiên bản BUILT-IN của copyArrayAndManipulate. JavaScript cung cấp sẵn vì pattern quá quan trọng!

```
HAND-BUILT vs BUILT-IN:
═══════════════════════════════════════════════════════════════

  HAND-BUILT (đã viết):
  ┌──────────────────────────────────────────────────────┐
  │ function map(array, instructions) {                  │
  │   const output = [];                                 │
  │   for (let i = 0; i < array.length; i++) {           │
  │     output.push(instructions(array[i]));             │
  │   }                                                  │
  │   return output;                                     │
  │ }                                                    │
  │                                                      │
  │ const result = map([1,2,3], input => input * 2);     │
  │ // [2, 4, 6]                                         │
  └──────────────────────────────────────────────────────┘

  BUILT-IN (sẵn có trên mọi array):
  ┌──────────────────────────────────────────────────────┐
  │ const result = [1,2,3].map(input => input * 2);      │
  │                        ^^^                           │
  │                    dot notation!                     │
  │                                                      │
  │ → [1,2,3] = data (tự động là arg đầu tiên)          │
  │ → .map() = HOF (trên prototype chain)                │
  │ → input => input * 2 = callback                      │
  │                                                      │
  │ Will: "These TWO are doing exactly the SAME THING.   │
  │ It's just that through the prototype chain, the map  │
  │ function is available on the array via dot notation." │
  └──────────────────────────────────────────────────────┘

  SO SÁNH TRỰC TIẾP:
  ─────────────────────────────────────────────────
  // Hand-built: truyền array làm argument
  map([1,2,3], x => x * 2);

  // Built-in: array TRƯỚC dấu chấm, callback SAU
  [1,2,3].map(x => x * 2);

  → KẾT QUẢ GIỐNG HỆT: [2, 4, 6]
  → Cơ chế bên trong GIỐNG HỆT!
```

### 4.2 Readability — Tại Sao Built-in Gọn Hơn

```
READABILITY EVOLUTION:
═══════════════════════════════════════════════════════════════

  IMPERATIVE (ban đầu):
  const output = [];
  for (let i = 0; i < [1,2,3].length; i++) {
    output.push([1,2,3][i] * 2);
  }
  // → 4 dòng, phải hiểu for loop

  HAND-BUILT HOF:
  const result = copyArrayAndManipulate([1,2,3], multiplyBy2);
  // → 1 dòng, tên dài nhưng rõ nghĩa

  RENAMED HOF:
  const result = map([1,2,3], input => input * 2);
  // → 1 dòng, gọn hơn

  BUILT-IN:
  const result = [1,2,3].map(input => input * 2);
  // → 1 dòng, NGẮN NHẤT, đọc tự nhiên!

  Will: "Look at the READABILITY of that. Map 1, 2, 3
  via these changes to [2, 4, 6]. NICE AND READABLE."
```

## §7. Arrow Function & `this` — Preview

Will teaser một khác biệt QUAN TRỌNG giữa arrow function và function keyword:

> _"Functions defined using this arrow syntax do have a distinctive behavior. Their `this` keyword will be different, and in a very useful way."_

```
ARROW vs FUNCTION KEYWORD — this BEHAVIOR (Preview):
═══════════════════════════════════════════════════════════════

  FUNCTION KEYWORD: this = DYNAMIC (ai gọi?)
  ┌──────────────────────────────────────────────────────┐
  │ const obj = {                                        │
  │   name: "Jun",                                       │
  │   greet: function() {                                │
  │     console.log(this.name); // "Jun" ← obj gọi      │
  │   }                                                  │
  │ };                                                   │
  │ obj.greet(); // this = obj ✅                        │
  │                                                      │
  │ const fn = obj.greet;                                │
  │ fn(); // this = window ❌ (mất context!)             │
  └──────────────────────────────────────────────────────┘

  ARROW FUNCTION: this = LEXICAL (khai báo ở đâu?)
  ┌──────────────────────────────────────────────────────┐
  │ const obj = {                                        │
  │   name: "Jun",                                       │
  │   greet: () => {                                     │
  │     console.log(this.name); // undefined ← this      │
  │   }                 // = outer scope, KHÔNG phải obj!│
  │ };                                                   │
  │ obj.greet(); // this = window (lexical!) ❌           │
  └──────────────────────────────────────────────────────┘

  NHƯNG — Arrow function HỮU ÍCH khi NESTED:
  ┌──────────────────────────────────────────────────────┐
  │ const obj = {                                        │
  │   name: "Jun",                                       │
  │   friends: ["Alice", "Bob"],                         │
  │   greetAll: function() {  // ← function keyword      │
  │     this.friends.map(friend => {  // ← arrow function│
  │       console.log(this.name + " hi " + friend);      │
  │       // ↑ this = obj ✅ vì arrow inherits this!     │
  │     });                                              │
  │   }                                                  │
  │ };                                                   │
  │ // Nếu dùng function keyword bên trong map():        │
  │ // this = undefined! Phải dùng .bind() hoặc self!   │
  └──────────────────────────────────────────────────────┘

  Will: "Functions defined with this arrow syntax...
  their this keyword will be different, and in a VERY
  USEFUL way when calling functions inside other functions."
  → Sẽ học chi tiết ở phần OOP/this!
```

## §8. Tự Implement: 4 Versions + Built-in Methods

```javascript
// CHỨNG MINH: 4 VERSIONS = CÙNG KẾT QUẢ

// Version 1: Function Declaration
function multiplyBy2_v1(input) {
  return input * 2;
}

// Version 2: Function Expression
const multiplyBy2_v2 = function (input) {
  return input * 2;
};

// Version 3: Arrow (full body)
const multiplyBy2_v3 = (input) => {
  return input * 2;
};

// Version 4: Arrow (implicit return)
const multiplyBy2_v4 = (input) => input * 2;

// TEST — TẤT CẢ return 6:
console.log(multiplyBy2_v1(3)); // 6
console.log(multiplyBy2_v2(3)); // 6
console.log(multiplyBy2_v3(3)); // 6
console.log(multiplyBy2_v4(3)); // 6
// → CHÍNH XÁC cùng kết quả! ✅
```

```javascript
// ANONYMOUS FUNCTIONS — 3 CÁCH TRUYỀN CALLBACK:

function map(array, instructions) {
  const output = [];
  for (let i = 0; i < array.length; i++) {
    output.push(instructions(array[i]));
  }
  return output;
}

// Cách 1: Named function (khai báo riêng)
function double(x) {
  return x * 2;
}
map([1, 2, 3], double); // [2, 4, 6]

// Cách 2: Arrow named (khai báo riêng)
const triple = (x) => x * 3;
map([1, 2, 3], triple); // [3, 6, 9]

// Cách 3: Anonymous inline (trực tiếp!)
map([1, 2, 3], (x) => x * 2); // [2, 4, 6]
map([1, 2, 3], (x) => x * 3); // [3, 6, 9]
map([1, 2, 3], (x) => x ** 2); // [1, 4, 9]
```

```javascript
// TỰ BUILD: SO SÁNH HAND-BUILT vs BUILT-IN ARRAY METHODS

// Hand-built versions:
function myMap(arr, cb) {
  const out = [];
  for (let i = 0; i < arr.length; i++) out.push(cb(arr[i], i, arr));
  return out;
}

function myFilter(arr, cb) {
  const out = [];
  for (let i = 0; i < arr.length; i++) {
    if (cb(arr[i], i, arr)) out.push(arr[i]);
  }
  return out;
}

function myReduce(arr, cb, init) {
  let acc = init;
  for (let i = 0; i < arr.length; i++) {
    acc = cb(acc, arr[i], i, arr);
  }
  return acc;
}

// Test: hand-built vs built-in — SAME RESULTS!
const data = [1, 2, 3, 4, 5];

// MAP:
console.log(myMap(data, (x) => x * 2)); // [2, 4, 6, 8, 10]
console.log(data.map((x) => x * 2)); // [2, 4, 6, 8, 10]

// FILTER:
console.log(myFilter(data, (x) => x > 3)); // [4, 5]
console.log(data.filter((x) => x > 3)); // [4, 5]

// REDUCE:
console.log(myReduce(data, (a, x) => a + x, 0)); // 15
console.log(data.reduce((a, x) => a + x, 0)); // 15

// CHAINING (built-in chỉ):
const result = data
  .filter((x) => x % 2 !== 0) // [1, 3, 5]
  .map((x) => x * 10) // [10, 30, 50]
  .reduce((a, x) => a + x, 0); // 90

console.log(result); // 90
// → Declarative chain! Đọc như tiếng Anh! ✅
```

## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Arrow Functions

```
5 WHYS: TẠI SAO CẦN ARROW FUNCTIONS?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao ES6 thêm arrow functions?
  └→ Vì function keyword quá dài dòng.
     Khi truyền inline callback, cần syntax GỌN hơn.

  WHY ②: Tại sao gọn hơn lại quan trọng?
  └→ Vì modern JS dùng HÀNG TRĂM callbacks mỗi file.
     Chaining: .filter(...).map(...).reduce(...)
     Mỗi callback = 1 arrow function inline!

  WHY ③: Tại sao không chỉ rút gọn function keyword?
  └→ Vì arrow functions còn sửa BUG THIẾT KẾ:
     this keyword trong inner function = undefined!
     Arrow function: this = lexical → GIẢI QUYẾT!

  WHY ④: Tại sao this cần lexical binding?
  └→ Vì trước ES6, khi gọi function bên trong method:
     this MẤT context → phải dùng var self = this hack.
     Arrow function kế thừa this từ outer scope tự động.

  WHY ⑤: Tại sao không BỎ HẲN function keyword?
  └→ Vì function keyword vẫn cần cho:
     - Hoisting (gọi trước khi khai báo)
     - Constructor functions (new keyword)
     - Generator functions (function*)
     - Khi CẦN dynamic this (methods trên object)
     ⭐ Arrow KHÔNG thay thế function, chúng BỔ SUNG!
```

### 9.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — ARROW FUNCTION BÊN DƯỚI:
═══════════════════════════════════════════════════════════════

  Arrow function trong V8:
  ┌──────────────────────────────────────────────────────┐
  │ const double = x => x * 2;                           │
  │                                                      │
  │ V8 tạo Function Object:                              │
  │ {                                                    │
  │   [[Call]]:       ✅ (có thể gọi bằng ())           │
  │   [[Construct]]:  ❌ (KHÔNG có → không dùng new!)    │
  │   [[ThisMode]]:   "lexical" (← KHÁC function keyword)│
  │   [[Scope]]:      outer lexical environment           │
  │   name:           "double",                          │
  │   length:         1,                                 │
  │   prototype:      ❌ (KHÔNG có → tiết kiệm memory!) │
  │ }                                                    │
  │                                                      │
  │ So sánh với function keyword:                        │
  │ {                                                    │
  │   [[Call]]:       ✅                                 │
  │   [[Construct]]:  ✅ (có thể dùng new!)             │
  │   [[ThisMode]]:   "global" hoặc "strict"             │
  │   [[Scope]]:      outer lexical environment           │
  │   name:           "double",                          │
  │   length:         1,                                 │
  │   prototype:      {} ← CÓ prototype object!         │
  │ }                                                    │
  │                                                      │
  │ → Arrow function = LIGHTER (ít features hơn)!       │
  │ → Không có prototype → tiết kiệm memory!            │
  │ → Không có [[Construct]] → an toàn (không new nhầm) │
  └──────────────────────────────────────────────────────┘
```

```javascript
// TỰ BUILD: CHỨNG MINH this KHÁC NHAU

const team = {
  name: "Frontend",
  members: ["Alice", "Bob", "Charlie"],

  // ❌ Function keyword bên trong — this MẤT!
  listMembersOld: function () {
    return this.members.map(function (member) {
      // this = undefined (strict mode) hoặc window!
      return `${this.name}: ${member}`;
      // → "undefined: Alice" 😱
    });
  },

  // ✅ Arrow function bên trong — this KẾ THỪA!
  listMembersNew: function () {
    return this.members.map((member) => {
      // this = team ✅ (lexical, kế thừa từ listMembersNew)
      return `${this.name}: ${member}`;
      // → "Frontend: Alice" ✅
    });
  },

  // ❌ Arrow function cho method — this MẤT!
  brokenMethod: () => {
    // this = outer scope (window/module), KHÔNG phải team!
    return this.name; // undefined!
  },
};

console.log(team.listMembersNew());
// ["Frontend: Alice", "Frontend: Bob", "Frontend: Charlie"] ✅

console.log(team.brokenMethod());
// undefined ❌ — arrow function KHÔNG nên dùng cho methods!
```

### 9.6 Pattern ⑥: Lịch Sử & Tiến Hoá

```
LỊCH SỬ: FUNCTION SYNTAX EVOLUTION
═══════════════════════════════════════════════════════════════

  1995: JavaScript 1.0 — function keyword ONLY
  │     → function add(a, b) { return a + b; }
  │     → Duy nhất cách khai báo function
  │
  ↓
  1999: ES3 — Function expressions
  │     → var add = function(a, b) { return a + b; };
  │     → Gán function vào biến (first-class!)
  │
  ↓
  2006: jQuery era — Anonymous callbacks phổ biến
  │     → $.ajax({ success: function(data) { ... } })
  │     → "function" keyword viết HÀNG TRĂM lần!
  │     → Community: "quá dài dòng!"
  │
  ↓
  2009: CoffeeScript — -> syntax (inspiration for =>!)
  │     → add = (a, b) -> a + b
  │     → Ngắn gọn, lexical this → cộng đồng YÊU THÍCH
  │     → TC39: "Chúng ta cần feature này trong JS!"
  │
  ↓
  2015: ES6 — Arrow functions CHÍNH THỨC
  │     → const add = (a, b) => a + b;
  │     → Lexical this (giải quyết bug cũ!)
  │     → Implicit return (gọn hơn!)
  │     → No prototype (nhẹ hơn!)
  │
  ↓
  2020+: Modern JS — Arrow functions = DEFAULT
        → React components: const App = () => ...
        → Callbacks: arr.map(x => x * 2)
        → Function keyword: chỉ khi CẦN hoisting/this/new

  ⭐ Will dùng function keyword cho TEACHING CLARITY.
     Modern production code dùng arrow functions 80%+!
```
