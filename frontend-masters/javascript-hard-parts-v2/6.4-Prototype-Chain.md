```javascript
// CHỨNG MINH 2: Simulate prototype chain lookup!

class MyPrototypeChain {
  static create(proto) {
    const obj = {};
    // Store the hidden bond:
    obj.__hiddenLink__ = proto;
    return obj;
  }

  static lookup(obj, prop) {
    console.log(`Looking for "${prop}" on object...`);

    // Step 1: Look on the object itself!
    if (obj.hasOwnProperty(prop)) {
      console.log(`  ✅ Found on object itself!`);
      return obj[prop];
    }

    // Step 2: NOT FOUND! Don't panic!
    console.log(`  ❌ Not on object! Don't panic!`);
    console.log(`  🔗 Following hidden link...`);

    // Step 3: Follow the hidden bond!
    const proto = obj.__hiddenLink__;
    if (proto && proto.hasOwnProperty(prop)) {
      console.log(`  ✅ Found on prototype! Using it!`);
      return proto[prop];
    }

    // Step 4: Not found anywhere!
    console.log(`  ❌ Not found anywhere! undefined!`);
    return undefined;
  }
}

const store = {
  increment() {
    this.score++;
  },
  greet() {
    console.log(`Hi, ${this.name}!`);
  },
};

const user = MyPrototypeChain.create(store);
user.name = "Ari";
user.score = 3;

// Simulate lookup:
MyPrototypeChain.lookup(user, "name");
// Looking for "name" on object...
//   ✅ Found on object itself!

MyPrototypeChain.lookup(user, "increment");
// Looking for "name" on object...
//   ❌ Not on object! Don't panic!
//   🔗 Following hidden link...
//   ✅ Found on prototype! Using it!

MyPrototypeChain.lookup(user, "nonexistent");
// Looking for "nonexistent" on object...
//   ❌ Not on object! Don't panic!
//   🔗 Following hidden link...
//   ❌ Not found anywhere! undefined!
```

## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Prototype Chain

```
5 WHYS: TẠI SAO CẦN PROTOTYPE CHAIN?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao không copy methods lên mỗi object?
  └→ Vì MEMORY WASTE! 100 methods × 1M users = 100M copies!
     Will: "Completely and utterly untenable!"

  WHY ②: Tại sao store methods trong separate object?
  └→ Vì CHỈ CẦN 1 COPY! All users share ONE store!
     Chris: "One copy is enough!"

  WHY ③: Tại sao cần hidden bond (không add trực tiếp)?
  └→ Vì Object.create returns EMPTY object!
     Bond is HIDDEN — user object stays clean!
     Only OWN data on user, shared methods via chain!

  WHY ④: Tại sao JS không panic khi property not found?
  └→ Vì DESIGN INTENT! Prototype chain = lookup chain!
     Not found on self → look on prototype!
     Not found on prototype → look on prototype's prototype!
     Until null → then undefined!

  WHY ⑤: Tại sao THIS works correctly qua chain?
  └→ Vì `this` = object TO THE LEFT of the dot!
     user1.increment() → this = user1!
     Dù increment lives on prototype!
     this.score++ → changes USER1's score!
```

### 9.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — [[Prototype]] INTERNAL SLOT:
═══════════════════════════════════════════════════════════════

  ECMAScript spec — Property Lookup:

  OrdinaryGet(O, P):
  1. Let desc = O.[[GetOwnProperty]](P)
  2. If desc is not undefined → return desc.value!
  3. Let parent = O.[[GetPrototypeOf]]()   ← FOLLOW CHAIN!
  4. If parent is null → return undefined!
  5. Return parent.[[Get]](P)              ← RECURSIVE! 🔄

  → Step 3: This is the HIDDEN BOND!
  → [[Prototype]] internal slot = __proto__!
  → Object.create(proto) sets [[Prototype]] = proto!
  → The lookup is RECURSIVE until null!

  CHAIN CAN BE DEEP:
  user → userFunctionStore → Object.prototype → null
  ① look on user
  ② look on userFunctionStore
  ③ look on Object.prototype
  ④ null → return undefined!
```

### 9.3 Pattern ③: Trade-off Analysis

```
TRADE-OFFS — SOLUTION 1 vs SOLUTION 2:
═══════════════════════════════════════════════════════════════

  ┌──────────────────┬──────────────────┬───────────────────┐
  │                  │ Sol 1 (Factory)  │ Sol 2 (Prototype) │
  ├──────────────────┼──────────────────┼───────────────────┤
```

### 9.5 Pattern ⑤: Reverse Engineering

```javascript
// PROVE: The chain exists with Object.getPrototypeOf

const store = {
  increment() {
    this.score++;
  },
};
const user = Object.create(store);
user.name = "Ari";
user.score = 3;

// See the chain:
console.log(Object.getPrototypeOf(user) === store); // true!
console.log(user.__proto__ === store); // true! (deprecated but works)

// Own properties:
console.log(user.hasOwnProperty("name")); // true (on user!)
console.log(user.hasOwnProperty("increment")); // false (NOT on user!)

// But still accessible!
console.log(typeof user.increment); // "function" (found via chain!)
console.log(user.increment === store.increment); // true! (same ref!)
```
