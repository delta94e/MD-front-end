## §5. Complete OOP Journey: Solution 1 → class + Private!

```
THE COMPLETE JOURNEY — ALL SOLUTIONS:
═══════════════════════════════════════════════════════════════

  SOLUTION 1: Factory Function
  ┌──────────────────────────────────────────────────┐
  │ function createUser(name, score) {               │
  │   const user = {};                               │
  │   user.name = name;                              │
  │   user.score = score;                            │
  │   user.increment = ƒ; // ← COPIED each time!   │
  │   return user;                                   │
  │ }                                                │
  │ ✅ DRY | ❌ Memory waste (copies methods!)      │
  └──────────────────────────────────────────────────┘

  SOLUTION 2: Object.create + Manual Store
  ┌──────────────────────────────────────────────────┐
  │ const store = { increment: ƒ };                  │
  │ function createUser(name, score) {               │
  │   const user = Object.create(store); // ← link! │
  │   user.name = name;                              │
  │   user.score = score;                            │
  │   return user;                                   │
  │ }                                                │
  │ ✅ Shared methods | ❌ Verbose, not standard    │
  └──────────────────────────────────────────────────┘

  SOLUTION 3: new Keyword
  ┌──────────────────────────────────────────────────┐
  │ function UserCreator(name, score) {              │
  │   this.name = name; this.score = score;          │
  │ }                                                │
  │ UserCreator.prototype.increment = ƒ;             │
  │ const user = new UserCreator("Ari", 3);          │
  │ ✅ Automates 3 steps | ⚠️ Forget new = doom!   │
  └──────────────────────────────────────────────────┘

  SOLUTION 4: class (Syntactic Sugar)
  ┌──────────────────────────────────────────────────┐
  │ class UserCreator {                              │
  │   constructor(name, score) {                     │
  │     this.name = name; this.score = score;        │
  │   }                                              │
  │   increment() { this.score++; }                  │
  │ }                                                │
  │ ✅ All-in-one! Standard! | ⚠️ "Sugar over proto"│
  └──────────────────────────────────────────────────┘

  SOLUTION 4+: class + Fields (2022!)
  ┌──────────────────────────────────────────────────┐
  │ class UserCreator {                              │
  │   static #count = 0;      ← private static!    │
```

## §6. Tự Implement: Private Static Fields

```javascript
// CHỨNG MINH 1: Private static field — limit instances

class UserCreator {
  static #count = 0;
  static #maxUsers = 2;

  constructor(name, score) {
    if (UserCreator.#count >= UserCreator.#maxUsers) {
      throw new Error(`Maximum of ${UserCreator.#maxUsers} users reached!`);
    }
    this.name = name;
    this.score = score;
    UserCreator.#count++;
  }

  increment() {
    this.score++;
  }

  static getCount() {
    return UserCreator.#count; // ✅ accessible from class method!
  }
}

const user1 = new UserCreator("Ari", 3);
console.log(UserCreator.getCount()); // 1

const user2 = new UserCreator("J", 5);
console.log(UserCreator.getCount()); // 2

try {
  const user3 = new UserCreator("Tam", 9);
} catch (e) {
  console.log(e.message); // "Maximum of 2 users reached!" ✅
}

// Cannot access from outside:
// console.log(UserCreator.#count); // SyntaxError! ❌
```

```javascript
// CHỨNG MINH 2: static #count is NOT reset on each call

class Counter {
  static #total = 0;

  constructor() {
    Counter.#total++;
    this.id = Counter.#total;
  }

  static getTotal() {
    return Counter.#total;
  }
}

// Chris's question answered:
// "Does it reset to 0 each time?"
// NO! static #total is on the CLASS, not recreated!

const a = new Counter(); // #total: 0 → 1
const b = new Counter(); // #total: 1 → 2
const c = new Counter(); // #total: 2 → 3

console.log(a.id); // 1
console.log(b.id); // 2
console.log(c.id); // 3
console.log(Counter.getTotal()); // 3

// The class declaration only runs ONCE!
// constructor runs each time but DOES NOT reset static!
```

```javascript
// CHỨNG MINH 3: All 4 types of fields

class UserCreator {
  // 1. PUBLIC INSTANCE FIELD (on each instance):
  loggedIn = false;

  // 2. PRIVATE INSTANCE FIELD (on each instance, hidden):
  #score;

  // 3. PUBLIC STATIC FIELD (on class, visible):
  static version = "2.0";

  // 4. PRIVATE STATIC FIELD (on class, hidden):
  static #instanceCount = 0;

  constructor(name, score) {
    this.name = name;
    this.#score = score;
    UserCreator.#instanceCount++;
  }

  // Regular method (→ .prototype):
  increment() {
    this.#score++;
  }
  getScore() {
    return this.#score;
  }

  // Static method (→ class object):
  static getInstanceCount() {
    return UserCreator.#instanceCount;
  }
}

const u = new UserCreator("Ari", 3);

// Public instance:
console.log(u.loggedIn); // false ✅
// Private instance:
console.log(u.getScore()); // 3 ✅
// console.log(u.#score);              // SyntaxError! ❌

// Public static:
console.log(UserCreator.version); // "2.0" ✅
// Private static:
console.log(UserCreator.getInstanceCount()); // 1 ✅
// console.log(UserCreator.#instanceCount);  // SyntaxError! ❌
```

## §7. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 7.1 Pattern ①: 5 Whys — Private Static Fields

```
5 WHYS: TẠI SAO PRIVATE STATIC?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao cần shared data trên class?
  └→ Vì cần track data ACROSS all instances!
     Count users! Global config! Shared state!
     Thuộc về CLASS, không thuộc instance nào!

  WHY ②: Tại sao shared data phải private?
  └→ Vì Chris: "I can OVERWRITE prototype!"
     Nếu public: userCreator.count = 0; ← RESET! 💀
     Private: không ai chạm được từ ngoài!

  WHY ③: Tại sao constructor CAN access?
  └→ Vì constructor DEFINED inside the class!
     Same class = same private scope!
     "Only available INSIDE of stuff associated
     with userCreator!"

  WHY ④: Tại sao KHÔNG reset mỗi lần gọi?
  └→ Chris's question! Vì static = on CLASS!
     Class definition runs ONCE!
     Constructor runs EACH TIME but doesn't re-declare!
     "We're calling the FUNCTION BIT, not the overall class!"

  WHY ⑤: Tại sao Will says "genuinely new"?
  └→ Vì BEFORE 2022: no way to truly hide data!
     Closures = workaround! WeakMap = hacky!
     # = enforced by parser! Real privacy!
```

### 7.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — 4 TYPES OF CLASS MEMBERS:

  ┌─────────────────┬────────────┬──────────────┐
  │                 │ Instance   │ Static       │
  ├─────────────────┼────────────┼──────────────┤
  │ Public          │ x = 0     │ static x = 0 │
  │                 │ → on each │ → on class   │

  │                 │ → hidden  │ → hidden on  │

  │                 │ → [[Priv. │ → [[Priv.    │
  │                 │   Elem.]] │   Elem.]]    │
  └─────────────────┴────────────┴──────────────┘

  + constructor → function bit
  + methods → .prototype (shared!)
  + static methods → object bit directly

  ALL built on PROTOTYPAL foundation!
```
