## §1. Code Phân Tích — outer + add1

```javascript
function outer() {
```

## §2. Full Trace — EC Lồng EC (3 Tầng Call Stack!)

### 2.1 Bước 1: Khai Báo outer

```
BƯỚC 1: KHAI BÁO outer

  ┌─────────────────────────────────┐    CALL STACK:

  │                                 │    │ Global()   │
  │ outer: ƒ                        │    └────────────┘
  │   (code: let counter = 0;       │
  │    function add1()...           │
  │    add1();)                     │
  │                                 │
  └─────────────────────────────────┘

  Will ← Matt:
  "What are we doing in line one?"
  "In the global execution context, we declare a function
   called outer."

  ⚠️ "Do we do any of the code inside yet?"
  → "No." → Chỉ LƯU, chưa chạy!
```

### 2.2 Bước 2: Gọi outer() — Tạo EC Mới

```
BƯỚC 2: outer() — TẠO EC!
═══════════════════════════════════════════════════════════════

  ┌─────────────────────────────────┐    CALL STACK:
  │ GLOBAL MEMORY:                  │    ┌────────────┐
  │                                 │    │ outer()    │
  │ outer: ƒ                        │    ├────────────┤
  │                                 │    │ Global()   │
  └─────────────────────────────────┘    └────────────┘

  ┌─────────────────────────────────┐
  │ EC: outer                       │
  │                                 │
  │ LOCAL MEMORY:                   │
  │   counter: 0                    │ ← let counter = 0
  │   add1: ƒ() {                   │ ← function add1()
  │     counter++;                  │
  │   }                             │
  │                                 │
  └─────────────────────────────────┘

  Matt: "We create counter and assign a value of zero.

  Will: "We don't go inside add1 — we just save its code."
```

### 2.3 Bước 3: Gọi add1() — EC TRONG EC! (3 tầng!)

```
BƯỚC 3: add1() — EC TRONG EC TRONG EC!
═══════════════════════════════════════════════════════════════

  CALL STACK = 3 TẦNG:
  ┌────────────────────┐
  │ add1()             │ ← TOP! Thread ở đây!
  ├────────────────────┤
  │ outer()            │ ← Đang chờ add1 xong
  ├────────────────────┤
  │ Global()           │ ← Đang chờ outer xong
  └────────────────────┘

  ┌─────────────────────────────────────────────┐
  │ EC: outer                                   │
  │ LOCAL MEMORY:                               │
  │   counter: 0                                │
  │   add1: ƒ                                   │
  │                                             │
  │   ┌──────────────────────────────────────┐  │
  │   │ EC: add1                             │  │
  │   │ LOCAL MEMORY:                        │  │
  │   │   (trống! không có biến local nào!)  │  │
  │   │                                      │  │
  │   │ THREAD:                              │  │
  │   │   counter++                          │  │
  │   │   → Tìm "counter" ở local... ❌     │  │
  │   │   → Tìm "counter" ở outer... ✅     │  │
  │   │   → counter = 0 + 1 = 1!            │  │
  │   └──────────────────────────────────────┘  │
  │                                             │
  │   counter: 0 → 1  ← BỊ SỬA!               │
  └─────────────────────────────────────────────┘

  Matt: "counter++ → find counter, add one to it.
  We check first in add1 — doesn't exist."
  Will: "Do we panic? — No!"
  Matt: "JavaScript looks next in outer, finds counter,
  adds one."
```

### 2.4 Bước 4: Pop add1 → Pop outer → Done!

```
BƯỚC 4: CLEAN UP — POP POP!
═══════════════════════════════════════════════════════════════

  ① add1 xong → POP:
  ┌────────────────────┐
  │ ╳ add1()           │ ← XOÁ!
  ├────────────────────┤
  │ outer()            │ ← Thread quay lại đây
  ├────────────────────┤
  │ Global()           │
  └────────────────────┘

  Will: "When we finish running add1, where will we
  go back out to?"
  Matt: "Back out to outer."
  Will: "How would we know? — What's BELOW on the stack."

  ② outer xong (không còn code) → POP:
  ┌────────────────────┐
  │ ╳ outer()          │ ← XOÁ!
  ├────────────────────┤
  │ Global()           │ ← Thread quay lại đây
  └────────────────────┘

  ③ Global tiếp tục (hết code → chương trình kết thúc)

  KẾT QUẢ: counter BỊ SỬA từ 0 → 1 bên trong outer
  NHƯNG outer cũng bị xoá → tất cả BIẾN MẤT!
```

## §3. counter++ — Không Tìm Thấy Ở Local → Đi Đâu?

### 3.1 Bài Viết: Variable Lookup — Cơ Chế Tìm Biến

Khi `add1` chạy `counter++`, JS **không tìm thấy** `counter` trong local memory của `add1`. Vậy JS đi đâu?

```
VARIABLE LOOKUP — TỪNG BƯỚC:
═══════════════════════════════════════════════════════════════

  Code: counter++ (bên trong add1)

  BƯỚC 1: Tìm "counter" trong LOCAL memory của add1:
  ┌──────────────────────────────────────────────────────┐
  │ add1 local memory:                                   │
  │   (trống!)                                           │
  │   → "counter" ở đâu? → KHÔNG TÌM THẤY!             │
  └──────────────────────────────────────────────────────┘

  BƯỚC 2: KHI KHÔNG TÌM THẤY → Đi ra NGOÀI tìm:
  ┌──────────────────────────────────────────────────────┐
  │ outer local memory:                                  │
  │   counter: 0  ← TÌM THẤY!                           │
  │   add1: ƒ                                           │
  │   → counter++ → counter = 0 + 1 = 1                 │
  └──────────────────────────────────────────────────────┘

  ✅ HOẠT ĐỘNG! counter tăng từ 0 → 1.

  NHƯNG... TẠI SAO JS đi đến outer?
```

## §4. CÂU HỎI THEN CHỐT: Defined Ở Đâu vs Called Ở Đâu?

### 4.1 Bài Viết: Câu Hỏi Mà Toàn Bộ Closure Phụ Thuộc Vào

> Will: _"Matt said something that I can't tell is true or not yet."_

Matt nói: _"We went down the call stack"_ — tức là JS tìm counter trong outer vì add1 được **GỌI** (called) bên trong outer.

Nhưng Will chỉ ra: **có thể** vì lý do khác!

```
CÂU HỎI THEN CHỐT — 2 HYPOTHESIS:
═══════════════════════════════════════════════════════════════

  Khi add1 không tìm thấy "counter" trong local memory:
  → JS đi tìm ở outer. TẠI SAO?

  ┌────────────────────────────────────────────────────────┐
  │                                                        │
  │  HYPOTHESIS A: VÌ add1 ĐƯỢC GỌI (called) trong outer  │
  │  ─────────────────────────────────────────────────────  │
  │  → "Đi xuống call stack!" (Matt's answer)              │
  │  → Call stack: add1 → outer → global                   │
  │  → Tìm ở dưới add1 trên stack = outer!                │
  │  → Tên gọi: DYNAMIC SCOPE                             │
  │  → "Tìm biến ở nơi function ĐANG CHẠY"               │
  │                                                        │
  │  HYPOTHESIS B: VÌ add1 ĐƯỢC KHAI BÁO (defined)        │
  │                trong outer                             │
  │  ─────────────────────────────────────────────────────  │
  │  → "Đi đến nơi function ĐƯỢC VIẾT!"                   │
  │  → add1 được khai báo BÊN TRONG outer                 │
  │  → Nên khi cần biến, tìm ở outer!                     │
  │  → Tên gọi: LEXICAL SCOPE (static scope)              │
  │  → "Tìm biến ở nơi function ĐƯỢC KHAI BÁO"           │
  │                                                        │
  └────────────────────────────────────────────────────────┘

  Will: "It could be for EITHER reason:
  ① That we STORED add1 in outer, OR
  ② That we RAN add1 in outer,
  that meant when we didn't find counter inside add1,
  we went out to outer."
```

### 4.2 Diagram: 2 Hypotheses Song Song

```
2 HYPOTHESES — TÌM "counter" BẰNG CÁCH NÀO?
═══════════════════════════════════════════════════════════════

  HYPOTHESIS A — DYNAMIC SCOPE (theo call stack):

  CALL STACK:
  ┌────────────┐
  │ add1()     │ ← counter không có ở đây
  │            │
  │ ↓ xuống   │ ← "Đi XUỐNG call stack!"
  ├────────────┤
  │ outer()    │ ← TÌM THẤY counter = 0!
  ├────────────┤
  │ Global()   │
  └────────────┘

  Logic: "add1 ĐANG CHẠY bên trong outer
  → nên tìm ở outer"

  ─────────────────────────────────────

  HYPOTHESIS B — LEXICAL SCOPE (theo nơi khai báo):

  CODE (SOURCE):
  ┌──────────────────────────────┐
  │ function outer() {           │
  │   let counter = 0;           │ ← counter ở đây!
  │   function add1() {  ←──────│── add1 KHAI BÁO ở đây!
  │     counter++;               │   → nên tìm ở đây!
  │   }                          │
  │   add1();                    │
  │ }                            │
  └──────────────────────────────┘

  Logic: "add1 ĐƯỢC VIẾT bên trong outer
  → nên tìm ở outer"
```

## §5. Tại Sao Không Phân Biệt Được Ở Ví Dụ Này?

### 5.1 Bài Viết: Hai Giả Thuyết Cho Cùng Kết Quả!

> Will: _"Right now I DON'T KNOW. But if we can work out the answer, then we have everything we need to implement closure."_

```
TẠI SAO AMBIGUOUS?
═══════════════════════════════════════════════════════════════

  Trong ví dụ này:

  function outer() {
    let counter = 0;
    function add1() {   ← DEFINED trong outer ✅
      counter++;
    }
    add1();             ← CALLED trong outer ✅
  }

  add1 vừa DEFINED vừa CALLED trong cùng 1 nơi (outer)!

  ┌──────────────────────────────────────────────────────┐
  │ Hypothesis A (dynamic): "tìm ở nơi CALLED"          │
  │ → Called trong outer → tìm ở outer → counter = 0 ✅ │
  │                                                      │
  │ Hypothesis B (lexical): "tìm ở nơi DEFINED"         │
  │ → Defined trong outer → tìm ở outer → counter = 0 ✅│
  │                                                      │
  │ → CẢ HAI cho cùng kết quả! Không phân biệt được!    │
  └──────────────────────────────────────────────────────┘

  ĐỂ PHÂN BIỆT: cần TÁCH nơi defined ≠ nơi called!
  → Define add1 trong outer
  → NHƯNG gọi add1 ở GLOBAL (hoặc nơi khác)
  → Lúc đó: 2 hypotheses cho kết quả KHÁC NHAU!
  → Hypothesis nào ĐÚNG = câu trả lời!
```

## §6. Joe's Insight: Return Function Ra → Test Hypothesis!

### 6.1 Bài Viết: Học Viên Nói Đúng Câu Trả Lời

Will hỏi: _"How could I run the add1 function somewhere else, other than where it was defined?"_

Joe trả lời: _"We could have a return at the end of function outer that returns the add1 function."_

> Will: _"That's just so good from Joe!"_

```
JOE'S INSIGHT — RETURN ADD1 RA!
═══════════════════════════════════════════════════════════════

  BAN ĐẦU (add1 defined + called ở CÙNG chỗ):
  ┌──────────────────────────────────────────────────────┐
  │ function outer() {                                   │
  │   let counter = 0;                                   │
  │   function add1() {   ← DEFINED trong outer          │
  │     counter++;                                       │
  │   }                                                  │
  │   add1();              ← CALLED trong outer          │
  │ }                      → KHÔNG thể phân biệt!       │
  └──────────────────────────────────────────────────────┘

  JOE'S VERSION (add1 defined ở outer, called ở GLOBAL):
  ┌──────────────────────────────────────────────────────┐
  │ function outer() {                                   │
  │   let counter = 0;                                   │
  │   function add1() {   ← DEFINED trong outer          │
  │     counter++;                                       │
  │   }                                                  │
  │   return add1;         ← RETURN RA GLOBAL!           │
  │ }                                                    │
  │                                                      │
  │ const myFunc = outer();                              │
  │ myFunc();              ← CALLED ở GLOBAL!            │
  │                        → Nơi defined ≠ nơi called!   │
  │                        → BÂY GIỜ phân biệt được!     │
  └──────────────────────────────────────────────────────┘

  THỬ NGHIỆM:
  ┌──────────────────────────────────────────────────────┐
  │ Nếu HYPOTHESIS A đúng (dynamic scope):               │
  │ → myFunc() chạy ở GLOBAL                            │
  │ → call stack: myFunc → Global                        │
  │ → tìm counter ở Global → KHÔNG CÒN! → ❌ ERROR!     │
  │                                                      │
  │ Nếu HYPOTHESIS B đúng (lexical scope):               │
  │ → add1 DEFINED trong outer                           │
  │ → dù gọi ở GLOBAL, vẫn tìm counter ở outer!        │
  │ → counter vẫn ĐÓ! → ✅ HOẠT ĐỘNG!                  │
  │                                                      │
  │ → NẾU CODE CHẠY ĐƯỢC → LEXICAL SCOPE = ĐÚNG!        │
  │ → NẾU LỖI → DYNAMIC SCOPE = ĐÚNG!                   │
  │                                                      │
  │ (Spoiler: Code chạy! JS = LEXICAL SCOPE!)            │
  └──────────────────────────────────────────────────────┘
```

### 6.2 Nhưng... Outer EC Đã Bị XOÁ!

Will đặt ra puzzle cuối cùng:

> _"I will be really clear: when we return our add1, that outer execution context is going to CLOSE. So we've got an interesting puzzle ahead of us."_

```
PUZZLE — OUTER EC BỊ XOÁ!
═══════════════════════════════════════════════════════════════

  function outer() {
    let counter = 0;          ← counter ở ĐÂY
    function add1() {
      counter++;
    }
    return add1;
  }

  const myFunc = outer();     ← outer chạy xong → EC XOÁ!

  Sau dòng này:
  ┌─────────────────────────────────┐
  │ GLOBAL MEMORY:                  │
  │   outer: ƒ                      │
  │   myFunc: ƒ add1() {            │
  │     counter++;                  │
  │   }                             │
  │                                 │
  │ ╳ EC outer → XOÁ RỒI!          │
  │ ╳ counter: 0 → XOÁ RỒI!        │
  │ ╳ add1: ƒ → XOÁ RỒI!           │
  └─────────────────────────────────┘

  myFunc();                   ← GỌI! counter++ ???

  CÂU HỎI: counter ĐÃ BỊ XOÁ!
  → add1 (giờ là myFunc) cần counter++
  → NHƯNG counter KHÔNG CÒN ở đâu cả!
  → GLOBAL không có counter!
  → outer EC đã BIẾN MẤT!
  → Vậy counter ở đâu?!?! 🤯

  ĐÂY = CLOSURE!
  → add1 "MANG THEO" counter khi return ra!

  → Bài tiếp theo sẽ giải thích!
```

## §7. Dynamic Scope vs Lexical Scope — Preview

```
DYNAMIC SCOPE vs LEXICAL SCOPE:
═══════════════════════════════════════════════════════════════

  DYNAMIC SCOPE (Hypothesis A):
  ┌──────────────────────────────────────────────────────┐
  │ → Tìm biến dựa vào CALL STACK (ai gọi ai)          │
  │ → "Tìm ở nơi function ĐANG CHẠY"                   │
  │ → Thay đổi TUỲ THEO ngữ cảnh gọi                   │
  │                                                      │
  │ Ngôn ngữ dùng: Bash, Emacs Lisp (cũ), Perl (option)│
  │                                                      │
  │ Ví dụ nếu JS dùng dynamic scope:                    │
  │ let counter = 100;       // global counter           │
  │ const fn = outer();      // fn = add1                │
  │ fn();                    // counter++ → 100 → 101!   │
  │ // → Tìm counter ở GLOBAL vì fn GỌI ở global!      │
  └──────────────────────────────────────────────────────┘

  LEXICAL SCOPE (Hypothesis B) — JS DÙNG CÁI NÀY! ✅
  ┌──────────────────────────────────────────────────────┐
  │ → Tìm biến dựa vào SOURCE CODE (viết ở đâu)        │
  │ → "Tìm ở nơi function ĐƯỢC KHAI BÁO"               │
  │ → CỐ ĐỊNH, không thay đổi theo ngữ cảnh gọi       │
  │ → "Lexical" = "liên quan đến text/code"              │
  │                                                      │
  │ Ngôn ngữ dùng: JavaScript, Python, C, Java, Rust    │
  │                                                      │
  │ Ví dụ (JS thực tế):                                 │
  │ let counter = 100;       // global counter           │
  │ const fn = outer();      // fn = add1                │
  │ fn();                    // counter++ → 0 → 1!      │
  │ // → Tìm counter ở OUTER vì add1 KHAI BÁO ở outer! │
  │ // → KHÔNG quan tâm gọi ở đâu!                      │
  └──────────────────────────────────────────────────────┘

  Will: "Where you DEFINE your functions determines
  what data it has access to when you call it."
```

## §8. Tự Implement: Test 2 Hypotheses

```javascript
// TEST: CHỨNG MINH JS DÙNG LEXICAL SCOPE (Hypothesis B)

function outer() {
  let counter = 0;
  function add1() {
    counter++;
    console.log("counter =", counter);
  }
  return add1;
}

let counter = 100; // Global counter — CÓ TỒN TẠI!

const myFunc = outer(); // outer chạy → return add1 → XOÁ EC outer

myFunc(); // counter = 1   ← KHÔNG PHẢI 101!
myFunc(); // counter = 2   ← counter CỦA OUTER (closure)!
myFunc(); // counter = 3

console.log(counter); // 100 ← global counter KHÔNG BỊ ĐỤNG!

// NẾU dynamic scope (Hypothesis A):
// → myFunc() chạy ở global → tìm counter ở global → 100 → 101
// → KẾT QUẢ SẼ LÀ: 101, 102, 103
// → global counter = 103

// NẾU lexical scope (Hypothesis B) — ĐÂY LÀ THỰC TẾ!
// → add1 DEFINED trong outer → tìm counter CỦA outer
// → counter CỦA outer đi theo add1 (closure/backpack!)
// → KẾT QUẢ: 1, 2, 3
// → global counter = 100 (KHÔNG BỊ ẢNH HƯỞNG!)

// → LEXICAL SCOPE WINS! ✅
```

```javascript
// TEST 2: CÒN MỘT BƯỚC NỮA — GỌI Ở NƠI CÓ counter KHÁC

function outer() {
  let message = "from outer";
  function inner() {
    return message; // tìm message ở đâu?
  }
  return inner;
}

function anotherPlace() {
  let message = "from anotherPlace"; // counter KHÁC!
  const fn = outer(); // fn = inner (DEFINED trong outer)
  return fn(); // GỌI fn bên trong anotherPlace
}

console.log(anotherPlace()); // "from outer" ← KHÔNG phải "from anotherPlace"!

// → inner DEFINED trong outer → tìm message CỦA outer!
// → DÙ gọi trong anotherPlace (có message riêng) → BỎ QUA!
// → LEXICAL SCOPE: "nơi VIẾT" quyết định, KHÔNG phải "nơi GỌI"!
```

```javascript
// TEST 3: Visual — Call Stack vs Scope Chain

function grandparent() {
  let x = "grandparent's x";

  function parent() {
    // KHÔNG khai báo x → tìm ở grandparent (lexical!)

    function child() {
      // KHÔNG khai báo x → tìm ở parent → rồi grandparent!
      console.log(x);
    }
    return child;
  }
  return parent;
}

const getChild = grandparent()(); // grandparent → parent → child
getChild(); // "grandparent's x"

// Call stack khi getChild() chạy:
// [getChild(), Global()] ← chỉ 2 tầng!
// NHƯNG x vẫn tìm ở grandparent — dù grandparent
// KHÔNG CÒN trên call stack!
// → CHỨNG MINH: KHÔNG dùng call stack để tìm biến!
// → Dùng LEXICAL scope (nơi VIẾT code)!
```

## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Lexical vs Dynamic Scope

```
5 WHYS: TẠI SAO JS CHỌN LEXICAL SCOPE?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao cần quy tắc tìm biến?
  └→ Vì function có thể dùng biến KHÔNG khai báo bên trong.
     Cần quy tắc: "nếu không có ở local, tìm ở ĐÂU?"

  WHY ②: Tại sao lexical (nơi viết) chứ không dynamic (nơi gọi)?
  └→ Vì lexical = PREDICTABLE! Nhìn code = biết ngay function
     truy cập biến nào. Dynamic = phải trace RUNTIME mới biết!

  WHY ③: Tại sao predictable quan trọng?
  └→ Vì nếu function A gọi function B, B dùng biến x,
     dynamic scope: x = tuỳ A có x gì → BUG khó tìm!
     lexical scope: x = luôn cố định → an toàn!

  WHY ④: Có ngôn ngữ nào dùng dynamic scope không?
  └→ Có! Bash shell, Emacs Lisp cũ, Perl (option).
     Nhưng PHẦN LỚN ngôn ngữ hiện đại → LEXICAL!
     Dynamic scope = footgun quá lớn.

  WHY ⑤: Lexical scope + return function = closure?
  └→ ĐÚNG! Lexical scope = function "nhớ" nơi nó SINH RA.
     Return function = function SỐNG Ở NƠI KHÁC.
     Kết hợp: function ở global vẫn "nhớ" local data!
     ⭐ Closure = hệ quả của LEXICAL SCOPE + FIRST-CLASS ƒ!
```

### 9.5 Pattern ⑤: Reverse Engineering & Implementation

```javascript
// TỰ BUILD: Giả lập SCOPE CHAIN lookup

function simulateScopeLookup(variableName, currentScope) {
  // Tìm trong scope hiện tại:
  if (variableName in currentScope.variables) {
    return {
      found: true,
      value: currentScope.variables[variableName],
      foundIn: currentScope.name,
    };
  }

  // Nếu có outer scope (LEXICAL!) → tìm tiếp:
  if (currentScope.outer) {
    return simulateScopeLookup(variableName, currentScope.outer);
  }

  // Hết scope chain → ReferenceError!
  return { found: false, error: `${variableName} is not defined` };
}

// Mô phỏng scope chain của ví dụ bài giảng:
const globalScope = {
  name: "Global",
  variables: { outer: "ƒ" },
  outer: null, // Global = đỉnh!
};

const outerScope = {
  name: "outer()",
  variables: { counter: 0, add1: "ƒ" },
  outer: globalScope, // ← LEXICAL! outer nằm trong global
};

const add1Scope = {
  name: "add1()",
  variables: {}, // TRỐNG!
  outer: outerScope, // ← LEXICAL! add1 nằm trong outer
};

// Tìm "counter" từ add1:
console.log(simulateScopeLookup("counter", add1Scope));
// { found: true, value: 0, foundIn: "outer()" }
// → Tìm ở add1 → không có → đi đến outer → TÌM THẤY! ✅

// Tìm "outer" từ add1:
console.log(simulateScopeLookup("outer", add1Scope));
// { found: true, value: "ƒ", foundIn: "Global" }
// → add1 → outer → Global → TÌM THẤY! ✅

// Tìm "xyz" từ add1:
console.log(simulateScopeLookup("xyz", add1Scope));
// { found: false, error: "xyz is not defined" }
// → add1 → outer → Global → KHÔNG CÓ! → ReferenceError! ❌
```

```
  │     → Brendan Eich: "Scheme in the browser"
```
