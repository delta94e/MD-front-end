## §4. Method = Function On Object — "Chris's Comeback!"

> Will: _"What do we call a function stored on an object? Chris, it's your chance for your big comeback from yesterday!"_
> Chris: _"A method."_
> Will: _"We call it a METHOD. Our brilliant engineer Chris is right!"_

```
METHOD vs FUNCTION:
═══════════════════════════════════════════════════════════════

  FUNCTION:
  → Standalone! Not attached to an object!
  → function increment() { ... }
  → Called: increment()

  METHOD:
  → A function STORED ON an object!
  → { increment: function() { ... } }
  → Called: user1.increment()
  → Has access to `this` (the object!)

  ┌──────────────────────────────────────────────────┐
  │ // function:                                     │
  │ function sayHello() { console.log("Hi!"); }      │
  │                                                  │
  │ // method:                                       │
  │ const user = {                                   │
  │   name: "Ari",                                   │
  │   greet() { console.log(`Hi, ${this.name}!`); }  │
  │ };                                               │
  │                                                  │
  │ sayHello();      // function call                │
  │ user.greet();    // method call — has `this`!    │
  └──────────────────────────────────────────────────┘
```

```
  user2.increment();
```

## §6. Way ③: Object.create — "Fine-Grained Control!"

> Matt: _"Object.create!"_
> Will: _"Well done, Matt! Object.create is going to give us FINE-GRAINED CONTROL later on over this object and what it can do."_

```
TRACE — Object.create(null)                    [~2ms]
═══════════════════════════════════════════════════════════════

  const user3 = Object.create(null);
  user3.name = "Tam";
  user3.score = 9;
  user3.increment = function() { this.score++; };

  WHAT Object.create DOES:
  ┌──────────────────────────────────────────────────┐
  │ Object.create(null)                              │
  │ → Creates a NEW EMPTY object!                   │
  │ → Passing null = no prototype link!             │
  │ → "Pure" object, no inherited methods!          │
  │ → Then add properties via dot notation!         │
  └──────────────────────────────────────────────────┘

  RESULT:
  ┌──────────────────────────────────────────────────┐
  │ user3: { name: "Tam", score: 9, increment: ƒ }  │
  │                                                  │
  │ → Tam is the expert! Score = 9!                 │
  └──────────────────────────────────────────────────┘

  user3.increment();
  → user3.score: 9 → 10!
  → THE DREAM ACHIEVED! ✅

  Will: "This last way is going to turn out to be
  VERY, VERY USEFUL for us later on, giving us
  FINE-GRAINED CONTROL over certain characteristics
  of our object."

  WHY Object.create MATTERS (PREVIEW!):
  ┌──────────────────────────────────────────────────┐
  │ Object.create(someObject)                        │
  │                                                  │
  │ → Creates empty object WITH a special bond!     │
  │ → Links __proto__ to someObject!                │
  │ → = THE PROTOTYPE CHAIN!                        │
  │ → = How we SHARE methods without copying!       │
```

## §8. Tự Implement: 3 Object Creation Patterns

```javascript
// CHỨNG MINH 1: 3 ways — verify they all work the same!

// Way 1: Object Literal
const user1 = {
  name: "Ari",
  score: 3,
  increment() {
    this.score++;
  },
};

// Way 2: Dot Notation
const user2 = {};
user2.name = "J";
user2.score = 5;
user2.increment = function () {
  this.score++;
};

// Way 3: Object.create
const user3 = Object.create(null);
user3.name = "Tam";
user3.score = 9;
user3.increment = function () {
  this.score++;
};

// All achieve THE DREAM:
user1.increment();
user2.increment();
user3.increment();

console.log(user1); // { name: "Ari", score: 4, increment: ƒ }
console.log(user2); // { name: "J", score: 6, increment: ƒ }
console.log(user3); // { name: "Tam", score: 10, increment: ƒ }
// All work! ✅
```

```javascript
// CHỨNG MINH 2: Methods are SEPARATE copies!

console.log(user1.increment === user2.increment); // false!
console.log(user2.increment === user3.increment); // false!
console.log(user1.increment === user3.increment); // false!

// → 3 DIFFERENT function objects in memory!
// → Same code but different memory addresses!
// → WASTEFUL! This is the problem we need to solve!
```

```javascript
// CHỨNG MINH 3: Object.create(null) vs Object.create(Object.prototype)

const pureObj = Object.create(null);
const normalObj = Object.create(Object.prototype); // same as {}

console.log(pureObj.toString); // undefined! No methods!
console.log(normalObj.toString); // ƒ toString() — inherited!

// Object.create(null) = truly empty!
// Object.create(Object.prototype) = has standard methods!
// {} = Object.create(Object.prototype) essentially!

// This is the "fine-grained control" Will mentions!
// We can choose WHAT our object inherits from!
```

```javascript
// CHỨNG MINH 4: Bracket notation (Joe's suggestion)

const user4 = {};
const propName = "name"; // dynamic key!

user4[propName] = "Dynamic"; // bracket notation!
user4["score"] = 7;

console.log(user4.name); // "Dynamic"
console.log(user4["score"]); // 7

// Bracket = dynamic property names!
// Dot = static property names!
// Both work for accessing and setting!
```

```javascript
// CHỨNG MINH 5: Encapsulation — prevent accidental usage

const user = {
  name: "Ari",
  score: 3,
  increment() {
    this.score++;
  },
};

const quizBoard = {
  name: "Quiz League",
  totalScore: 100,
  resetScores() {
    this.totalScore = 0;
  },
};

// ENCAPSULATION prevents:
// user.resetScores → undefined! Can't use quiz methods on user!
// quizBoard.increment → undefined! Can't use user methods on quiz!

// Each object's methods are ENCAPSULATED with its data!
// Will: "I don't want to accidentally use increment on
// my Quiz League board!"
```

## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Object Creation

```
5 WHYS: TẠI SAO CÓ 3 CÁCH TẠO OBJECT?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao cần Object Literal?
  └→ Vì COMENIENCE! Tạo object nhanh nhất!
     Tất cả properties defined cùng lúc!
     Most common way in day-to-day code!

  WHY ②: Tại sao cần Dot Notation?
  └→ Vì FLEXIBILITY! Add properties dynamically!
     Don't need to know all props at creation!
     Can add/modify AFTER creation!

  WHY ③: Tại sao cần Object.create?
  └→ Vì CONTROL! Choose WHAT object inherits from!
     Object.create(null) = pure empty!
     Object.create(proto) = linked to proto!
     = THE KEY to prototype chain!

  WHY ④: Tại sao Object.create quan trọng nhất?
  └→ Vì nó cho phép LINK objects together!
     Object.create(sharedMethods) = share methods!
     = Solve the copy problem!
     = Foundation of OOP in JS!

  WHY ⑤: Tại sao function on object gọi là "method"?
  └→ Vì convention from other OOP languages!
     Method = function WITH context (this)!
     Called via dot notation on an object!
     Different invocation = different `this`!
```

### 9.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — OBJECT CREATION:
═══════════════════════════════════════════════════════════════

  ECMAScript spec — Object creation:

  ① Object Literal { }:
  → Parser sees { → creates ObjectExpression
  → Each key:value = Property definition
  → OrdinaryObjectCreate(Object.prototype)
  → DefineOwnProperty for each key

  ② Dot Notation (assignment):
  → obj.prop = value
  → [[Set]](prop, value, obj)
  → CreateDataProperty or Set existing

  ③ Object.create(proto):
  → OrdinaryObjectCreate(proto)
  → Sets [[Prototype]] internal slot to proto!
  → This is WHERE the magic happens!

  [[Prototype]] INTERNAL SLOT:
  ┌──────────────────────────────────────────────────┐
  │ Every object has [[Prototype]]!                  │
  │ → It's the LINK to another object!              │
  │ → Property lookup follows this link!            │
  │ → Object.create(X) sets it to X!               │
  │ → { } sets it to Object.prototype!             │
  │ → Object.create(null) sets it to null!          │
  └──────────────────────────────────────────────────┘
```

### 9.3 Pattern ③: Trade-off Analysis

```
TRADE-OFFS — 3 WAYS:
═══════════════════════════════════════════════════════════════

  ┌──────────────────┬──────────┬──────────┬──────────┐
  │                  │ Literal  │ Dot      │ .create  │
  ├──────────────────┼──────────┼──────────┼──────────┤
  │ Speed to write   │ ✅ Fast! │ ⚠️ Lines │ ⚠️ Lines │
  │ Readability      │ ✅ Clear │ ✅ Clear │ ⚠️ Less  │
  │ Dynamic props    │ ❌ Fixed │ ✅ Yes!  │ ✅ Yes!  │
  │ Prototype ctrl   │ ❌ No    │ ❌ No    │ ✅ YES!  │
  │ Pure empty obj   │ ❌ No    │ ❌ No    │ ✅ null! │
  │ Common usage     │ ✅ Most  │ ✅ Often │ ⚠️ OOP   │
  └──────────────────┴──────────┴──────────┴──────────┘

  KEY: Object.create wins for OOP because it controls
  the [[Prototype]] link — the FOUNDATION of sharing!
```

### 9.5 Pattern ⑤: Reverse Engineering

```javascript
// PROVE: All 3 create normal objects

const literal = { x: 1 };
const dotNotation = {};
dotNotation.x = 1;
const created = Object.create(Object.prototype);
created.x = 1;

// All have same structure:
console.log(JSON.stringify(literal)); // {"x":1}
console.log(JSON.stringify(dotNotation)); // {"x":1}
console.log(JSON.stringify(created)); // {"x":1}

// But different prototype chains:
const pure = Object.create(null);
pure.x = 1;
console.log(pure.toString); // undefined! No prototype!
console.log(literal.toString); // ƒ toString() — from Object.prototype!

// Object.create = CONTROL the chain! This is the key!
```

### 9.6 Pattern ⑥: Lịch Sử

```

  │     → Inspired by Self language!
  │

  │
  ↓

  │     → Douglas Crockford proposed!

  │     → Fine-grained prototype control!
  │     → Will: "VERY VERY useful later!"

  ↓

  │     → class User { constructor() {} }
  │     → Still Object.create underneath!

  ↓

        → { ...obj } for shallow copy
        → structuredClone(obj) for deep copy

  Crockford (2008): "JavaScript: The Good Parts"
  → Object.create is the preferred way!
  → Constructor functions are confusing!
  → Prototypal inheritance > classical!
```
