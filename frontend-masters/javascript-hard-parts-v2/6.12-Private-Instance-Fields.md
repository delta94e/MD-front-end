## §3. The # Symbol — Private Fields!

> Will: _"Using the hashtag, using the number symbol — we can set up a property that when we create our object, will have a very SPECIAL characteristic."_

```
THE # SYNTAX:
═══════════════════════════════════════════════════════════════

  class UserCreator {
    #score;                      // ← PRIVATE field declaration!

    constructor(name, score) {
      this.name = name;
      this.#score = score;       // ← Set private field!
    }

    increment() {
      this.#score++;             // ← Access from method = OK!
    }

    getScore() {
      return this.#score;        // ← Access from method = OK!
    }
  }

  const user1 = new UserCreator("Ari", 3);

  user1.increment();             // ✅ Works!
  user1.getScore();              // ✅ Returns 4!

  user1.#score;                  // ❌ SyntaxError!
  user1.#score++;                // ❌ SyntaxError!

  Will: "This one is a SPECIAL type of property
  on the object that is only going to be available
  in methods called on this object directly."

  "We will NO LONGER be able to write user1.score++.
  We will get an ERROR."
```

## §8. Tự Implement: Private Fields In Action

```javascript
// CHỨNG MINH 1: Private fields basic usage

class UserCreator {
  #score; // declare private field

  constructor(name, score) {
    this.name = name;
    this.#score = score;
  }

  increment() {
    this.#score++; // ✅ Access from class method!
  }

  getScore() {
    return this.#score; // ✅ Access from class method!
  }
}

const user1 = new UserCreator("Ari", 3);

user1.increment();
console.log(user1.getScore()); // 4 ✅

// Cannot access from outside:
// console.log(user1.#score); // SyntaxError!

// Regular property "score" doesn't exist:
console.log(user1.score); // undefined!

// Can only see public properties:
console.log(Object.keys(user1)); // ["name"] — no #score!
```

```javascript
// CHỨNG MINH 2: Private vs Public comparison

class BankAccount {
  #balance; // PRIVATE — can't be touched from outside!
  owner; // PUBLIC — anyone can see/change!

  constructor(owner, balance) {
    this.owner = owner;
    this.#balance = balance;
  }

  deposit(amount) {
    if (amount > 0) {
      this.#balance += amount; // ✅ controlled update!
      console.log(`Deposited ${amount}. Balance: ${this.#balance}`);
    }
  }

  withdraw(amount) {
    if (amount > 0 && amount <= this.#balance) {
      this.#balance -= amount; // ✅ controlled update!
      console.log(`Withdrew ${amount}. Balance: ${this.#balance}`);
    } else {
      console.log("Insufficient funds!");
    }
  }

  getBalance() {
    return this.#balance; // ✅ read-only access!
  }
}

const account = new BankAccount("Ari", 1000);

// The PROPER way:
account.deposit(500); // "Deposited 500. Balance: 1500"
account.withdraw(200); // "Withdrew 200. Balance: 1300"

// Can't cheat!
// account.#balance = 999999; // SyntaxError! ❌
// account.#balance;          // SyntaxError! ❌

// Can only READ via getter method:
console.log(account.getBalance()); // 1300 ✅
```

```javascript
// CHỨNG MINH 3: Private fields are PER-CLASS, not per-instance

class User {

```

```javascript
// CHỨNG MINH 4: Private fields NOT inherited by subclass

class Animal {
```

```javascript
// CHỨNG MINH 5: # is part of the name, not an operator!

class Example {
```

## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Private Fields

```
5 WHYS: TẠI SAO CẦN PRIVATE FIELDS?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao cần private?
  └→ Vì public properties = anyone can change!
     user1.score = -1; ← NONSENSE!
     "100,000 lines of code, I don't want someone
     else saying 'I'll just change that value.'"

  WHY ②: Tại sao convention (underscore) không đủ?
  └→ Vì _score = still PUBLIC! Just a "hint"!
     Anyone CAN still do user1._score = 999!
     No enforcement!

  WHY ③: Tại sao dùng # syntax?
  └→ Vì cần compile-time detection!
     JS parser sees # → knows it's private!
     SyntaxError BEFORE code runs!
     Không thể bypass bằng bracket notation!

  WHY ④: Tại sao separate [[PrivateElements]]?
  └→ Vì regular properties accessible by anyone!
     Need SEPARATE storage that JS engine guards!
     "Not some syntactic sugar — REAL DEAL!"

  WHY ⑤: Tại sao class methods CAN access?
  └→ Vì methods defined in SAME class!
     JS engine knows which class declared #field!
     Same class = trusted code = ACCESS GRANTED!
```

### 9.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — PRIVATE FIELDS MECHANISM:
═══════════════════════════════════════════════════════════════

  ECMAScript Internal Slots:

  Each object can have [[PrivateElements]]:
  → A List of { [[Key]], [[Kind]], [[Value]] }
  → [[Key]] = Private Name (e.g., #score)
  → [[Kind]] = "field", "method", or "accessor"
  → [[Value]] = the actual value

  ACCESS RULES:
  ┌──────────────────────────────────────────────────┐
  │ PrivateFieldGet(O, P):                           │
  │ 1. Let entry = find P in O.[[PrivateElements]]   │
  │ 2. If not found → TypeError!                    │
  │ 3. Return entry.[[Value]] ✅                     │
  │                                                  │
  │ WHO CAN CALL PrivateFieldGet?                    │
  │ → Only code LEXICALLY inside the class body!    │
  │ → Parser enforces at PARSE TIME!                │
  │ → Not a runtime check — a SYNTAX check!         │
  └──────────────────────────────────────────────────┘

  KEY: # is checked by the PARSER, not runtime!
  → user1.#score outside class = SyntaxError!
  → Not even TypeError — SYNTAX error!
  → Code won't even START running!
```

```javascript
class Secret {
```
