```javascript
// CHỨNG MINH 2: Build Promise + fetch from scratch!

class MyPromise {
  constructor() {
    this._result = undefined; // [[result]] — hidden!
    this._fulfillReactions = []; // [[fulfillReactions]] — hidden!
    this._status = "pending";
  }

  // .then() — the ONLY way to access hidden array!
  then(onFulfill) {
    if (this._status === "fulfilled") {
      // Already resolved! Run immediately:
      onFulfill(this._result);
    } else {
      // Not yet! Push into reactions array:
      this._fulfillReactions.push(onFulfill);
      console.log(`  [.then] Pushed ${onFulfill.name} into fulfillReactions`);
    }
    return this; // allow chaining!
  }

  // Called when browser work completes:
  _resolve(value) {
    console.log(`  [resolve] result = "${value}"`);
    this._result = value;
    this._status = "fulfilled";

    // Trigger ALL fulfillReactions!
    this._fulfillReactions.forEach((fn) => {
      console.log(`  [auto] Running ${fn.name}("${value}")`);
      fn(value); // auto-pass result as argument!
    });
  }
}

// Two-pronged facade function:
function myFetch(url) {
  console.log(`\n[fetch] "${url}" — TWO PRONGS!`);

  // PRONG 1 — JS: Create & return Promise!
  const promise = new MyPromise();
  console.log(`  [PRONG 1] Promise created: { result: undefined }`);

  // PRONG 2 — Browser: Network request!
  console.log(`  [PRONG 2] Network request → ${url}`);
  console.log(`  [PRONG 2] Method: GET (default)`);

  // Simulate browser network work:
  setTimeout(() => {
    console.log(`\n[Browser 270ms] Response from TikTok!`);
    promise._resolve("cute puppy"); // intimate bond!
  }, 270);

  return promise; // returned IMMEDIATELY!
}
```

## §11. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 11.1 Pattern ①: 5 Whys — fetch Internals

```
5 WHYS: TẠI SAO fetch HOẠT ĐỘNG NHƯ VẬY?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao fetch trả về Promise, không data?
  └→ Vì network request MẤT THỜI GIAN!
     Nếu return data → phải BLOCK = UI freeze!
     Return Promise = placeholder ngay lập tức!

  WHY ②: Tại sao cần "intimate bond"?
  └→ Vì data arrives ở BROWSER (network thread)!
     Nhưng cần update object ở JS (Promise)!
     Bond = browser biết WHERE to put data!

  WHY ③: Tại sao properties là hidden ([[brackets]])?
  └→ Vì KHÔNG MUỐN developer modify trực tiếp!
     promise.result = "fake" → would break everything!
     Hidden = chỉ JS engine mới modify được!
     .then() = controlled interface!

  WHY ④: Tại sao .then() thay vì push() trực tiếp?
  └→ Vì fulfillReactions = hidden!
     .then() = public API cho hidden array!
     Also: .then() comes from prototype chain!
     = Shared across ALL promise objects!

  WHY ⑤: Tại sao auto-pass result vào function?
  └→ Vì developer MUỐN data ngay khi available!
     Auto-pass = convenient!
     display(data) → data = result value!
     KHÔNG cần developer access .result manually!
```

### 11.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — FETCH API SPEC:
═══════════════════════════════════════════════════════════════

  WHATWG Fetch Living Standard:

  "fetch(input, init)"
  → Returns a Promise<Response>
  → input = URL string or Request object
  → init = optional: method, headers, body, etc.

  DEFAULT behavior:
  → method: "GET"
  → mode: "cors"
  → credentials: "same-origin"

  ECMAScript 25.6.5.4 — Promise.prototype.then:
  → "Let C be SpeciesConstructor(promise, %Promise%)"
  → "Let resultCapability = NewPromiseCapability(C)"
  → "Return PerformPromiseThen(promise, onFulfilled,
     onRejected, resultCapability)"

  KEY: PerformPromiseThen:
  → If promise is pending:
     → Append onFulfilled to [[PromiseFulfillReactions]]
  → If promise is fulfilled:
     → Enqueue a Job to call onFulfilled with result

  "Job" = microtask!
  → Sẽ học chi tiết bài tiếp theo!
```

### 11.3 Pattern ③: Trade-off Analysis

```
TRADE-OFFS: RESULT PROPERTY:
═══════════════════════════════════════════════════════════════

  ┌──────────────────┬──────────────────┬──────────────────┐
  │                  │ Hidden [[result]]│ Public result     │
  ├──────────────────┼──────────────────┼──────────────────┤
  │ Safety           │ ✅ Engine only!  │ ❌ Anyone modify! │
  │ Predictability   │ ✅ Controlled!   │ ❌ Chaotic!       │
  │ Developer UX     │ ⚠️ Need .then()  │ ✅ Direct access  │
  │ Debugging        │ ⚠️ Harder        │ ✅ Easy inspect   │
  │ One-time set     │ ✅ Guaranteed!   │ ❌ Could re-set!  │
  └──────────────────┴──────────────────┴──────────────────┘

  → JS chose SAFETY over convenience!
  → Hidden = immutable after being set!
  → .then() = the ONLY controlled access!
  → Same philosophy as closure's [[Scope]]!
```

### 11.6 Pattern ⑥: Lịch Sử

```
LỊCH SỬ: XHR → FETCH — CÙNG GOAL, KHÁC WAY:
═══════════════════════════════════════════════════════════════

  1999: XMLHttpRequest (IE5)
  │     → ONE-pronged: browser only!
  │     → No Promise, no tracking in JS!
  │     → Callback-based, verbose:
  │
  │     var xhr = new XMLHttpRequest();
  │     xhr.open("GET", url);
  │     xhr.onload = function() { /* callback */ };
  │     xhr.send();
  │     // 4 lines just to make a request! 💀
  │
  ↓
  2010: jQuery.ajax()
  │     → Simplified XHR!
  │     → $.ajax(url).done(fn) — nicer API!
  │     → Still one-pronged underneath!
  │
  ↓
  2015: fetch() — ES2015!
  │     → TWO-pronged! 🔀
  │     → Returns Promise immediately!
  │     → 1 line: fetch(url).then(fn)
  │     → Will: "The 5 most powerful letters!"
  │
  ↓
  2017: async/await
  │     → const data = await fetch(url);
  │     → LOOKS synchronous! Is still async!
  │
  ↓
  2024: fetch + AbortController + streaming!
        → const controller = new AbortController();
        → fetch(url, { signal: controller.signal });
        → controller.abort();
        → = Cancel fetch mid-flight!
```
