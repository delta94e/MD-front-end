## §3. Primitives = By Value (Stack)

```
PRIMITIVES — LƯU TRỰC TIẾP (BY VALUE):
═══════════════════════════════════════════════════════════════

  const a = 7;
  let b = a;    // COPY giá trị 7!
  b = 42;       // thay đổi b, KHÔNG ảnh hưởng a!

  STACK:
  ┌──────────────────┐
  │ a: 7             │ ← giá trị 7 LƯU TRỰC TIẾP
  │ b: 7 → 42       │ ← COPY giá trị, sau đó thay đổi
  └──────────────────┘

  console.log(a); // 7 ← KHÔNG BỊ ẢNH HƯỞNG!
  console.log(b); // 42

  PRIMITIVES:
  → number, string, boolean, null, undefined, symbol, bigint
  → Lưu TRỰC TIẾP trong stack
  → Copy = tạo BẢN SAO MỚI (independent!)
  → So sánh = so sánh GIÁ TRỊ!

  7 === 7     → true (cùng giá trị!)
  "abc" === "abc" → true (cùng giá trị!)

  Will: "The values that we've been seeing — 7, the
  string '3', true — have been what's known as PRIMITIVES.
  They're stored DIRECTLY where we save them."
```

## §4. Objects = By Reference (Heap) — Chỉ Lưu Pointer!

### 4.1 Bài Viết: Objects Không Nằm Trong Stack

> Will: _"Rather than directly storing in memory 'name: Will, id: 105' — that was instead stored off in our HEAP. What we get in JavaScript is a LINK to this underlying position in memory."_

```
OBJECTS — LƯU REFERENCE (BY REFERENCE):
═══════════════════════════════════════════════════════════════

  const userStored = { name: "Will", id: 105 };

  THỰC TẾ XẢY RA:
  ┌─────────────────────────────────────────────────────┐
  │                                                     │
  │ STACK:                         HEAP:                │
  │ ┌────────────────────┐        ┌──────────────────┐  │
  │ │ userStored: @1001  │──────→ │ @1001:           │  │
  │ └────────────────────┘        │ name: "Will"     │  │
  │                               │ id: 105          │  │
  │ Object KHÔNG nằm             │                  │  │
  │ trong stack!                  └──────────────────┘  │
  │ Chỉ có POINTER @1001!                              │
  │                                                     │
  └─────────────────────────────────────────────────────┘

  const userSubmitted = { name: "Will", id: 105 };

  THỰC TẾ:
  ┌─────────────────────────────────────────────────────┐
  │                                                     │
  │ STACK:                         HEAP:                │
  │ ┌────────────────────┐        ┌──────────────────┐  │
  │ │ userStored: @1001  │──────→ │ @1001:           │  │
  │ └────────────────────┘        │ name: "Will"     │  │
  │                               │ id: 105          │  │
  │ ┌────────────────────┐        ├──────────────────┤  │
  │ │ userSubmitted:     │──────→ │ @1002:           │  │
  │ │            @1002   │        │ name: "Will"     │  │
  │ └────────────────────┘        │ id: 105          │  │
  │                               └──────────────────┘  │
  │                                                     │
  │ 2 OBJECTS KHÁC NHAU ở 2 VỊ TRÍ KHÁC NHAU!         │
  │ @1001 ≠ @1002 (dù contents giống nhau!)            │
  │                                                     │
  └─────────────────────────────────────────────────────┘

  Will: "In some languages, we can actually directly
  work with these numbers — pointer MATH — to exactly,
  specifically position data in our heap."
```

## §5. === So Sánh REFERENCE, Không Phải CONTENTS! 🔥

### 5.1 Bài Viết: "False? But They Look The Same!"

```
=== SO SÁNH REFERENCE!
═══════════════════════════════════════════════════════════════

  userSubmitted === userStored

  STACK:
  ┌────────────────────┐
  │ userStored: @1001  │
  │ userSubmitted:@1002│
  └────────────────────┘

  === COMPARE:
  ┌──────────────────────────────────────────────────┐
  │ @1002 === @1001                                  │
  │                                                  │
  │ → SO SÁNH ĐỊA CHỈ, không so sánh contents!     │
  │ → @1002 ≠ @1001                                  │
  │ → false!                                         │
  │                                                  │
  │ === KHÔNG "mở" object ra để xem bên trong!      │
  │ === CHỈ kiểm tra: "cùng vị trí trong heap?"     │
  └──────────────────────────────────────────────────┘

  ❌ userSubmitted == userStored   → false (cùng logic!)
  ❌ userSubmitted === userStored  → false

  Will: "Am I comparing the CONTENTS of the objects?
  No. What am I comparing? The POINTERS, the ADDRESSES,
  the REFERENCES."

  Will: "Is 1002 equal to 1001? Is it the same position
  in memory? Absolutely NOT. So I get false."

  HÌNH ẢNH:
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  userStored ──→ @1001 ──→ {name:"Will",id:105}  │
  │                   ↕                              │
  │              @1001 ≠ @1002  → false!             │
  │                   ↕                              │
  │  userSubmitted→ @1002 ──→ {name:"Will",id:105}  │
  │                                                  │
  │  Contents: GIỐNG!                                │
  │  References: KHÁC! → === → false!                │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

## §6. anotherLink = userStored — Copy Pointer, Không Copy Data!

### 6.1 Bài Viết: Assignment Copies Reference!

> Will: _"Am I doing a COPY of userStored when I assign it to anotherLink? No, not at all. Instead, I am doing a copy of its ADDRESS, its pointer, its reference. Another link and userStored — now they're the same. But they ain't a copy and they ain't a backup."_

```
COPY REFERENCE, KHÔNG PHẢI DATA:
═══════════════════════════════════════════════════════════════

  const userStored = { name: "Will", id: 105 };
  const anotherLink = userStored;  // COPY POINTER!

  STACK:                           HEAP:
  ┌────────────────────┐          ┌──────────────────┐
  │ userStored: @1001  │────┐     │ @1001:           │
  │ anotherLink: @1001 │────┘──→  │ name: "Will"     │
  └────────────────────┘          │ id: 105          │
                                  └──────────────────┘

  CẢ HAI trỏ tới CÙNG 1 OBJECT!
  → anotherLink === userStored → true! (cùng @1001!)

  NGUY HIỂM:
  ┌──────────────────────────────────────────────────┐
  │ anotherLink.name = "Hacked!";                    │
  │                                                  │
  │ console.log(userStored.name);                    │
  │ // "Hacked!" ← THAY ĐỔI THEO!                  │
  │                                                  │
  │ VÌ CẢ HAI TRỎ TỚI CÙNG 1 OBJECT @1001!         │
  │ Sửa qua anotherLink = sửa qua userStored!       │
  └──────────────────────────────────────────────────┘

  STACK:                           HEAP:
  ┌────────────────────┐          ┌──────────────────┐
  │ userStored: @1001  │────┐     │ @1001:           │
  │ anotherLink: @1001 │────┘──→  │ name: "Hacked!" │
  └────────────────────┘          │ id: 105          │
                                  └──────────────────┘

  → KHÔNG PHẢI backup!
  → KHÔNG PHẢI copy!
  → Chỉ là MỘT TÊN KHÁC cho CÙNG 1 object!
```

## §7. Deep Comparison — Phải Traverse Thủ Công!

### 7.1 Bài Viết: "No Built-In Comparison"

> Will: _"To compare the object's contents, there's NO SUCH built-in thing in JavaScript. I will have to manually TRAVERSE — go in and say grab userSubmitted.name, .id, pull them into the memory directly, where I can then make comparisons."_

```
DEEP COMPARISON — PHẢI TỰ VIẾT!
═══════════════════════════════════════════════════════════════

  ❌ KHÔNG THỂ:
  userSubmitted === userStored  → so sánh REFERENCE!

  ✅ PHẢI LÀM:
  → So sánh TỪNG property một!
  → Dùng primitives (by value → so sánh giá trị!)

  userSubmitted.name === userStored.name   // "Will" === "Will" → true
  userSubmitted.id === userStored.id       // 105 === 105 → true
  // CẢ HAI true → contents giống nhau!

  NHƯNG... nếu object lồng nhau?
  ┌──────────────────────────────────────────────────┐
  │ const a = {                                      │
  │   name: "Will",                                  │
  │   address: {                                     │
  │     city: "London",                              │
  │     zip: "SW1"                                   │
  │   }                                              │
  │ };                                               │
  │                                                  │
  │ const b = {                                      │
  │   name: "Will",                                  │
  │   address: {                                     │
  │     city: "London",                              │
  │     zip: "SW1"                                   │
  │   }                                              │
  │ };                                               │
  │                                                  │
  │ a.name === b.name → true (primitives!)           │
  │ a.address === b.address → false!                  │
  │ (2 objects KHÁC NHAU → khác reference!)          │
  │                                                  │
  │ → PHẢI RECURSIVE đi sâu từng level!             │
```

## §8. Deep Clone — Cùng Vấn Đề!

```
DEEP CLONE — COPY CONTENTS, KHÔNG PHẢI REFERENCE:
═══════════════════════════════════════════════════════════════

  ❌ SHALLOW COPY (chỉ copy 1 level):
  const backup = { ...userStored };
  // hoặc: Object.assign({}, userStored)

  → backup.name = primitives → COPIED!
  → backup.address = object → COPY REFERENCE! ⚠️

  ✅ DEEP CLONE (copy TẤT CẢ levels):
  → Phải traverse RECURSIVE!
  → JS modern: structuredClone(obj) ← ES2022!
  → Legacy: JSON.parse(JSON.stringify(obj)) ← hack!

  Will: "A deep clone — a deep copy — would be
  grabbing ALL of the elements, ALL of the properties,
  and maybe they have objects stored on them. You might
  see WHY having a flexible place (heap) for this data
  where it could have many embedded objects is sensible."

  SO SÁNH:
  ┌──────────────────────────────────────────────────┐
  │ const original = { a: 1, b: { c: 2 } };         │
  │                                                  │
  │ // Shallow copy:                                 │
  │ const shallow = { ...original };                 │
  │ shallow.a = 99;       // OK, independent!        │
  │ shallow.b.c = 99;     // ⚠️ CŨNG ĐỔI original!  │
  │ console.log(original.b.c); // 99! 💀             │
  │                                                  │
  │ // Deep clone:                                   │
  │ const deep = structuredClone(original);           │
  │ deep.a = 99;          // OK, independent!        │
  │ deep.b.c = 99;        // OK, independent! ✅     │
  │ console.log(original.b.c); // 2! ← giữ nguyên!  │
  └──────────────────────────────────────────────────┘
```

> Will: _"Even functions, technically, are not stored directly — because they are first-class objects — also stored in the heap. A lot of the time it doesn't have profound consequences, but when talking about comparisons...that's when it becomes incredibly important."_

```
FUNCTIONS = OBJECTS → CŨNG Ở HEAP!
═══════════════════════════════════════════════════════════════

  function onSubmit() { ... }

  STACK:                           HEAP:
  ┌────────────────────┐          ┌──────────────────┐
  │ onSubmit: @1003    │────────→ │ @1003:           │
  └────────────────────┘          │ ƒ onSubmit() {   │
                                  │   ...             │
                                  │ }                 │
                                  └──────────────────┘

  → Function = first-class OBJECT!
  → Lưu trong HEAP, không phải stack!

  │ fn1 === fn2 → false! (khác reference!)           │
```

## §10. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 10.1 Pattern ①: 5 Whys — Stack vs Heap

```
5 WHYS: TẠI SAO CÓ STACK VÀ HEAP?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao objects so sánh = false dù cùng nội dung?
  └→ Vì === so sánh REFERENCE (pointer), không phải content!
     Hai objects khác nhau → 2 vị trí khác nhau trong heap!

  WHY ②: Tại sao objects lưu bằng reference?
  └→ Vì objects có SIZE KHÔNG CỐ ĐỊNH!
     {a:1} = nhỏ, {a:1,...,z:26} = lớn!
     Stack cần biết size TRƯỚC → KHÔNG PHÙ HỢP!

  WHY ③: Tại sao stack cần biết size trước?
  └→ Vì stack = LIFO structure, cấp phát LIÊN TỤC!
     Mỗi frame = fixed size → push/pop nhanh O(1)!
     Object flexible → không fit vào stack model!

  WHY ④: Tại sao heap flexible hơn stack?
  └→ Vì heap = RANDOM ACCESS memory!
     Allocate BẤT KỲ ĐÂU, bất kỳ SIZE!
     Quản lý bằng GC (Garbage Collector)!
     Trade-off: chậm hơn stack nhưng linh hoạt!

  WHY ⑤: Tại sao primitives ở stack?
  └→ Vì primitives = FIXED SIZE!
     number = 8 bytes (IEEE 754)
     boolean = 1 byte
     → Stack biết chính xác cần bao nhiêu bộ nhớ!
     → Nhanh! Efficient! By value = independent copies!
```

### 10.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — ECMASCRIPT SPEC:
═══════════════════════════════════════════════════════════════

  7.2.15 IsStrictlyEqual(x, y):
  1. If Type(x) ≠ Type(y) → false
  2. If x is Number → Number::equal(x, y)
  3. Return SameValueNonNumber(x, y)

  SameValueNonNumber cho Objects:
  → "Return TRUE if x and y are THE SAME Object value"
  → "THE SAME" = cùng identity = cùng reference!
  → KHÔNG phải cùng contents!

  ECMAScript KHÔNG nói "stack" hay "heap" explicitly!
  → Spec nói "Object value" vs "primitive value"
  → Implementation (V8, SpiderMonkey) CHỌN stack/heap
  → NHƯNG behavior BY VALUE vs BY REFERENCE = spec-defined!

  V8 Engine internals:
  → Small numbers (Smi = Small Integer): stored inline
  → HeapNumber: 64-bit float → trong heap!
  → HeapObject: base class cho Object, String, Array...
  → EVERYTHING non-Smi = heap allocated!
```

### 10.3 Pattern ③: Trade-off Analysis

```
TRADE-OFFS: STACK vs HEAP:
═══════════════════════════════════════════════════════════════

  ┌──────────────┬──────────────────┬──────────────────┐
  │              │ STACK             │ HEAP              │
  ├──────────────┼──────────────────┼──────────────────┤
  │ Speed        │ Cực nhanh! O(1)  │ Chậm hơn (alloc) │
  │ Size         │ Nhỏ, cố định     │ Lớn, flexible    │
  │ Lifetime     │ Auto (scope end) │ GC manages        │
  │ Access       │ LIFO only        │ Random access     │
  │ Copy behavior│ By value (clone) │ By reference (ptr)│
  │ Compare      │ So sánh GIÁ TRỊ  │ So sánh ĐỊA CHỈ  │
  │ Thread-safe  │ Per-thread       │ Shared            │
  │ Stores       │ Primitives       │ Objects, arrays,  │
  │              │                  │ functions         │
  └──────────────┴──────────────────┴──────────────────┘

  Will: "Objects can combine multiple primitives and
  can be AS BIG AS WE WANT — stored in a flexible
  memory store called the heap."
```

### 10.4 Pattern ④: Mental Mapping

```
MENTAL MAP: BY REFERENCE TRONG CÁC NGÔN NGỮ:
═══════════════════════════════════════════════════════════════

  ┌─────────────────────────────────────────────────────────┐
  │ C — Explicit pointers!                                  │
  │                                                         │
  │ int x = 7;                                              │
  │ int *p = &x;    // p = ADDRESS of x!                   │
  │ *p = 42;         // dereference → x = 42!              │
  │                                                         │
  │ → C: developer THẤY pointers, điều khiển trực tiếp!   │
  │ → JS: pointers HIDDEN, auto-managed!                   │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │ PYTHON — Giống JS! Everything = reference!              │
  │                                                         │
  │ a = [1, 2, 3]                                           │
  │ b = a              # b = same reference!                │
  │ b.append(4)        # a = [1,2,3,4] too!                │
  │ a is b → True      # same identity!                    │
  │ a == b → True      # same contents! ← KHÁC JS!        │
  │                                                         │
  │ → Python == so sánh CONTENTS! (gọi __eq__)             │
  │ → Python is so sánh REFERENCE! (giống JS ===)          │
  │ → JS KHÔNG CÓ built-in content comparison!             │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │ REACT — Reference = Rendering decision!                 │
  │                                                         │
  │ setState(prevState) → new ref? → RE-RENDER!            │
  │ setState(sameRef) → same ref → NO RE-RENDER!           │
  │                                                         │
  │ ⚠️ GOTCHA:                                             │
  │ state.items.push(newItem);                              │
  │ setState(state);  // ❌ SAME reference → NO re-render! │
  │                                                         │
```

### 10.5 Pattern ⑤: Reverse Engineering

```javascript
// TỰ BUILD: Deep comparison từ đầu

function deepEqual(a, b) {
  // 1. Primitives → so sánh by value
  if (a === b) return true;

  // 2. Nếu 1 trong 2 là null/undefined → false
  if (a === null || b === null) return false;
  if (a === undefined || b === undefined) return false;

  // 3. Khác type → false
  if (typeof a !== typeof b) return false;

  // 4. NaN check (NaN !== NaN)
  if (typeof a === "number" && Number.isNaN(a) && Number.isNaN(b)) {
    return true;
  }

  // 5. Objects → RECURSIVE comparison!
  if (typeof a === "object") {
    const keysA = Object.keys(a);
    const keysB = Object.keys(b);

    // Khác số lượng keys → false
    if (keysA.length !== keysB.length) return false;

    // So sánh TỪNG key RECURSIVE
    for (const key of keysA) {
      if (!keysB.includes(key)) return false;
      if (!deepEqual(a[key], b[key])) return false; // RECURSIVE!
    }
    return true;
  }

  return false;
}

// Test:
const userStored = { name: "Will", id: 105 };
const userSubmitted = { name: "Will", id: 105 };

console.log(userStored === userSubmitted); // false (reference!)
console.log(deepEqual(userStored, userSubmitted)); // true ✅ (contents!)

// Nested objects:
const a = { user: { name: "Will", address: { city: "London" } } };
const b = { user: { name: "Will", address: { city: "London" } } };
console.log(a === b); // false
console.log(deepEqual(a, b)); // true ✅ (recursive!)

// Different contents:
const c = { name: "Will", id: 999 };
console.log(deepEqual(userStored, c)); // false ✅
```

```javascript
// TỰ BUILD: Deep clone từ đầu

function deepClone(obj) {
  // Primitives → trả về trực tiếp (by value!)
  if (obj === null || typeof obj !== "object") return obj;

  // Arrays
  if (Array.isArray(obj)) {
    return obj.map((item) => deepClone(item)); // RECURSIVE!
  }

  // Objects
  const clone = {};
  for (const key of Object.keys(obj)) {
    clone[key] = deepClone(obj[key]); // RECURSIVE!
  }
  return clone;
}

// Test:
const original = { a: 1, b: { c: 2, d: [3, 4, { e: 5 }] } };
const cloned = deepClone(original);

cloned.b.c = 99;
cloned.b.d[2].e = 99;

console.log(original.b.c); // 2 ← KHÔNG BỊ ẢNH HƯỞNG!
console.log(original.b.d[2].e); // 5 ← KHÔNG BỊ ẢNH HƯỞNG!
console.log(cloned.b.c); // 99
console.log(cloned.b.d[2].e); // 99

// Bonus: modern JS way
const modern = structuredClone(original);
console.log(deepEqual(original, modern)); // true
console.log(original === modern); // false (new reference!)
```
