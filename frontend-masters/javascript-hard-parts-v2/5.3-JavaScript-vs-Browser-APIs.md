### 6.2 Bài Viết: Facade = Accessor

> Will: _"You can see why I called it a facade function — because it LOOKS like it's doing a bunch of JavaScript type stuff. No! This is an ACCESSOR, a facade for functionality very much OUTSIDE of our JavaScript engine."_

```
FACADE FUNCTION — ANALYSIS:
═══════════════════════════════════════════════════════════════

  setTimeout(printHello, 1000);

  NHÌN BỀ NGOÀI:
  → Trông như gọi 1 JS function bình thường!
  → Trông giống createTimer() hay delay() tự viết!
  → KHÔNG CÓ GÌ cho thấy "đây không phải JS"!

  THỰC TẾ BÊN TRONG:
  → setTimeout KHÔNG tạo execution context!
  → setTimeout KHÔNG lưu gì trong JS memory!
  → setTimeout CHỈ nói với browser:
     "Bật timer 1000ms, khi xong chạy printHello!"
  → Rồi XONG — JS tiếp tục dòng tiếp theo!

  WHAT setTimeout ACTUALLY DOES IN JS:
  ┌──────────────────────────────────────────────────┐
  │ ① Lấy arguments (function + delay)              │
  │ ② Gửi xuống browser timer feature               │
  │ ③ Return timer ID (số, để cancel nếu cần)       │
  │ ④ DONE — line complete!                          │
  │                                                  │
  │ → KHÔNG có while loop đếm!                      │
  │ → KHÔNG block thread!                            │
  │ → KHÔNG tạo EC!                                  │
  └──────────────────────────────────────────────────┘

  COMPARISON:
  ┌──────────────────────────────────────────────────┐
  │ REGULAR FUNCTION:         FACADE FUNCTION:       │
  │ multiplyBy2(3)            setTimeout(fn, 1000)   │

  │ → DO WORK in JS!          → DO WORK in Browser!  │
  │ → RETURN value from JS!   → RETURN id only!      │
  │ → Synchronous!            → Async!               │
  └──────────────────────────────────────────────────┘
```

```javascript
// CHỨNG MINH 3: Simulate TWO WORLDS

class JavaScriptEngine {
  constructor(browser) {
    this.memory = {};
    this.callStack = ["global()"];
    this.browser = browser;
    this.time = 0;
  }

  declare(name, value) {
    this.memory[name] = value;
    console.log(`[JS ${this.time}ms] Declare: ${name}`);
  }

  // Facade function — does work in BROWSER!
  setTimeout(callback, delay) {
    console.log(`[JS ${this.time}ms] setTimeout called!`);
    console.log(`  → NOT creating EC in JS!`);
    console.log(`  → Sending to BROWSER timer!`);

    // Send to browser:
    this.browser.startTimer(callback, delay, this.time);

    // Return timer ID (only JS side-effect):
    const id = Math.floor(Math.random() * 1000);
    console.log(`  → Returns timer ID: ${id}`);
    console.log(`  → JS LINE COMPLETE! Moving on...`);
    return id;
  }

  consoleLog(message) {
    this.time++;
    console.log(`[JS ${this.time}ms] console.log("${message}")`);
    this.browser.console.push({ time: this.time, message });
  }

  // When browser sends callback back:
  executeCallback(funcName, func) {
    console.log(`[JS ${this.time}ms] ${funcName}() → CALL STACK!`);
    this.callStack.push(`${funcName}()`);
    console.log(`  Stack: [${this.callStack.join(" → ")}]`);
    func(this); // execute!
    this.callStack.pop();
    console.log(`  Stack: [${this.callStack.join(" → ")}]`);
  }
}

class WebBrowser {
  constructor() {
    this.timers = [];
    this.console = [];
  }

  startTimer(callback, duration, startTime) {
    console.log(`[BROWSER] Timer started!`);
    console.log(`  Duration: ${duration}ms`);
    console.log(`  onCompletion: ${callback.name || "anonymous"}`);
    console.log(`  Complete? ❌ (${startTime}ms < ${duration}ms)`);

    this.timers.push({
      callback,
      duration,
      startTime,
      complete: false,
    });
  }

  tick(currentTime, jsEngine) {
    for (const timer of this.timers) {
      if (!timer.complete && currentTime >= timer.startTime + timer.duration) {
        timer.complete = true;
        console.log(`\n[BROWSER ${currentTime}ms] Timer COMPLETE! ✅`);
        console.log(`  → Sending ${timer.callback.name} back to JS!`);
        jsEngine.time = currentTime;
        jsEngine.executeCallback(timer.callback.name, timer.callback);
      }
    }
  }
}

// RUN THE SIMULATION:
console.log("═══ SIMULATION START ═══\n");
const browser = new WebBrowser();
const js = new JavaScriptEngine(browser);

// Line 1: function printHello
js.declare("printHello", function printHello(engine) {
  engine.consoleLog("Hello");
});

// Line 5: setTimeout(printHello, 1000)
js.setTimeout(js.memory.printHello, 1000);

// Line 7: console.log("Me first!")
js.consoleLog("Me first!");

console.log(`\n[JS] Global code DONE! Waiting for browser...`);

// Time passes...
browser.tick(1000, js); // At 1000ms, timer fires!

console.log("\n═══ FINAL CONSOLE OUTPUT ═══");
browser.console.forEach((entry) => {
  console.log(`  [${entry.time}ms] ${entry.message}`);
});
```

## §8. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 8.1 Pattern ①: 5 Whys — setTimeout Design

```
5 WHYS: TẠI SAO setTimeout HOẠT ĐỘNG NHƯ VẬY?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao setTimeout không block?
  └→ Vì timer KHÔNG chạy trong JS thread!
     Timer chạy trong browser's timer thread!
     JS thread FREE để tiếp tục!

  WHY ②: Tại sao timer ở browser, không ở JS?
  └→ Vì JS single-threaded! Nếu đếm trong JS
     → thread bị CHIẾM → UI freeze!
     Browser = multi-threaded → đếm song song!

  WHY ③: Tại sao callback PHẢI chờ stack empty?
  └→ Vì JS single-threaded = chỉ 1 việc/lúc!
     Nếu callback nhảy vào giữa chừng → race condition!
     Phải chờ code hiện tại XONG → predictable!

  WHY ④: Tại sao trả về timer ID?
  └→ Vì developer cần CONTROL!
     clearTimeout(id) → cancel timer!
     Useful cho: debounce, abort, cleanup!

  WHY ⑤: Tại sao setTimeout(fn, 0) vẫn chậm hơn sync?
  └→ Vì callback qua callback queue → event loop!
     Event loop CHỈ check khi call stack EMPTY!
     → Sync code ALWAYS runs first!
     → Sẽ học chi tiết ở bài Event Loop!
```

### 8.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — HTML SPEC (WHATWG):
═══════════════════════════════════════════════════════════════

  HTML Living Standard — 8.6 Timers:

  "setTimeout(handler, timeout)"
  → Schedule a callback to run after timeout ms
  → Returns a positive integer (timer ID)
  → Timer is tracked by the USER AGENT (browser!)
  → NOT by the JavaScript engine!

  Algorithm:
  1. Let handler be the callback function
  2. Let timeout be the delay value
  3. Let nesting level = current timer nesting
  4. If nesting > 5 AND timeout < 4ms → clamp to 4ms!
     → = Tại sao setTimeout(fn, 0) ≠ thực sự 0ms!
     → Minimum = 4ms (for nested timers!)
  5. Return timer handle (ID number)
  6. Run steps IN PARALLEL:
     → Wait until timeout ms elapsed
     → Queue a task to run handler

  KEY: "run steps IN PARALLEL"
  → = Browser thread! NOT JS thread!
  → = Tại sao non-blocking!
```

```
  ┌─────────────────────────────────────────────────────────┐
  │ BROWSER:                                                │
  │ setTimeout → Browser's Timer Thread                     │
  │ → Uses OS timer APIs (high-precision timers!)           │
  │ → Event loop picks up callback                          │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │ NODE.JS:                                                │
  │ setTimeout → libuv Timer Handle                         │
  │ → Uses OS event notification (epoll/kqueue/IOCP)        │
  │ → Node event loop picks up callback                     │
  │ → (same concept, different implementation!)             │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │ DENO:                                                   │
  │ setTimeout → Rust tokio runtime                         │
  │ → Uses mio (platform event loop)                        │
  │ → Same model, Rust implementation!                     │
  └─────────────────────────────────────────────────────────┘

  → EVERY runtime: timer = OUTSIDE JS engine!
  → JS engine = NEVER does the counting!
  → Host environment (browser/node/deno) = does the work!
```
