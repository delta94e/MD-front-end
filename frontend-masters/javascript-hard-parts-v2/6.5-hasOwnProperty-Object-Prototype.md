## §5. **proto** vs .prototype — "Naming Conventions Are ROUGH!"

> Will: _"This is where I find JavaScript's naming conventions a bit ROUGH."_

```
THE NAMING CONFUSION — RESOLVED!:
═══════════════════════════════════════════════════════════════

  ⚠️ TWO DIFFERENT THINGS WITH SIMILAR NAMES!

  ① __proto__ (HIDDEN property on EVERY object):
  ┌──────────────────────────────────────────────────┐
  │ → Also known as: [[Prototype]]                  │
  │ → Lives on: EVERY object!                       │
  │ → Purpose: LINK to prototype object!            │
  │ → Hidden: Yes! (not enumerable)                │
  │ → Access: obj.__proto__ (legacy, deprecated!)   │
  │ → Access: Object.getPrototypeOf(obj) (modern!)  │
  │                                                  │
  │ Will: "I used to call this [[proto]]. You can    │
  │ still access it via __proto__, a legacy,         │
  │ almost deprecated way. The SPEC defines it       │
  │ as [[Prototype]]."                               │
  └──────────────────────────────────────────────────┘

  ② .prototype (REGULAR property on FUNCTIONS):
  ┌──────────────────────────────────────────────────┐
  │ → Lives on: Function objects (not all objects!)  │
  │ → Purpose: Object that new instances link to!   │
  │ → Hidden: NO! Regular, visible property!        │
  │ → Access: MyFunc.prototype                      │
  │                                                  │
  │ Object.prototype = the actual OBJECT             │
  │ where hasOwnProperty and other built-in          │
  │ functions live!                                  │
  └──────────────────────────────────────────────────┘

  THE CONFUSION:
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │ Object.prototype                                 │
  │ ↑                 ↑                              │
  │ │                 │                              │
  │ "Object" is a    ".prototype" is a REGULAR      │
  │ built-in object!  property on that object!       │
  │                   It holds the object with       │
  │                   hasOwnProperty, toString, etc! │
  │                                                  │
  │ user1.__proto__                                  │
  │ ↑         ↑                                      │
  │ │         │                                      │
  │ our       "__proto__" is the HIDDEN              │
  │ object!   link to the prototype object!          │
  │                                                  │
  │ THEY ARE DIFFERENT THINGS!                       │
  │ • __proto__ = hidden link (on every object)     │
  │ • .prototype = regular prop (on functions)      │
  │                                                  │
  │ Will: "Object.prototype is accessed by all       │
  │ objects. Hidden prototype REFERENCE to that       │
  │ object, and you can see how the naming is rough."│
  │                                                  │
  └──────────────────────────────────────────────────┘
```

```
CHEAT SHEET:
```

## §8. Tự Implement: Full Prototype Chain Traversal

```javascript
// CHỨNG MINH 1: The full chain lookup!

const userFunctionStore = {
  increment() {
    this.score++;
  },
  login() {
    console.log(`${this.name} logged in`);
  },
};

function userCreator(name, score) {
  const newUser = Object.create(userFunctionStore);
  newUser.name = name;
  newUser.score = score;
  return newUser;
}

const user1 = userCreator("Ari", 3);

// Own properties:
console.log(user1.hasOwnProperty("name")); // true ✅
console.log(user1.hasOwnProperty("score")); // true ✅
console.log(user1.hasOwnProperty("increment")); // false! ❌
// → increment is NOT on user1! It's on the prototype!

// The full chain:
console.log(Object.getPrototypeOf(user1) === userFunctionStore); // true!
console.log(Object.getPrototypeOf(userFunctionStore) === Object.prototype); // true!
console.log(Object.getPrototypeOf(Object.prototype) === null); // true! END!
```

```javascript
// CHỨNG MINH 2: Simulate the FULL chain walk!

function walkPrototypeChain(obj, prop) {
  let current = obj;
  let depth = 0;

  while (current !== null) {
    depth++;
    const label =
      depth === 1
        ? "object itself"
        : current === Object.prototype
          ? "Object.prototype"
          : `prototype level ${depth - 1}`;

    if (current.hasOwnProperty(prop)) {
      console.log(`✅ [${label}] Found "${prop}"!`);
      return current[prop];
    }

    console.log(`❌ [${label}] "${prop}" not here!`);
    current = Object.getPrototypeOf(current);
  }

  console.log(`🛑 Reached null — "${prop}" not found anywhere!`);
  return undefined;
}

// Test with user1:
const store = {
  increment() {
    this.score++;
  },
};
const user1 = Object.create(store);
user1.name = "Ari";
user1.score = 3;

console.log("--- Looking for 'name': ---");
walkPrototypeChain(user1, "name");
// ✅ [object itself] Found "name"!

console.log("\n--- Looking for 'increment': ---");
walkPrototypeChain(user1, "increment");
// ❌ [object itself] "increment" not here!
// ✅ [prototype level 1] Found "increment"!

console.log("\n--- Looking for 'hasOwnProperty': ---");
walkPrototypeChain(user1, "hasOwnProperty");
// ❌ [object itself] "hasOwnProperty" not here!
// ❌ [prototype level 1] "hasOwnProperty" not here!
// ✅ [Object.prototype] Found "hasOwnProperty"!

console.log("\n--- Looking for 'nonexistent': ---");
walkPrototypeChain(user1, "nonexistent");
// ❌ [object itself] "nonexistent" not here!
// ❌ [prototype level 1] "nonexistent" not here!
// ❌ [Object.prototype] "nonexistent" not here!
// 🛑 Reached null — "nonexistent" not found anywhere!
```

## §9. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 9.1 Pattern ①: 5 Whys — Object.prototype

```
5 WHYS: TẠI SAO CẦN Object.prototype?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao hasOwnProperty không trên user1?
  └→ Vì ta chỉ add name + score! Không add utilities!

  WHY ②: Tại sao không trên userFunctionStore?
  └→ Vì ta chỉ add increment + login! Domain methods!
     hasOwnProperty = UNIVERSAL utility!

  WHY ③: Tại sao cần Object.prototype?
  └→ Vì MỌI object đều cần hasOwnProperty, toString!
     Không lẽ copy vào MỌI function store!
     ONE central place = Object.prototype!

  WHY ④: Tại sao chain ends at null?
  └→ Vì phải END somewhere! Infinite loop = bug!
     null = explicit "no more prototypes"!
     Return undefined for truly non-existent props!

  WHY ⑤: Tại sao userFunctionStore auto-links?
  └→ Vì ALL objects created in JS auto-get __proto__!
     Default __proto__ → Object.prototype!
     EXCEPT Object.create(null) → __proto__ = null!
```

### 9.2 Pattern ②: First Principles

```
FIRST PRINCIPLES — PROPERTY LOOKUP ALGORITHM:
═══════════════════════════════════════════════════════════════

  ECMAScript OrdinaryGet(O, P, Receiver):

  1. desc = O.OwnPropertyKeys → find P?
  2. If found → return desc.[[Value]]!
  3. parent = O.[[GetPrototypeOf]]()
  4. If parent === null → return undefined! 🛑
  5. Return parent.[[Get]](P, Receiver) ← RECURSIVE!

  CHAIN: 3 levels deep (our case):
  user1 → userFunctionStore → Object.prototype → null

  GENERAL CHAIN:
  instance → custom prototype → ... → Object.prototype → null

  IMPORTANT: `this` (Receiver) stays as original!
  → user1.hasOwnProperty("score")
  → Found on Object.prototype BUT this = user1!
  → So it correctly checks user1's own properties!
```

### 9.3 Pattern ③: Trade-off Analysis

```
TRADE-OFFS — CHAIN DEPTH:
═══════════════════════════════════════════════════════════════

  ┌────────────────┬─────────────────┬───────────────────────┐
  │                │ Shallow (1 lvl) │ Deep (3+ levels)      │
  ├────────────────┼─────────────────┼───────────────────────┤
  │ Lookup speed   │ ✅ Fast!        │ ⚠️ Slower (more hops) │
  │ Organization   │ ⚠️ All in one   │ ✅ Separated by type  │
  │ Extensibility  │ ⚠️ Limited      │ ✅ Rich inheritance   │
  │ Debugging      │ ✅ Simple       │ ⚠️ Complex chain      │
  │ Memory         │ ✅ Efficient    │ ✅ Still efficient     │
  └────────────────┴─────────────────┴───────────────────────┘

  Our chain (3 levels) is the sweet spot!
  → user methods on userFunctionStore
  → universal methods on Object.prototype
  → Clean separation of concerns!
```

### 9.5 Pattern ⑤: Reverse Engineering

```javascript
// PROVE: Every type has a chain ending at Object.prototype

const types = [
  { name: "Object", val: {} },
  { name: "Array", val: [] },
  { name: "Function", val: function () {} },
  { name: "String", val: new String("hi") },
  { name: "Number", val: new Number(42) },
  { name: "RegExp", val: /abc/ },
  { name: "Date", val: new Date() },
];

types.forEach(({ name, val }) => {
  let chain = [];
  let current = val;
  while (current !== null) {
    chain.push(current.constructor?.name || "null");
    current = Object.getPrototypeOf(current);
  }
  chain.push("null");
  console.log(`${name}: ${chain.join(" → ")}`);
});
```
