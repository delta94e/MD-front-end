## §1. Code Gốc — copyArrayAndManipulate + multiplyBy2

```javascript
function copyArrayAndManipulate(array, instructions) {
  const output = [];
  for (let i = 0; i < array.length; i++) {
    output.push(instructions(array[i]));
  }
  return output;
}

function multiplyBy2(input) {
  return input * 2;
}

const result = copyArrayAndManipulate([1, 2, 3], multiplyBy2);
```

### 1.1 Bài Viết: Đọc Code Trước Khi Chạy

Nhìn lại code, có 2 functions:

- **`copyArrayAndManipulate`** — Higher-Order Function (HOF): nhận array + instructions (một function), duyệt mảng, gọi instructions cho mỗi phần tử
- **`multiplyBy2`** — Callback: nhận 1 số, nhân đôi, return

Will xây function này **từng dòng** trước mặt học viên, nhấn mạnh: _"We haven't seen `multiplyBy2` in this code yet"_ — nó phải được **khai báo riêng**, sau đó **truyền vào** copyArrayAndManipulate.

> **Key insight:** Khi truyền `multiplyBy2` vào, bên trong function nó **đổi tên** thành `instructions`. Label `multiplyBy2` **mất**, chỉ còn `instructions`.

## §4. Full Trace #3: For Loop — 3 Nested Execution Contexts

### 4.1 Bài Viết: Mỗi instructions() = 1 EC Mới!

Bên trong for loop, mỗi lần gọi `instructions(array[i])`, JS sẽ:

1. Evaluate `instructions` → tìm function code trong local memory
2. Evaluate `array[i]` → tìm giá trị tại vị trí i
3. Gọi function → **TẠO Execution Context MỚI** (nested EC!)
4. Push lên Call Stack → nay có 3 layers: Global → copyArray → instructions
5. Return giá trị → Pop instructions khỏi stack
6. Push giá trị vào output

```
LẦN 1: i = 0 → instructions(array[0]) = instructions(1)
═══════════════════════════════════════════════════════════════

  EVALUATE dòng "instructions(array[i])":
  ┌───────────────────────────────────────────────────────────┐
  │ instructions(array[i])                                    │
  │      ↓              ↓                                     │
  │  evaluate:      evaluate:                                 │
  │  → tìm trong    → array = [1,2,3]                        │
  │    local memory  → array[0] = 1                           │
  │  → ƒ(input) {   → GIÁ TRỊ: 1                            │
  │      return      ↓                                        │
  │      input * 2   thực chất đang gọi:                      │
  │    }             multiplyBy2(1)                            │
  └───────────────────────────────────────────────────────────┘

  TẠO NESTED EXECUTION CONTEXT:

  CALL STACK:                    EC: instructions(1)
  ┌────────────────────┐        ┌──────────────────────┐
  │ instructions(1) ←──│── top  │ LOCAL MEMORY:        │
  ├────────────────────┤        │  input: 1            │
  │ copyArray&Manip... │        │                      │
  ├────────────────────┤        │ THREAD:              │
  │ Global()           │        │  return 1 * 2 → 2    │
  └────────────────────┘        └──────────────────────┘

  Return 2 → POP instructions(1) off Call Stack!

  CALL STACK:                    LOCAL MEMORY (copyArray):
  ┌────────────────────┐        ┌──────────────────────┐
  │ ╳ instructions(1)  │→ pop   │ output: [2] ← push! │
```

## §6. Insight: Label Thay Đổi — multiplyBy2 → instructions

### 6.1 Bài Viết: "Previously Known As" — Sự Biến Đổi Của Label

Đây là insight **quan trọng nhất** mà Will nhấn mạnh nhiều lần:

Khi bạn truyền `multiplyBy2` vào `copyArrayAndManipulate`, function code được **COPY** vào local memory dưới label **mới** là `instructions`. Label cũ `multiplyBy2` **không tồn tại** bên trong EC.

> Will: _"This was previously known as multiplyBy2, but we know that we can't refer to it as that. It's lost that name, it doesn't care about that name."_

**Tại sao quan trọng?** Vì điều này cho phép **BẤT KỲ function nào** được truyền vào:

```
LABEL TRANSFORMATION:
═══════════════════════════════════════════════════════════════

  BÊN NGOÀI:                    BÊN TRONG copyArray&Manipulate:
  ┌───────────────┐              ┌───────────────┐
  │ multiplyBy2   │──argument──→ │ instructions   │
  │ = ƒ(input) {  │              │ = ƒ(input) {   │
  │   return      │   label      │   return        │
  │   input * 2   │   changes!   │   input * 2     │
  │ }             │              │ }               │
  └───────────────┘              └───────────────┘

  → CÙNG function code
  → KHÁC label (multiplyBy2 → instructions)
  → Bên trong EC, CHỈ biết "instructions"!

  VÀ ĐÂY LÀ SUC MẠNH:
  ┌──────────────────────────────────────────────────────┐
  │ copyArrayAndManipulate([1,2,3], multiplyBy2)         │
  │   → instructions = ƒ multiplyBy2 → [2, 4, 6]       │
  │                                                      │
  │ copyArrayAndManipulate([1,2,3], divideBy2)           │
  │   → instructions = ƒ divideBy2   → [0.5, 1, 1.5]   │
  │                                                      │
  │ copyArrayAndManipulate([1,2,3], add3)                │
  │   → instructions = ƒ add3        → [4, 5, 6]        │
  │                                                      │
  │ → Parameter "instructions" = placeholder              │
  │ → Mỗi lần gọi, truyền function KHÁC vào!            │
  │ → CÙNG HOF, VÔ HẠN behaviors!                       │
  └──────────────────────────────────────────────────────┘
```

### 6.2 Evaluation — Mọi Label Phải Trở Thành Value

Will nhấn mạnh: _"Nothing in JavaScript when it's running stays a label. It has to be evaluated, turned into whatever it actually is in memory."_

```
EVALUATION — LABELS → VALUES:
═══════════════════════════════════════════════════════════════

  Code:  output.push( instructions( array[i] ) );

  Bước 1: Evaluate từ trong ra ngoài:
  ┌─────────────────────────────────────────────────────┐
  │                                                     │
  │  array[i]                                           │
  │    ↓ evaluate array → [1,2,3]                       │
  │    ↓ evaluate i → 0                                 │
  │    ↓ [1,2,3][0] → 1                                │
  │    = GIÁ TRỊ: 1                                    │
  │                                                     │
  │  instructions(1)                                    │
  │    ↓ evaluate instructions → ƒ(input){return input*2}│
  │    ↓ GỌI function với argument 1                    │
  │    ↓ TẠO EC → input=1 → 1*2=2 → return 2           │
  │    = GIÁ TRỊ: 2                                    │
  │                                                     │
  │  output.push(2)                                     │
  │    ↓ evaluate output → [] (mảng hiện tại)           │
  │    ↓ push 2 vào mảng                                │
  │    = output giờ là [2]                              │
  │                                                     │
  └─────────────────────────────────────────────────────┘

  → MỌI THỨ đều phải evaluate trước khi dùng!
  → "instructions" = label → evaluate → function code
  → "array[i]" = label → evaluate → số
  → Parentheses () trên instructions = GỌI function!
```

## §7. "How Do We Wrap Functionality?" — Câu Hỏi Then Chốt

### 7.1 Bài Viết: Function = Bọc Functionality

Will hỏi lớp nhiều lần: _"How do we wrap up functionality that we want to pass around in JavaScript?"_ → Cả lớp trả lời: **"FUNCTION!"**

Đây là câu hỏi then chốt. Trong JS:

- **Số, chuỗi, boolean** = data values → truyền qua parameter
- **Operators** (+, -, \*) = syntax → KHÔNG truyền được
- **Functionality** = cần **bọc** trong function → truyền function

```
FUNCTION = CÁCH DUY NHẤT BỌC FUNCTIONALITY:
═══════════════════════════════════════════════════════════════

  Bạn muốn truyền "NHÂN ĐÔI" vào function:
  ┌──────────────────────────────────────────────────────┐
  │                                                      │
  │ ❌ Truyền operator: copyArray(arr, * 2)              │
  │    → SyntaxError! * 2 không phải value!              │
  │                                                      │
  │ ❌ Truyền string:  copyArray(arr, "* 2")             │
  │    → Phải dùng eval() → NGUY HIỂM!                  │
  │    → Will: "very high risk"                          │
  │                                                      │
  │ ✅ Truyền function: copyArray(arr, x => x * 2)      │
  │    → Function = VALUE → truyền bình thường!          │
  │    → Bên trong GỌI bằng () → chạy code!             │
  │                                                      │
  └──────────────────────────────────────────────────────┘

  PHÉP ẨN DỤ:
  ┌──────────────────────────────────────────────────────┐
  │                                                      │
  │  Operator = HÀNH ĐỘNG (verb) → không truyền được     │
  │  Function = HỘP CHỨA HÀNH ĐỘNG → truyền được!      │
  │                                                      │
  │  Bạn không thể gửi "sơn" qua email                  │
  │  Nhưng bạn có thể gửi "hộp sơn" → người nhận mở    │
  │  hộp (gọi function) → có sơn (chạy code)!           │
  │                                                      │
  │  Function = HỘP                                      │
  │  Parentheses () = MỞ HỘP                             │
  │  Code bên trong = NỘI DUNG                           │
  └──────────────────────────────────────────────────────┘
```

## §8. Const & Uninitialized — Chi Tiết Nhỏ, Ý Nghĩa Lớn

### 8.1 Bài Viết: Tại Sao const result = Uninitialized?

Will giải thích chi tiết kỹ thuật: với `const`, biến **không thể** được gán `undefined` rồi đổi thành giá trị khác — vì const = không đổi data type!

```
CONST vs VAR — INITIALIZATION:
═══════════════════════════════════════════════════════════════

  VAR (ES5):
  ┌──────────────────────────────────────────────────────┐
  │ var result = someFunction();                         │
  │                                                      │
  │ HOISTING:                                            │
  │ ① Khai báo: result = undefined  (hoisted)            │
  │ ② Chạy:     result = someFunction()                 │
  │ → Data type: undefined → array                       │
  │ → Cho phép vì var = mutable                          │
  └──────────────────────────────────────────────────────┘

  CONST (ES6):
  ┌──────────────────────────────────────────────────────┐
  │ const result = someFunction();                       │
  │                                                      │
  │ BEHAVIOR:                                            │
  │ ① Khai báo: result = ⟨uninitialized⟩ ← KHÔNG phải   │
  │                                         undefined!    │
  │ ② Chạy:     result = someFunction()                  │
  │ → KHÔNG qua trung gian undefined                     │
  │ → "Temporal Dead Zone" cho đến khi được gán          │
  │ → Truy cập trước khi gán → ReferenceError!          │
  └──────────────────────────────────────────────────────┘

  ⚠️ Will: "Used to be undefined. But now with const,
     we definitely can't change a data type from
     undefined to the return value."
```

## §9. Tự Implement: copyArrayAndManipulate + Full Runtime Trace

```javascript
// TỰ XÂY LẠI TOÀN BỘ WALKTHROUGH — code chạy được!

// ═══ GIẢI PHÁP CUỐI CÙNG: copyArrayAndManipulate ═══

function copyArrayAndManipulate(array, instructions) {
  const output = [];
  for (let i = 0; i < array.length; i++) {
    output.push(instructions(array[i]));
  }
  return output;
}

// Callbacks — mỗi cái bọc 1 hành vi:
function multiplyBy2(input) {
  return input * 2;
}
function divideBy2(input) {
  return input / 2;
}
function add3(input) {
  return input + 3;
}
function square(input) {
  return input * input;
}

// 1 HOF, 4 behaviors:
console.log(copyArrayAndManipulate([1, 2, 3], multiplyBy2)); // [2, 4, 6]
console.log(copyArrayAndManipulate([1, 2, 3], divideBy2)); // [0.5, 1, 1.5]
console.log(copyArrayAndManipulate([1, 2, 3], add3)); // [4, 5, 6]
console.log(copyArrayAndManipulate([1, 2, 3], square)); // [1, 4, 9]
```

```javascript
// TỰ XÂY LẠI VỚI FULL RUNTIME TRACE — mô phỏng EC + Call Stack

class RuntimeTracer {
  constructor() {
    this.callStack = ["Global()"];
    this.logs = [];
    this.indent = 0;
  }

  log(msg) {
    this.logs.push("  ".repeat(this.indent) + msg);
  }

  pushStack(name) {
    this.callStack.push(name);
    this.log(`[PUSH] ${name}`);
    this.log(`  Call Stack: [${this.callStack.join(" → ")}]`);
  }

  popStack() {
    const removed = this.callStack.pop();
    this.log(`[POP] ${removed}`);
    this.log(`  Call Stack: [${this.callStack.join(" → ")}]`);
    return removed;
  }

  printAll() {
    this.logs.forEach((l) => console.log(l));
  }
}

function tracedCopyArrayAndManipulate(array, instructions, tracer) {
  tracer.pushStack("copyArrayAndManipulate");
  tracer.indent++;

  tracer.log(`\nLOCAL MEMORY:`);
  tracer.log(`  array: [${array}]`);
  tracer.log(`  instructions: ƒ ${instructions.name}`);
  tracer.log(`  → "multiplyBy2" RENAMED to "instructions"!`);

  const output = [];
  tracer.log(`  output: [] (empty)\n`);

  for (let i = 0; i < array.length; i++) {

    tracer.log(`  array[${i}] → evaluate → ${array[i]}`);
    tracer.log(`  instructions → evaluate → ƒ ${instructions.name}`);
    tracer.log(`  → Calling: instructions(${array[i]})`);

    // Nested EC cho instructions!
    tracer.pushStack(`instructions(${array[i]})`);
    tracer.indent++;
    tracer.log(`  LOCAL: input = ${array[i]}`);

    const result = instructions(array[i]);

    tracer.log(`  RETURN: ${array[i]} → ${result}`);
    tracer.indent--;
    tracer.popStack();

    output.push(result);
    tracer.log(`  output.push(${result}) → output = [${output}]\n`);
  }

  tracer.log(`RETURN output → [${output}]`);
  tracer.indent--;
  tracer.popStack();

  return output;
}
```

## §10. 🔬 Deep Analysis Patterns — 6 Tư Duy Phân Tích Sâu

### 10.1 Pattern ①: 5 Whys — Nested Execution Contexts

```
5 WHYS: TẠI SAO INSTRUCTIONS TẠO EC MỚI?
═══════════════════════════════════════════════════════════════

  WHY ①: Tại sao instructions(array[i]) tạo EC mới?
  └→ Vì instructions là FUNCTION, dấu () = GỌI function,
     mọi function call = Execution Context mới!

  WHY ②: Tại sao mỗi lần trong loop lại tạo EC MỚI?
  └→ Vì mỗi call có argument KHÁC (1, 2, 3).
     EC cần local memory RIÊNG cho input khác nhau.

  WHY ③: Vậy 3 lần loop = 3 ECs tạo rồi xoá?
  └→ ĐÚNG! Push → chạy → return → pop → GC xoá.
     Input 1000 phần tử = 1000 ECs! Nhưng mỗi cái
     chỉ sống vài microsecond → GC handle tốt.

  WHY ④: Tại sao Call Stack lúc này depth = 3?
  └→ Global + HOF + callback = 3 levels.
     Nếu callback GỌI function khác = 4 levels!
     Đệ quy trong callback = nguy cơ stack overflow.

  WHY ⑤: Tại sao V8 không tối ưu bỏ EC cho callback nhỏ?
  └→ Thực ra V8 CÓ tối ưu! TurboFan JIT compiler sẽ
     INLINE callback nhỏ → bỏ EC → thực chất chạy
     như for loop thường → gần = hiệu năng C!
     ⭐ Đó là lý do .map() nhanh gần bằng for loop.
```

### 10.5 Pattern ⑤: Reverse Engineering & Implementation

```javascript
// TỰ BUILD: GENERIC HOF FACTORY — tạo HOF từ pattern

function createArrayHOF(strategy) {
  // strategy = object mô tả HOF gồm: init, process, finalize

  return function (array, callback) {
    let accumulator = strategy.init();

    for (let i = 0; i < array.length; i++) {
      accumulator = strategy.process(accumulator, callback, array[i], i);
    }

    return strategy.finalize(accumulator);
  };
}

// Tạo MAP từ factory:
const myMap = createArrayHOF({
  init: () => [],
  process: (acc, cb, el) => {
    acc.push(cb(el));
    return acc;
  },
  finalize: (acc) => acc,
});

// Tạo FILTER từ factory:
const myFilter = createArrayHOF({
  init: () => [],
  process: (acc, cb, el) => {
    if (cb(el)) acc.push(el);
    return acc;
  },
  finalize: (acc) => acc,
});

// Tạo REDUCE từ cùng factory:
// (cần tweak nhẹ vì reduce có initialValue)
function myReduce(array, callback, initial) {
  let acc = initial;
  for (let i = 0; i < array.length; i++) {
    acc = callback(acc, array[i], i);
  }
  return acc;
}

// TEST:
console.log(myMap([1, 2, 3], (x) => x * 2)); // [2, 4, 6]
console.log(myFilter([1, 2, 3], (x) => x > 1)); // [2, 3]
console.log(myReduce([1, 2, 3], (a, x) => a + x, 0)); // 6

// ⭐ map, filter = cùng pattern, chỉ khác strategy!
// ⭐ "What I cannot create, I do not understand" ✅
```
